# Страница 4. Feed-Forward Network в GPT-2

## 1. Что мы строим на этой странице

На предыдущих страницах мы уже прошли несколько уровней работы языковой модели:

1. научились превращать текст в токены;
2. разобрали token IDs;
3. разобрали embeddings токенов;
4. добавили информацию о позиции;
5. научились генерировать следующий токен;
6. соединили encoder, модель, sampling и decoder.

Теперь мы начинаем собирать **сам Transformer изнутри**.

Первый компонент, который нужно реализовать, — Feed-Forward Network, или FFN.

В рамках задания нужно написать три функции:

```python
linear(x, w, b)
gelu(x)
ffn(x, c_fc, c_proj)
```

Итоговая логика FFN:

```text
входной embedding
    ↓
линейное расширение размерности
    ↓
GELU
    ↓
линейное сжатие обратно
    ↓
выходной embedding
```

В математическом виде:

\[
FFN(x) =
\operatorname{Linear}_{proj}
\left(
\operatorname{GELU}
\left(
\operatorname{Linear}_{fc}(x)
\right)
\right)
\]

Для GPT-2 Small:

```text
768 → 3072 → 768
```

То есть первый линейный слой расширяет embedding в четыре раза, а второй возвращает исходную размерность.

# Часть II. Что такое нейронная сеть

## 4. Нейрон как математическая операция

Упрощённый искусственный нейрон получает несколько чисел:

\[
x_1, x_2, \ldots, x_n
\]

У каждого входа есть вес:

\[
w_1, w_2, \ldots, w_n
\]

Нейрон вычисляет:

\[
z = x_1w_1 + x_2w_2 + \ldots + x_nw_n + b
\]

Затем применяет функцию активации:

\[
a = \phi(z)
\]

где:

- \(x_i\) — входные признаки;
- \(w_i\) — обучаемые веса;
- \(b\) — bias;
- \(\phi\) — нелинейная функция;
- \(a\) — выход нейрона.

---

## 5. Зачем нужен bias

Без bias:

\[
z = xW
\]

линейное преобразование всегда проходит через начало координат.

Bias позволяет сдвинуть преобразование:

\[
z = xW + b
\]

Простой одномерный пример:

\[
y = 2x
\]

проходит через точку \((0, 0)\).

А:

\[
y = 2x + 5
\]

сдвинута вверх на 5.

Bias даёт модели дополнительную гибкость.

---

# Часть III. Линейный слой

## 6. Формула

В задании линейный слой реализуется как:

\[
Y = XW + b
\]

Код:

```python
return x @ w + b
```

Знак `@` в Python означает матричное умножение.

---

## 7. Размерности

В комментарии шаблона:

```python
# [m, in], [in, out], [out] -> [m, out]
```

Это означает:

```text
x: [m, in]
w: [in, out]
b: [out]
результат: [m, out]
```

Где:

- `m` — число объектов или токенов;
- `in` — входная размерность;
- `out` — выходная размерность.

---

## 8. Пример маленьких матриц

Пусть:

```python
x.shape == (2, 3)
```

То есть есть два объекта, у каждого три признака:

```text
x =
[
    [x11, x12, x13],
    [x21, x22, x23],
]
```

Пусть:

```python
w.shape == (3, 4)
```

Мы хотим получить четыре новых признака.

После:

```python
x @ w
```

форма результата:

```python
(2, 4)
```

Bias:

```python
b.shape == (4,)
```

NumPy автоматически прибавит его к каждой строке результата.

Итог:

```python
(x @ w + b).shape == (2, 4)
```

---

## 9. Что такое broadcasting

Пусть результат матричного умножения:

```text
[
    [10, 20, 30],
    [40, 50, 60],
]
```

А bias:

```text
[1, 2, 3]
```

NumPy интерпретирует сложение так:

```text
[
    [10, 20, 30],
    [40, 50, 60],
]
+
[
    [1, 2, 3],
    [1, 2, 3],
]
```

И получает:

```text
[
    [11, 22, 33],
    [41, 52, 63],
]
```

Нам не нужно вручную повторять bias.

# Часть IV. Почему одних линейных слоёв недостаточно

## 10. Композиция линейных преобразований остаётся линейной

Представим два слоя без активации:

\[
h = xW_1 + b_1
\]

\[
y = hW_2 + b_2
\]

Подставим первое во второе:

\[
y = (xW_1 + b_1)W_2 + b_2
\]

Раскроем скобки:

\[
y = xW_1W_2 + b_1W_2 + b_2
\]

Обозначим:

\[
W' = W_1W_2
\]

\[
b' = b_1W_2 + b_2
\]

Тогда:

\[
y = xW' + b'
\]

То есть два линейных слоя без нелинейности эквивалентны одному линейному слою.

Можно поставить сто линейных слоёв, но без функций активации вся сеть останется одним линейным преобразованием.

---

## 11. Зачем нужна нелинейность

Реальные зависимости редко являются чисто линейными.

Например, значение слова зависит от контекста:

```text
bank
```

может означать:

- финансовый банк;
- берег реки;
- крен самолёта.

Нелинейные преобразования позволяют сети строить сложные границы и комбинировать признаки.

---

# Часть V. Функции активации

## 12. Sigmoid

\[
\sigma(x) = \frac{1}{1 + e^{-x}}
\]

Диапазон:

\[
(0, 1)
\]

Проблемы:

- насыщение;
- исчезающие градиенты;
- выходы не центрированы около нуля;
- вычислительно дороже ReLU.

---

## 13. Tanh

\[
\tanh(x)
\]

Диапазон:

\[
(-1, 1)
\]

Лучше центрирован около нуля, но также страдает от насыщения и исчезающих градиентов.

---

## 14. ReLU

\[
ReLU(x) = \max(0, x)
\]

Плюсы:

- простая;
- быстрая;
- хорошо работает во многих сетях.

Минус:

- для отрицательной области градиент равен нулю;
- нейрон может перестать обновляться;
- возникает проблема dying ReLU.

Важно: ошибочная реализация:

```python
np.max(0, x)
```

не является правильным поэлементным ReLU.

Корректно:

```python
np.maximum(0, x)
```

Но в текущем задании ReLU вообще не требуется.

---

## 15. Leaky ReLU

\[
LeakyReLU(x) =
\begin{cases}
x, & x \geq 0 \\
\alpha x, & x < 0
\end{cases}
\]

Отрицательная область не обнуляется полностью.

---

## 16. ELU

ELU использует экспоненциальную отрицательную ветвь:

\[
ELU(x) =
\begin{cases}
x, & x \geq 0 \\
\alpha(e^x - 1), & x < 0
\end{cases}
\]

---

# Часть VI. GELU

## 17. Полное название

GELU:

```text
Gaussian Error Linear Unit
```

GELU используется в GPT-2.

---

## 18. Основная интуиция

ReLU жёстко принимает решение:

```text
x < 0  → 0
x >= 0 → x
```

GELU делает переход плавным.

Она масштабирует вход с учётом положения значения относительно стандартного нормального распределения.

Точная форма может быть записана как:

\[
GELU(x) = x\Phi(x)
\]

где \(\Phi(x)\) — функция распределения стандартной нормальной случайной величины.

---

## 19. Аппроксимация GPT-2

В GPT-2 используется приближение через `tanh`:

\[
GELU(x)
=
0.5x
\left(
1 +
\tanh
\left[
\sqrt{\frac{2}{\pi}}
\left(
x + 0.044715x^3
\right)
\right]
\right)
\]

Код:

```python
return 0.5 * x * (
    1.0
    + np.tanh(
        np.sqrt(2.0 / np.pi)
        * (x + 0.044715 * x**3)
    )
)
```

---

## 20. Разбор формулы по частям

### Кубическая поправка

```python
x + 0.044715 * x**3
```

Число `0.044715` является частью аппроксимации.

### Масштаб

```python
np.sqrt(2.0 / np.pi)
```

### Плавное ограничение

```python
np.tanh(...)
```

Tanh переводит значение в диапазон приблизительно от -1 до 1.

### Перевод множителя

```python
1.0 + np.tanh(...)
```

Диапазон становится приблизительно от 0 до 2.

### Деление пополам

```python
0.5 * (...)
```

Получается плавный коэффициент приблизительно от 0 до 1.

### Умножение на вход

```python
0.5 * x * (...)
```

Вход не просто обнуляется или сохраняется. Он плавно масштабируется.

---

## 21. Поведение GELU

Для большого положительного \(x\):

\[
GELU(x) \approx x
\]

Для большого отрицательного \(x\):

\[
GELU(x) \approx 0
\]

Около нуля переход плавный.

Некоторые небольшие отрицательные значения дают небольшие отрицательные выходы. Это отличает GELU от ReLU.

---

# Часть VII. Устройство FFN в GPT-2

## 22. Первый слой `c_fc`

Для входа:

```python
x.shape == (n_seq, n_embd)
```

первый слой имеет:

```text
w: [n_embd, 4 * n_embd]
b: [4 * n_embd]
```

Выход:

```text
[n_seq, 4 * n_embd]
```

Для GPT-2 Small:

```text
[n_seq, 768]
    ↓
[n_seq, 3072]
```

---

## 23. Зачем расширять размерность

Расширенное пространство даёт модели больше промежуточных признаков.

Упрощённая интуиция:

```text
768 входных координат
    ↓
3072 промежуточных детектора и преобразования
    ↓
768 обновлённых координат
```

Это похоже на временное «разворачивание» информации в более богатое пространство.

---

## 24. GELU между слоями

После первого линейного слоя:

```python
hidden = gelu(linear(x, **c_fc))
```

GELU добавляет нелинейность.

Без GELU два линейных слоя можно было бы схлопнуть в один.

---

## 25. Второй слой `c_proj`

Второй слой имеет:

```text
w: [4 * n_embd, n_embd]
b: [n_embd]
```

Он возвращает исходную размерность:

```text
[n_seq, 3072]
    ↓
[n_seq, 768]
```

---

## 26. Почему размер выхода должен совпасть с размером входа

Transformer использует residual connection:

\[
x_{out} = x + FFN(x)
\]

Для сложения формы должны совпадать:

```text
x.shape == FFN(x).shape
```

Поэтому FFN внутри расширяет размерность, но наружу возвращает `n_embd`.

---

# Часть VIII. Как представлены веса

## 27. `c_fc` и `c_proj` — словари

Утилита загрузки GPT-2 формирует вложенные словари параметров.

Структура линейного слоя:

```python
c_fc = {
    "w": ...,
    "b": ...,
}
```

Вызов:

```python
linear(x, **c_fc)
```

эквивалентен:

```python
linear(
    x,
    w=c_fc["w"],
    b=c_fc["b"],
)
```

То же самое для:

```python
linear(hidden, **c_proj)
```

# Часть IX. Точное решение

## 28. Linear

```python
def linear(x, w, b):
    return x @ w + b
```

---

## 29. GELU

```python
def gelu(x):
    return 0.5 * x * (
        1.0
        + np.tanh(
            np.sqrt(2.0 / np.pi)
            * (x + 0.044715 * x**3)
        )
    )
```

---

## 30. FFN

```python
def ffn(x, c_fc, c_proj):
    hidden = gelu(linear(x, **c_fc))
    return linear(hidden, **c_proj)
```

---

# Часть X. Проверка размерностей

## 31. Пример

Пусть:

```python
n_seq = 5
n_embd = 3
hidden_size = 12
```

Тогда:

```text
x:              [5, 3]
c_fc["w"]:      [3, 12]
c_fc["b"]:      [12]
c_proj["w"]:    [12, 3]
c_proj["b"]:    [3]
```

Первый linear:

```text
[5, 3] @ [3, 12] + [12]
=
[5, 12]
```

GELU:

```text

```

Второй linear:

```text
[5, 12] @ [12, 3] + [3]
=
[5, 3]
```

Форма восстановилась.

# Часть XI. Что проверят тесты

## 32. GELU — 40%

Тесты, вероятнее всего, сравнивают выход функции с эталонной формулой на:

- отрицательных числах;
- нуле;
- положительных числах;
- одномерных массивах;
- многомерных массивах.

Нельзя использовать ReLU или sigmoid.

Нужно сохранить поэлементную NumPy-реализацию.

---

## 33. Linear — 40%

Проверяется:

```python
x @ w + b
```

Возможные ошибки:

- `w @ x`;
- `x * w`;
- забыть bias;
- использовать `np.dot(w, x)`;
- транспонировать веса без основания.

---

## 34. FFN — 20%

Проверяется точный порядок:

```text
linear c_fc
    ↓
gelu
    ↓
linear c_proj
```

Ошибки:

- GELU после второго слоя;
- GELU два раза;
- поменять `c_fc` и `c_proj`;
- сложить вход и выход внутри `ffn`;
- вернуть hidden размерности `4*n_embd`.

Residual connection добавляется на уровне Transformer block, а не внутри этой функции FFN.

---

# Часть XII. Критические ошибки

## 35. Использовать `np.max` вместо `np.maximum`

Это относится к ReLU, но часто встречается в решениях:

```python
np.max(0, x)
```

`np.max` выполняет агрегацию и имеет другую сигнатуру.

---

## 36. Реализовать точную GELU через erf

В математике:

\[
GELU(x) = \frac{x}{2}
\left(
1 + erf\left(\frac{x}{\sqrt{2}}\right)
\right)
\]

Но GPT-2 использует tanh-аппроксимацию. Проверка курса ожидает именно её.

---

## 37. Добавить residual connection внутри FFN

Неправильно:

```python
return x + linear(hidden, **c_proj)
```

Почему?

FFN должна возвращать только преобразование.

Residual connection будет реализована отдельно:

```python
x = x + ffn(...)
```

---

## 38. Использовать `.T` для веса

Неправильно:

```python
x @ w.T + b
```

В условии форма веса уже задана:

```text
[in, out]
```

Поэтому нужен:

```python
x @ w
```

---

# Часть XIII. Вычислительная стоимость

## 39. Число параметров

Для первого слоя:

\[
n_{embd} \times 4n_{embd}
\]

Для второго:

\[
4n_{embd} \times n_{embd}
\]

Без bias:

\[
8n_{embd}^2
\]

Для GPT-2 Small:

\[
8 \times 768^2
=
4\,718\,592
\]

То есть примерно 4,7 миллиона весов FFN в одном Transformer-блоке.

Плюс bias.

При 12 блоках FFN занимает десятки миллионов параметров.

---

## 40. Почему FFN занимает много параметров

Attention часто получает больше концептуального внимания, но матрицы FFN очень крупные.

Это влияет на:

- размер checkpoint;
- GPU memory;
- время матричных умножений;
- пропускную способность;
- стоимость inference;
- возможность quantization.

---

# Часть XIV. Production trade-offs

## 41. Качество против стоимости

Увеличение hidden dimension может повысить выразительность, но увеличивает:

- параметры;
- FLOPs;
- память;
- latency.

---

## 42. Quantization

FFN-матрицы можно хранить с меньшей точностью:

- FP16;
- BF16;
- INT8;
- INT4.

Это уменьшает память и иногда ускоряет inference.

Но может влиять на качество.

---

## 43. Pruning

Можно удалять малозначимые веса или нейроны.

Цель:

- уменьшить размер;
- ускорить модель;
- сохранить качество.

---

## 44. Mixture of Experts

В некоторых современных Transformer-моделях dense FFN заменяется Mixture of Experts.

Вместо одного FFN существует несколько экспертов, но для токена активируется только часть из них.

Это позволяет увеличить число параметров, не увеличивая пропорционально вычисления на каждый токен.

---

# Часть XV. Как рассказать на интервью

## 45. Что такое FFN в Transformer

Хороший ответ:

> FFN — это position-wise двухслойная полносвязная сеть. Она независимо применяется к каждому токену с одинаковыми весами. В GPT-2 первый линейный слой расширяет hidden dimension примерно в четыре раза, затем применяется GELU, после чего projection layer возвращает исходную размерность. Благодаря этому результат можно добавить к residual stream.

---

## 46. Чем Attention отличается от FFN

> Attention смешивает информацию между позициями последовательности. FFN не смешивает токены между собой: он отдельно преобразует представление каждого токена. Условно Attention отвечает за коммуникацию между токенами, а FFN — за нелинейную обработку полученной информации.

---

## 47. Почему нужна GELU

> Без нелинейной активации два линейных слоя FFN были бы эквивалентны одному линейному преобразованию. GELU добавляет нелинейность и использует плавное вероятностное gating-поведение вместо жёсткого обнуления отрицательных значений, как у ReLU.

---

## 48. Почему размерность расширяется в четыре раза

> Расширение создаёт более богатое промежуточное пространство признаков. Второй слой проецирует его обратно в размер residual stream. Коэффициент четыре — архитектурный выбор GPT-2, балансирующий качество и вычислительную стоимость.

---

# Часть XVI. Ментальная модель

Можно запомнить FFN как работу одного токена в отдельном кабинете.

Attention:

```text
токен общается с другими токенами
```

FFN:

```text
токен остаётся один и перерабатывает полученную информацию
```

Первый linear:

```text
раскладывает смысл на большее число признаков
```

GELU:

```text
нелинейно фильтрует и масштабирует признаки
```

Второй linear:

```text
собирает признаки обратно в embedding исходной размерности
```

Итог:

```text
x
  ↓ Linear: D → 4D
hidden features
  ↓ GELU
nonlinear hidden features
  ↓ Linear: 4D → D
FFN output
```
