# Страница 5. Self-Attention, Softmax и QKV в GPT-2

## 1. Что реализуем

На предыдущей странице мы написали FFN: `Linear → GELU → Linear`. FFN обрабатывает каждый токен отдельно. Но чтобы слово понимало контекст, токены должны обмениваться информацией. Это делает Self-Attention.

На этой странице нужны три функции:

```python
softmax(x)
attention(q, k, v)
self_attention(x, c_attn, c_proj)
```

Полный путь данных:

```text
x
↓ Linear(c_attn)
объединённая матрица QKV
↓ split
Q, K, V
↓ scaled dot-product attention
контекстные представления
↓ Linear(c_proj)
выход Self-Attention
```

Главная формула:

\[
Attention(Q,K,V)=softmax\left(\frac{QK^T}{\sqrt{d_k}}\right)V
\]

## 2. Бизнесовый смысл контекста

Одно и то же слово может иметь разные значения. Например, `bank` означает финансовый банк, берег реки или крен самолёта. Нельзя понять слово изолированно — нужен контекст.

В AI SEO это критично. В карточке товара слово `light` может означать:

- лёгкий по весу;
- светлый оттенок;
- источник света;
- облегчённую версию продукта.

Attention помогает модели связывать характеристики товара между собой и не путать, к какому объекту относится свойство.

### E-commerce

Модель связывает название, материал, размер, совместимость и ограничения. В запросе:

```text
Чехол для iPhone 15, прозрачный, совместимый с MagSafe
```

она должна понять, что `совместимый с MagSafe` относится к чехлу, а `iPhone 15` — это модель совместимого устройства.

### Fintech

Attention связывает сумму, валюту, получателя, дату и статус операции. Ошибка связи может привести к неверному объяснению банковского события.

### Two-sided marketplace

Модель должна связать требования заказчика со сроком, бюджетом, обязательными навыками и форматом работы.

### Dating

Attention помогает учитывать предыдущие сообщения, тон диалога и ссылки местоимений.

### iGaming и casino

Модель может связывать бонус с условиями, сроками и ограничениями, но юридически значимые условия нужно дополнительно проверять детерминированно.

### Crypto

Attention полезен для анализа технической документации, governance proposals и описаний транзакций. Адреса, суммы и актуальные данные нельзя доверять только генеративной модели.

### Big Tech и промышленность

Self-Attention лежит в основе поиска, генерации кода, внутренних knowledge assistants, анализа инцидентов, инженерной документации и мультимодальных моделей.

## 3. Query, Key и Value

У каждого токена создаются три разных вектора.

### Query

Query отвечает на вопрос:

> Какой информации не хватает текущему токену?

### Key

Key отвечает на вопрос:

> Для каких запросов этот токен может быть полезен?

### Value

Value содержит информацию, которую токен передаст другим токенам.

Можно представить поиск:

- поисковый запрос — Query;
- индекс документа — Key;
- содержимое документа — Value.

Query сравнивается с Keys. Чем выше совместимость, тем больший вес получает соответствующий Value.

---

## 4. От embeddings к QKV

Теоретически можно хранить три отдельные матрицы:

\[
W_Q, W_K, W_V \in \mathbb{R}^{d\times d}
\]

и считать:

\[
Q=XW_Q,\quad K=XW_K,\quad V=XW_V
\]

GPT-2 объединяет их в одну матрицу:

\[
W_{QKV}\in\mathbb{R}^{d\times 3d}
\]

Поэтому сначала выполняется один linear:

```python
x = linear(x, **c_attn)
```

Форма меняется:

```text
[n_seq, n_embd] → [n_seq, 3*n_embd]
```

Затем результат делится:

```python
q, k, v = np.split(x, 3, axis=-1)
```

Каждая часть получает форму:

```text
[n_seq, n_embd]
```

`axis=-1` означает, что делится последний размер — признаки, а не последовательность токенов.

---

## 5. Dot-product scores

Для всех пар Query и Key вычисляется:

\[
Scores=QK^T
\]

Формы:

```text
Q:   [n_q, d_k]
K:   [n_k, d_k]
K.T: [d_k, n_k]
```

Результат:

```text
[n_q, n_k]
```

Строка соответствует одному Query, столбцы — Keys всех токенов. Элемент `score[i, j]` показывает, насколько токен `i` заинтересован в информации токена `j`.

---

## 6. Почему делим на sqrt(d_k)

Используется:

\[
\frac{QK^T}{\sqrt{d_k}}
\]

При большой размерности скалярные произведения растут по модулю. Softmax становится почти one-hot, насыщается и даёт плохие градиенты. Деление на корень из размерности стабилизирует масштаб.

Код:

```python
scores = q @ k.T / np.sqrt(q.shape[-1])
```

`q.shape[-1]` — это размерность Query и Key, то есть `d_k`.

---

## 7. Softmax

Scores могут быть любыми числами. Attention нужны веса:

- от 0 до 1;
- сумма по Keys для каждого Query равна 1.

Формула:

\[
softmax(x_i)=\frac{e^{x_i}}{\sum_j e^{x_j}}
\]

В матрице Attention Softmax применяется по последней оси:

```python
axis=-1
```

То есть каждая строка нормируется отдельно.

### Численно стабильная реализация

Наивный код может переполниться:

```python
np.exp(1000)
```

Поэтому сначала вычитается максимум строки:

```python
shifted_x = x - np.max(x, axis=-1, keepdims=True)
```

Softmax не меняется, потому что общий множитель сокращается.

Затем:

```python
exp_x = np.exp(shifted_x)
return exp_x / np.sum(exp_x, axis=-1, keepdims=True)
```

`keepdims=True` сохраняет форму `[n_q, 1]`, поэтому broadcasting работает корректно.

## 8. Взвешенная сумма Values

После Softmax:

```text
weights: [n_q, n_k]
V:       [n_k, d_v]
```

Умножение:

```python
weights @ v
```

даёт:

```text
[n_q, d_v]
```

Для одного Query это:

\[
a_1v_1+a_2v_2+\ldots+a_nv_n
\]

где сумма коэффициентов `a` равна 1.

Пример:

```text
weights = [0.8, 0.2]
v1 = [10, 0]
v2 = [0, 5]
```

Результат:

```text
0.8*[10, 0] + 0.2*[0, 5] = [8, 1]
```

## 9. Полная функция Attention

```python
def attention(q, k, v):
    scores = q @ k.T / np.sqrt(q.shape[-1])
    return softmax(scores) @ v
```

Порядок строго такой:

```text
QKᵀ → scale → Softmax → умножение на V
```

Нельзя применять scale после Softmax или Softmax после умножения на V.

## 10. Self-Attention

Self-Attention означает, что Q, K и V строятся из одной и той же последовательности `x`.

```python
def self_attention(x, c_attn, c_proj):
    x = linear(x, **c_attn)
    q, k, v = np.split(x, 3, axis=-1)
    x = attention(q, k, v)
    return linear(x, **c_proj)
```

### `c_attn`

Это словарь:

```python
{
    "w": qkv_weight,
    "b": qkv_bias,
}
```

`**c_attn` эквивалентно передаче `w=c_attn["w"]` и `b=c_attn["b"]`.

### `c_proj`

После Attention выполняется ещё одна обучаемая линейная проекция. Она смешивает координаты результата и сохраняет форму `[n_seq, n_embd]`.

---

## 11. Размерности по шагам

```text
x:                       [n_seq, n_embd]
linear(x, c_attn):        [n_seq, 3*n_embd]
q, k, v:                  [n_seq, n_embd]
scores = q @ k.T:         [n_seq, n_seq]
attention weights:        [n_seq, n_seq]
weights @ v:              [n_seq, n_embd]
linear(..., c_proj):       [n_seq, n_embd]
```

Форма входа и выхода совпадает, поэтому позже можно применить residual connection:

\[
x+SelfAttention(x)
\]

---

## 12. Что пока отсутствует

### Causal mask

В текущем задании `attention(q, k, v)` не получает mask. Значит, реализуется базовый Self-Attention без запрета смотреть в будущее. Causal Attention будет следующим усложнением.

### Multi-Head Attention

Q, K и V пока не разделяются на головы. Здесь реализуется single-head версия. Позже embedding будет разбит на несколько частей, каждая голова сможет изучать разные типы связей.

## 13. Типичные ошибки

### Softmax по всей матрице

Неправильно:

```python
np.exp(x) / np.sum(np.exp(x))
```

Это одна общая нормализация вместо отдельной для каждого Query.

### Неправильная ось

Неправильно:

```python
axis=0
```

Нужно нормировать строки по Keys:

```python
axis=-1
```

### Неверное произведение

Правильно:

```python
q @ k.T
```

Неправильно:

```python
q.T @ k
```

### Неправильный scale

Нужно делить на:

```python
np.sqrt(q.shape[-1])
```

а не на число токенов.

### Поэлементное умножение на V

Неправильно:

```python
softmax(scores) * v
```

Правильно:

```python
softmax(scores) @ v
```

### Split по axis=0

Это разделит последовательность токенов. Нужно:

```python
np.split(x, 3, axis=-1)
```

### Забыть output projection

Задание требует после Attention применить `linear(x, **c_proj)`.

## 14. Баллы

### Softmax — 20%

Нужны правильная ось, стабильность и нормализация строк.

### Attention — 40%

Нужна точная формула:

\[
softmax(QK^T/\sqrt{d_k})V
\]

### Распаковка QKV — 20%

Нужна строка:

```python
q, k, v = np.split(x, 3, axis=-1)
```

Оставшиеся требования покрываются корректной сборкой `self_attention`, сохранением API и качеством кода.

## 15. Вычислительная сложность

Attention-матрица имеет размер:

```text
[n_seq, n_seq]
```

Память растёт как:

\[
O(n^2)
\]

Время вычисления основных произведений:

\[
O(n^2d)
\]

При удвоении длины контекста Attention-матрица увеличивается примерно в четыре раза. Поэтому длинный контекст дорог по памяти и latency.

Современные оптимизации:

- FlashAttention;
- sparse attention;
- sliding-window attention;
- KV cache;
- memory-efficient kernels.

## 16. Ответы на интервью

### Что такое Self-Attention?

Self-Attention позволяет каждому токену собрать релевантную информацию из других токенов. Из embeddings строятся Q, K и V. Query сравнивается с Keys через scaled dot product. Softmax превращает scores в веса, которыми суммируются Values.

### Зачем нужны разные Q, K и V?

Query описывает, какую информацию ищет токен. Key определяет, на какие запросы токен релевантен. Value содержит информацию, которую он передаёт.

### Почему делим на sqrt(d_k)?

Чтобы стабилизировать масштаб dot products и избежать насыщения Softmax при большой размерности.

### Почему Attention дорогой?

Потому что каждая позиция сравнивается с каждой другой, создавая матрицу размера `n × n`.

### Attention против FFN

Attention смешивает информацию между токенами. FFN независимо преобразует каждый токен после того, как контекст уже собран.

---

## 17. Финальная ментальная схема

```text
каждый токен формирует Query:
«что мне нужно?»

каждый токен формирует Key:
«по каким запросам я полезен?»

каждый токен формирует Value:
«что я передам?»

Query сравнивается со всеми Keys
↓
Softmax формирует доли внимания
↓
Values смешиваются с этими долями
↓
получается контекстно-зависимый embedding
```
