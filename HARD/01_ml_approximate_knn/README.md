# Контракт distance

Вход:

```text
pointA.shape = (D,)
documents.shape = (N, D)
```

Выход:

```text
result.shape = (N, 1)
```

Каждая строка результата содержит евклидово расстояние от `pointA` до соответствующего документа.

---

# Контракт create_sw_graph

Вход:

```text
data.shape = (N, D)
```

Выход:

```python
{
    0: [индексы соседей],
    1: [индексы соседей],
    ...,
    N - 1: [индексы соседей]
}
```

Для каждой точки выбираются:

- короткие связи — среди ближайших кандидатов;
- длинные связи — среди самых далёких кандидатов.

Короткие связи помогают уточнять поиск внутри локальной области. Длинные связи позволяют быстро перескакивать между удалёнными областями векторного пространства.

---

# Контракт nsw

Функция получает:

- `query_point` — вектор запроса;
- `all_documents` — все векторы документов;
- `graph_edges` — построенный SW-граф;
- `search_k` — сколько соседей вернуть;
- `num_start_points` — сколько случайных маршрутов запустить.

Возвращается:

```text
numpy.ndarray длины search_k
```

В массиве лежат индексы найденных документов.

# Минимальная локальная проверка

```python
import numpy as np

from gp_page_1_solution import create_sw_graph, distance, nsw


np.random.seed(42)

documents = np.random.randn(100, 8)
query = np.random.randn(8)

graph = create_sw_graph(
    documents,
    num_candidates_for_choice_long=10,
    num_edges_long=5,
    num_candidates_for_choice_short=10,
    num_edges_short=5
)

result = nsw(
    query,
    documents,
    graph,
    search_k=10,
    num_start_points=5
)

print(result)
print(result.shape)
print(distance(query, documents).shape)
```

Проверяемые свойства:

```text
result.shape == (10,)
distance(query, documents).shape == (100, 1)
```

Конкретные индексы могут меняться между запусками, потому что рёбра и стартовые точки выбираются случайно. Для воспроизводимости используется `np.random.seed(...)`.

# Самая важная идея урока

Exact KNN отвечает:

> Какие точки математически являются ближайшими?

Но обычно для этого сравнивает запрос со всеми объектами.

Approximate KNN отвечает:

> Какие точки с высокой вероятностью достаточно близкие, если у нас есть строгий лимит по времени и стоимости?

В реальной системе важен компромисс:

```text
качество поиска ↔ задержка ↔ память ↔ стоимость
```

Эта задача показывает фундаментальную идею современных векторных поисковых систем: заранее организовать эмбеддинги в граф, а во время запроса исследовать только небольшую часть коллекции.

---

# Что важно не перепутать

- `search_k` — сколько соседей вернуть.
- `num_start_points` — сколько независимых маршрутов запустить.
- `num_candidates_for_choice_short` — размер пула близких кандидатов.
- `num_edges_short` — сколько коротких рёбер выбрать.
- `num_candidates_for_choice_long` — размер пула далёких кандидатов.
- `num_edges_long` — сколько длинных рёбер выбрать.
- `sampling_share` используется при построении графа, а не при выдаче результата.
