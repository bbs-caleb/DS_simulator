# Урок 46 DL: TFT — Шаг 2. Gated Residual Network (GRN)

## Контекст задачи

### Бизнесовый контекст

В прошлом шаге для «МидлМаркета» мы собрали LSTM — блок, который читает историю продаж по шагам. Но у сети сотни признаков: продажи, цена, промо, погода, праздники, район, категория товара. Не все они полезны в каждый момент: в спокойный вторник погода и промо не важны, а перед праздником — критичны. Если кормить модель всем подряд без отбора, она переобучится на шуме, и прогнозы станут хуже, чем у простой модели. Для бизнеса это прямые потери: неверный прогноз → неверный заказ → залежавшийся или отсутствующий товар.

Нужен механизм, который **сам решает, насколько сложно обрабатывать каждый признак** — вплоть до полного пропуска бесполезного преобразования. Именно это делает **Gated Residual Network (GRN)** — рабочая лошадка TFT, встречающаяся в нём буквально везде: при отборе переменных, при обработке статических признаков, при обогащении временных представлений.

### Технический контекст

Чтобы понять GRN, нужны два кирпичика.

**1. Gated Linear Unit (GLU)** — упрощённый наследник фильтров LSTM. Если в LSTM было три фильтра (forget, input, output), то GLU оставляет по сути только выходной. Он решает, какую часть информации пропустить дальше:

```
GLU(x) = (W_l · x + b_l) * σ(W_s · x + b_s)
```

Линейная часть `(W_l · x + b_l)` несёт информацию, а сигмоидная `σ(...)` работает «краном» от 0 до 1. Плюс: градиент свободно течёт через линейную часть, что смягчает затухание градиента в глубоких сетях.

**2. Skip (residual) connection** — идея из ResNet: выход ранних слоёв идёт напрямую к поздним, минуя преобразования. Это даёт градиенту прямой путь назад (борьба с затуханием), позволяет сети «обнулить» ненужное преобразование (защита от переобучения) и сохраняет исходную информацию.

**GRN** собирает это вместе. Формально:

```
η_2 = ELU(W_2 · a + W_3 · c + b_2)      # нелинейное преобразование
η_1 = W_1 · η_2 + b_1                     # линейный слой
GRN(a, c) = LayerNorm(a + GLU(η_1))       # gate + skip + нормализация
```

Где `a` — вход, `c` — опциональный контекст (например, статические признаки; в нашем шаблоне контекст не используется). Дополнительно после линейного слоя стоит Dropout.

Ключевой нюанс шаблона: если размерность входа не совпадает с `hidden_size`, «прямой» путь (residual) нельзя просто прибавить — его сначала прогоняют через проекцию `self.project`, чтобы размерности сошлись. Эта развилка `if` уже написана за нас, нам остаётся собрать основной путь.
