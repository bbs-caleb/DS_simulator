# SKU Embeddings — математика TF-IDF и BM25 без пропусков

# 1. Обозначения

Пусть User-Item Matrix обозначается:

\[
X \in \mathbb{R}^{N \times M}
\]

Где:

- \(N\) — число пользователей;
- \(M\) — число товаров;
- \(x_{ij}\) — количество покупок товара \(j\) пользователем \(i\).

---

# 2. Нормализация по строкам

Сумма строки:

\[
s_i = \sum_{j=1}^{M}x_{ij}
\]

Нормализованный элемент:

\[
r_{ij} =
\begin{cases}
x_{ij}/s_i, & s_i > 0 \\
0, & s_i = 0
\end{cases}
\]

Для ненулевой строки:

\[
\sum_j r_{ij}=1
\]

Матричная форма:

\[
R = D_r^{-1}X
\]

Где:

\[
D_r = diag(s_1,\ldots,s_N)
\]

В коде вместо прямого обращения берётся диагональ обратных сумм.

---

# 3. Нормализация по столбцам

Сумма столбца:

\[
c_j = \sum_{i=1}^{N}x_{ij}
\]

Нормализованный элемент:

\[
q_{ij} =
\begin{cases}
x_{ij}/c_j, & c_j > 0 \\
0, & c_j = 0
\end{cases}
\]

Матричная форма:

\[
Q = X D_c^{-1}
\]

Где:

\[
D_c = diag(c_1,\ldots,c_M)
\]

---

# 4. Term Frequency

Для пользователя \(i\) и товара \(j\):

\[
TF_{ij} =
\frac{x_{ij}}
{\sum_{k=1}^{M}x_{ik}}
\]

То есть:

\[
TF = D_r^{-1}X
\]

TF совпадает с нормализацией по строкам.

---

# 5. Document Frequency

Для товара \(j\):

\[
df_j =
\sum_{i=1}^{N}
\mathbb{1}[x_{ij}>0]
\]

Важно:

\[
df_j
\neq
\sum_i x_{ij}
\]

`df` считает пользователей, а не количество проданных единиц.

Пример:

```text
товар купил один пользователь в количестве 100
```

Тогда:

```text
sum quantity = 100
df = 1
```

---

# 6. IDF

\[
IDF_j =
\log
\left(
\frac{N}{df_j}
\right)
\]

Свойства.

## Товар встречается у всех

\[
df_j=N
\]

\[
IDF_j=\log(1)=0
\]

## Товар встречается у одного пользователя

\[
df_j=1
\]

\[
IDF_j=\log(N)
\]

Вес выше.

---

# 7. TF-IDF

\[
TFIDF_{ij}=TF_{ij}\cdot IDF_j
\]

Матрично:

\[
TFIDF = TF \cdot D_{idf}
\]

Где:

\[
D_{idf}=diag(IDF_1,\ldots,IDF_M)
\]

---

# 8. Почему IDF умножается справа

TF имеет размер:

```text
N × M
```

Диагональная IDF-матрица:

```text
M × M
```

Поэтому:

```text
(N × M) × (M × M) = N × M
```

Умножение справа масштабирует столбцы.

---

# 9. Длина пользовательского профиля

В BM25:

\[
|d_i|=\sum_j x_{ij}
\]

Это исходное количество покупок пользователя.

Средняя длина:

\[
avgdl=
\frac{1}{N}
\sum_{i=1}^{N}|d_i|
\]

---

# 10. Коэффициент длины

\[
\Delta_i =
k_1
\left(
1-b+b\frac{|d_i|}{avgdl}
\right)
\]

## Если профиль средней длины

\[
|d_i|=avgdl
\]

Тогда:

\[
\Delta_i=k_1
\]

## Если профиль длиннее среднего

\[
|d_i|>avgdl
\]

Тогда \(\Delta_i\) больше, а итоговый вес ниже.

## Если профиль короче среднего

\[
|d_i|<avgdl
\]

Тогда \(\Delta_i\) меньше.

---

# 11. BM25-вес

\[
BM25_{ij}
=
IDF_j
\cdot
\frac{
TF_{ij}(k_1+1)
}{
TF_{ij}+\Delta_i
}
\]

В коде сначала вычисляется часть без IDF:

\[
S_{ij}
=
\frac{
TF_{ij}(k_1+1)
}{
TF_{ij}+\Delta_i
}
\]

Затем:

\[
BM25 = S D_{idf}
\]

---

# 12. Насыщение TF

Рассмотрим:

\[
f(TF)=
\frac{TF(k_1+1)}
{TF+\Delta}
\]

Производная положительна, поэтому вес растёт.

Но рост замедляется.

При больших TF:

\[
f(TF)
\rightarrow
k_1+1
\]

То есть значение ограничивается сверху.

Это и есть насыщение.

---

# 13. Почему линейный TF может быть плох

Сырой линейный подход считает:

```text
10 покупок в 10 раз важнее одной
100 покупок в 100 раз важнее одной
```

В реальности:

- первая покупка сильно сигнализирует интерес;
- вторая усиливает сигнал;
- сотая покупка может добавлять мало новой информации.

BM25 моделирует именно такую нелинейность.

---

# 14. Алгебраическое преобразование для sparse

Начинаем:

\[
S=
\frac{TF(k_1+1)}
{TF+\Delta}
\]

Выносим TF из знаменателя:

\[
TF+\Delta
=
TF
\left(
1+\frac{\Delta}{TF}
\right)
\]

Подставляем:

\[
S=
\frac{
TF(k_1+1)
}{
TF
\left(
1+\Delta/TF
\right)
}
\]

Сокращаем TF:

\[
S=
\frac{k_1+1}
{1+\Delta/TF}
\]

Теперь:

\[
\frac{\Delta}{TF}
=
\left(
\frac{TF}{\Delta}
\right)^{-1}
\]

И:

\[
S=
(k_1+1)
\left[
\left(
TF/\Delta
\right)^{-1}+1
\right]^{-1}
\]

---

# 15. Соответствие строкам кода

## Вычислить \(TF/\Delta\)

```python
saturated_tf = (
    diags(inverse_delta)
    .dot(tf_matrix)
    .tocsr()
)
```

## Вычислить \((TF/\Delta)^{-1}\)

```python
saturated_tf = saturated_tf.power(-1)
```

## Прибавить 1

```python
saturated_tf.data += 1.0
```

## Ещё раз взять обратное

```python
saturated_tf = saturated_tf.power(-1)
```

## Умножить на \(k_1+1\)

```python
saturated_tf.data *= k1 + 1.0
```

---

# 16. Почему `data += 1` математически корректно

Мы вычисляем веса только для существующих взаимодействий:

\[
x_{ij}>0
\]

Для нулевых взаимодействий итоговый вес должен остаться 0.

Если прибавить 1 ко всей матрице, нулевые позиции станут ненулевыми.

Если прибавить 1 только к `.data`, меняются только существующие пары user-item.

---

# 17. Пример ручного расчёта BM25

Пусть:

```text
TF = 0.5
k1 = 2
b = 0.75
row_length = avgdl
```

Тогда:

\[
\Delta=
2(1-0.75+0.75\cdot1)=2
\]

Сатурированная TF:

\[
\frac{0.5(2+1)}{0.5+2}
=
\frac{1.5}{2.5}
=
0.6
\]

Если:

```text
IDF = 1.2
```

то:

\[
BM25=0.6\cdot1.2=0.72
\]

---

# 18. Влияние длины строки

Пусть TF одинаков:

```text
TF = 0.5
```

Но пользователь A имеет короткий профиль, B — длинный.

Для длинного профиля:

\[
|d_i|/avgdl > 1
\]

Поэтому:

\[
\Delta_i
\]

увеличивается.

Знаменатель растёт, вес снижается.

BM25 считает одно и то же относительное взаимодействие менее удивительным в очень длинном профиле.

---

# 19. Крайний случай `k1 = 0`

\[
\frac{TF(k_1+1)}
{TF+k_1(...)}
\]

При \(k_1=0\):

\[
\frac{TF}{TF}=1
\]

для ненулевого TF.

Итоговый вес становится:

\[
IDF_j
\]

Поэтому код заполняет все хранящиеся TF-значения единицами.

---

# 20. Крайний случай `b = 0`

\[
\Delta_i=k_1
\]

Длина строки полностью игнорируется.

---

# 21. Крайний случай `b = 1`

\[
\Delta_i=
k_1
\frac{|d_i|}{avgdl}
\]

Используется полная коррекция длины.

---

# 22. Почему форма не меняется

Все операции:

- диагональное масштабирование;
- поэлементная степень;
- изменение `.data`;
- умножение на IDF;

не меняют множество пользователей и товаров.

Поэтому:

\[
shape(output)=shape(input)
\]

---

# 23. Вычислительная сложность

Пусть:

```text
nnz = число ненулевых взаимодействий
```

Основные sparse-операции работают примерно за:

```text
O(nnz)
```

Векторы сумм:

```text
O(nnz)
```

Дополнительная память:

```text
O(nnz + N + M)
```

Плотная реализация потребовала бы:

```text
O(N × M)
```

# 24. Финальная математическая карта

## By row

\[
D_r^{-1}X
\]

## By column

\[
XD_c^{-1}
\]

## TF-IDF

\[
D_r^{-1}XD_{idf}
\]

## BM25

\[
S(TF,|d|,avgdl,k_1,b)D_{idf}
\]

Где \(S\) — нелинейная насыщенная трансформация TF.
