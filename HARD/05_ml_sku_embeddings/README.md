## Главная идея страницы в одном предложении

**Эмбеддинг товара — это компактный числовой вектор, в котором товары с похожим смыслом или похожим поведением покупателей должны располагаться близко друг к другу.**

---

## Самая важная оговорка

Слово **«похожий»** само по себе ничего не значит, пока мы не уточнили критерий.

Товары могут быть похожи:

- по категории;
- по бренду;
- по цене;
- по характеристикам;
- по покупательскому поведению;
- по совместным покупкам;
- по взаимозаменяемости;
- по динамике продаж;
- по реакции на промо;
- по аудитории;
- по риску;
- по контексту использования.

Например:

- Coca-Cola 1 л и Pepsi 1 л похожи как **заменители**;
- смартфон и зарядный кабель похожи как **комплементы**;
- две пачки молока разных брендов похожи по **спросу и назначению**;
- два дорогих товара могут быть близки по **аудитории**, но не по категории.

Поэтому перед обучением эмбеддингов DS должен ответить:

> Какой именно тип близости должен кодировать наш вектор?

## Как проходить материал

Рекомендуемый порядок:

1. Сначала прочитать бизнесовый раздел.
2. Затем разобраться, что такое вектор и пространство.
3. После этого изучить евклидово расстояние.
4. Потом косинусное сходство.
5. Затем сравнить их на простых примерах.
6. После этого прочитать про реальные источники данных.
7. В конце пройти разделы про production и интервью.

Не нужно пытаться запомнить все формулы сразу. Сначала важно понять смысл:

- евклидово расстояние отвечает на вопрос **«насколько далеко точки»**;
- косинусное сходство отвечает на вопрос **«насколько одинаково направлены векторы»**.

## Короткий словарь

**SKU** — конкретная товарная позиция, которую можно отдельно учитывать в каталоге и запасах.

**Embedding** — вектор чисел, описывающий объект.

**Vector** — упорядоченный набор чисел.

**Dimension** — одна координата вектора.

**Distance** — мера различия.

**Similarity** — мера сходства.

**Nearest neighbors** — ближайшие объекты в пространстве.

**Substitute** — товар-заменитель.

**Complement** — товар-дополнение.

**Cannibalization** — ситуация, когда продажи одного похожего товара уменьшают продажи другого.

**Cold start** — проблема нового товара, по которому еще нет истории поведения.

**Retrieval** — быстрый этап поиска кандидатов.

**Ranking** — более точный этап сортировки кандидатов.

## Итог страницы

Эмбеддинги — это не просто математическая техника. Это способ:

- сжать сложное описание товара;
- перенести поведение пользователей в числовое пространство;
- быстро искать похожие объекты;
- строить рекомендации;
- находить заменители;
- искать комплементы;
- оценивать каннибализацию;
- переносить знания с популярных товаров на новые;
- ускорять модели поиска, прогноза и персонализации.

Следующий шаг курса, вероятнее всего, покажет, **как именно обучить такие векторы на данных о взаимодействиях покупателей с товарами**.
