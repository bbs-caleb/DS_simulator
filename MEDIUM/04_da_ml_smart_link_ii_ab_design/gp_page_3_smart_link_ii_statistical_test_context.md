## Задача 1. Сгенерировать выборку CPC

Нужно реализовать функцию:

```python
def cpc_sample(
    n_samples: int,
    conversion_rate: float,
    reward_avg: float,
    reward_std: float,
) -> np.ndarray:
```

Она должна смоделировать результат работы CPA-платформы для набора кликов.

Для каждого клика:

1. определяется, произошло действие или нет;
2. если действие произошло, генерируется вознаграждение;
3. CPC рассчитывается как произведение признака действия на вознаграждение.

## Задача 2. Сравнить две выборки CPC

Нужно реализовать функцию:

```python
def t_test(
    cpc_a: np.ndarray,
    cpc_b: np.ndarray,
    alpha: float = 0.05,
) -> Tuple[bool, float]:
```

Она должна:

1. провести t-тест для двух независимых выборок;
2. получить `p-value`;
3. сравнить `p-value` с уровнем значимости `alpha`;
4. вернуть:
   - `True`, если различие статистически значимо;
   - `False`, если статистической значимости нет;
   - само значение `p-value`.

# 2. Бизнес-контекст OrangeChicken

OrangeChicken — CPA-платформа.

CPA означает:

```text
Cost Per Action
```

Рекламодатель платит только тогда, когда пользователь совершил целевое действие.

Целевым действием может быть:

- покупка;
- регистрация;
- заявка;
- установка приложения;
- первый депозит;
- оформление подписки;
- заполнение анкеты;
- бронирование;
- подтвержденный заказ.

На платформе участвуют три основные стороны.

## 2.1. Рекламодатель

Рекламодатель:

- создает оффер;
- описывает целевое действие;
- задает вознаграждение;
- принимает или отклоняет конверсии;
- платит платформе.

Пример:

```text
За каждую подтвержденную заявку рекламодатель платит 20 евро.
```

## 2.2. Арбитражник

Арбитражник:

- покупает или получает трафик;
- направляет пользователей на Smart-Link;
- получает плату за подтвержденные действия.

## 2.3. Платформа

Платформа:

- принимает клики;
- выбирает оффер;
- отслеживает конверсии;
- учитывает выплату;
- оптимизирует распределение трафика;
- получает свою маржу.

---

# 3. Что делает Smart-Link

У арбитражника может быть одна универсальная ссылка.

Когда пользователь кликает по ней, Smart-Link выбирает конкретный оффер.

Упрощенная схема:

```text
клик пользователя
        ↓
Smart-Link
        ↓
выбор оффера
        ↓
лендинг рекламодателя
        ↓
action или отсутствие action
        ↓
вознаграждение
```

Smart-Link должен направлять клик туда, где ожидаемый доход выше.

# 4. CVR, CPA и CPC

## 4.1. CVR

CVR — conversion rate.

Формула:

```text
CVR = число действий / число кликов
```

Пример:

```text
1 000 кликов
50 действий
```

Тогда:

```text
CVR = 50 / 1 000 = 0.05 = 5%
```

CVR можно интерпретировать как приблизительную вероятность действия для нового похожего клика.

---

## 4.2. CPA

CPA — cost per action.

В рамках этой задачи это вознаграждение за одно действие.

Пример:

```text
CPA = 20 евро
```

Если действие произошло, клик принес 20 евро.

Если действие не произошло, клик принес 0 евро.

---

## 4.3. CPC

CPC — cost per click.

В задаче:

```text
CPC = CVR × CPA
```

Это формула ожидаемого дохода на один клик.

Например:

```text
CVR = 5%
CPA = 20 евро
```

Тогда ожидаемый CPC:

```text
0.05 × 20 = 1 евро
```

То есть в среднем один клик приносит 1 евро.

Но в симуляции мы моделируем не только среднее значение, а отдельный результат каждого клика.

Для отдельного клика:

```text
CPC_i = action_i × reward_i
```

где:

```text
action_i = 1, если действие произошло;
action_i = 0, если действие не произошло.
```

---

# 5. Почему выборка CPC содержит много нулей

Предположим:

```text
conversion_rate = 0.05
```

Это означает, что только примерно 5% кликов приводят к действию.

Для остальных 95%:

```text
action = 0
```

Тогда:

```text
CPC = 0 × reward = 0
```

Поэтому распределение CPC обычно выглядит так:

- очень много нулей;
- небольшая доля положительных значений;
- положительные значения имеют собственный разброс.

Это распределение не является обычным нормальным распределением.

Оно представляет смесь:

1. точечной массы в нуле;
2. распределения вознаграждения для конвертировавшихся кликов.

---

# 6. Математическая модель симуляции

Для каждого клика `i` генерируются две случайные величины.

## 6.1. Событие действия

```text
Action_i ~ Bernoulli(conversion_rate)
```

В NumPy это удобно реализовать через биномиальное распределение:

```python
np.random.binomial(
    n=1,
    p=conversion_rate,
    size=n_samples,
)
```

Почему `n=1`?

Биномиальное распределение считает число успехов в `n` испытаниях.

Если:

```text
n = 1
```

результат может быть только:

```text
0 или 1
```

То есть:

- `0` — действие не произошло;
- `1` — действие произошло.

---

## 6.2. Вознаграждение

```text
Reward_i ~ Normal(reward_avg, reward_std)
```

В NumPy:

```python
np.random.normal(
    loc=reward_avg,
    scale=reward_std,
    size=n_samples,
)
```

Здесь:

- `loc` — среднее;
- `scale` — стандартное отклонение;
- `size` — число генерируемых значений.

---

## 6.3. Итоговый CPC

```text
CPC_i = Action_i × Reward_i
```

В NumPy массивы умножаются поэлементно:

```python
cpc = actions * rewards
```

Пример:

```text
actions = [0, 1, 0, 1]
rewards = [10.5, 9.8, 12.1, 11.0]
```

Тогда:

```text
cpc = [0, 9.8, 0, 11.0]
```

---

# 7. Почему reward генерируется для всех кликов

Можно спросить:

> Зачем генерировать reward даже для кликов без действия?

Потому что после умножения на `0` эти значения все равно становятся нулями.

```text
0 × любое число = 0
```

Такой подход:

- очень простой;
- векторизованный;
- не требует циклов;
- соответствует описанной модели.

---

# 8. Почему не используется цикл

Можно было бы написать:

```python
results = []

for _ in range(n_samples):
    action = ...
    reward = ...
    results.append(action * reward)
```

Но NumPy умеет работать сразу с целым массивом.

Векторизованная версия:

```python
actions = np.random.binomial(...)
rewards = np.random.normal(...)
cpc = actions * rewards
```

Преимущества:

- код короче;
- код понятнее;
- выполнение быстрее;
- меньше вероятность ошибки;
- результат сразу имеет тип `np.ndarray`.

---

# 9. Математическое ожидание CPC

Пусть:

```text
A ~ Bernoulli(p)
R ~ Normal(mu, sigma)
CPC = A × R
```

Если `A` и `R` независимы:

```text
E[CPC] = E[A] × E[R]
```

Для Bernoulli:

```text
E[A] = p
```

Для Normal:

```text
E[R] = mu
```

Следовательно:

```text
E[CPC] = p × mu
```

То есть:

```text
Expected CPC = conversion_rate × reward_avg
```

Пример:

```text
conversion_rate = 0.10
reward_avg = 20
```

Тогда ожидаемое среднее:

```text
0.10 × 20 = 2
```

На большой сгенерированной выборке среднее CPC должно быть близко к 2.

---

# 10. Важное допущение о нормальном распределении reward

Нормальное распределение теоретически может генерировать отрицательные числа.

Например, если:

```text
reward_avg = 2
reward_std = 10
```

часть reward будет отрицательной.

В реальном CPA-бизнесе отрицательная выплата за action обычно не имеет прямого смысла.

Можно было бы:

- ограничить значения нулем;
- использовать lognormal;
- использовать gamma;
- моделировать фиксированные выплаты;
- моделировать смесь по офферам.

Но в условии прямо сказано использовать нормальное распределение.

Поэтому в решении нельзя самовольно делать:

```python
rewards = np.maximum(rewards, 0)
```

Это изменило бы заданную модель и могло бы сломать автоматические тесты.

---

# 11. Зачем нужен статистический тест

После A/B-эксперимента получены две выборки:

```text
cpc_a — CPC контрольной группы;
cpc_b — CPC тестовой группы.
```

Даже если алгоритмы полностью одинаковы, средние выборок почти никогда не совпадут идеально.

Например:

```text
mean(A) = 1.98
mean(B) = 2.03
```

Разница может быть вызвана:

- настоящим эффектом;
- обычным случайным шумом.

Статистический тест помогает оценить, насколько наблюдаемая разница совместима с отсутствием эффекта.

---

# 12. Нулевая и альтернативная гипотезы

Для двустороннего t-теста:

```text
H0: средние CPC в группах равны.
H1: средние CPC в группах различаются.
```

Математически:

```text
H0: μ_A = μ_B
H1: μ_A ≠ μ_B
```

Двусторонняя альтернатива позволяет обнаружить:

- улучшение B относительно A;
- ухудшение B относительно A.

Это безопаснее, если направление теста отдельно не зафиксировано в API.

---

# 13. Что такое t-статистика

Упрощенно t-статистика сравнивает:

```text
наблюдаемую разницу средних
```

с:

```text
типичным случайным разбросом этой разницы
```

Логика:

```text
t = разница средних / стандартная ошибка разницы
```

Если средние различаются сильно относительно шума, абсолютное значение t-статистики становится большим.

---

# 14. Почему выбран Welch t-test

В SciPy:

```python
ttest_ind(cpc_a, cpc_b, equal_var=False)
```

`equal_var=False` включает Welch t-test.

Он не требует предполагать, что дисперсии A и B строго одинаковы.

Это особенно разумно для CPC, потому что новая версия Smart-Link может изменить:

- conversion rate;
- среднее вознаграждение;
- долю дорогих офферов;
- разброс дохода.

Следовательно, вместе со средним может измениться дисперсия.

Формула стандартной ошибки в условии:

```text
sqrt(s_A² / n_A + s_B² / n_B)
```

соответствует логике Welch t-test.

---

# 15. Что возвращает SciPy

Вызов:

```python
test_result = ttest_ind(
    cpc_a,
    cpc_b,
    equal_var=False,
)
```

возвращает объект, содержащий:

```text
statistic
pvalue
```

В задании требуется только `p-value` и логический ответ.

Поэтому:

```python
p_value = float(test_result.pvalue)
```

---

# 16. Что такое p-value

Корректное определение:

> P-value — вероятность получить наблюдаемую статистику или еще более экстремальную при условии, что нулевая гипотеза верна и предпосылки модели соблюдены.

Некорректные трактовки:

```text
P-value — вероятность того, что H0 верна.
```

Это неверно.

```text
P-value — вероятность, что результат случайный.
```

Это слишком грубая и вводящая в заблуждение формулировка.

---

# 17. Как принимается решение

```python
is_significant = bool(p_value < alpha)
```

Если:

```text
p_value < alpha
```

нулевая гипотеза отвергается на выбранном уровне значимости.

Если:

```text
p_value >= alpha
```

у нас недостаточно оснований отвергнуть H0.

Это не доказывает, что средние абсолютно одинаковы.

---

# 18. Пример интерпретации

## Случай 1

```text
alpha = 0.05
p-value = 0.012
```

Поскольку:

```text
0.012 < 0.05
```

возвращается:

```python
(True, 0.012)
```

Различие статистически значимо.

## Случай 2

```text
alpha = 0.05
p-value = 0.31
```

Поскольку:

```text
0.31 >= 0.05
```

возвращается:

```python
(False, 0.31)
```

Статистической значимости не обнаружено.

---

# 19. Почему t-test может работать с ненормальным CPC

CPC на уровне отдельного клика сильно ненормален:

- много нулей;
- правый хвост;
- возможные выбросы.

Но t-test проверяет средние.

При достаточно больших независимых выборках центральная предельная теорема помогает распределению выборочного среднего приближаться к нормальному.

Однако в реальных задачах нужно проверить:

- размер выборки;
- тяжелые хвосты;
- экстремальные выбросы;
- зависимость кликов одного пользователя;
- кластеризацию по арбитражнику;
- устойчивость результата;
- качество логирования.

Для учебной задачи требуется именно t-test.

# 20. Статистическая значимость не равна бизнес-значимости

Предположим:

```text
A mean CPC = 1.0000
B mean CPC = 1.0005
p-value = 0.001
```

Различие статистически значимо.

Но абсолютный рост:

```text
0.0005
```

может быть слишком мал, чтобы:

- покрыть стоимость ML-инфраструктуры;
- оправдать разработку;
- компенсировать latency;
- оправдать риск;
- изменить бизнес-результат.

Поэтому реальный анализ должен включать:

- uplift;
- доверительный интервал;
- MDE;
- дополнительную выручку;
- стоимость решения;
- guardrail-метрики.

# 21. Применение в разных индустриях

## 21.1. Финтех

Можно моделировать:

```text
доход на заявку = факт выдачи × маржа выдачи
```

Это похоже на CPC:

```text
0 для невыданных заявок;
положительная ценность для выданных.
```

Но дополнительно нужно учитывать default и fraud.

## 21.2. E-commerce

Можно сравнивать:

```text
revenue per visitor
profit per session
gross margin per user
```

Много пользователей ничего не покупают, поэтому распределение тоже содержит много нулей.

## 21.3. Marketplace

Метрика:

```text
commission per visitor
```

Но рост покупок может сопровождаться:

- отменами;
- возвратами;
- ухудшением распределения seller exposure.

## 21.4. iGaming

Можно сравнивать revenue per user, но обязательны guardrails:

- responsible gaming;
- fraud;
- self-exclusion;
- жалобы;
- риск зависимости.

## 21.5. Dating

Прямая денежная метрика может быть заменена на:

```text
meaningful conversations per user
```

Значение также часто равно нулю для большинства пользователей.

## 21.6. AI-продукты

Можно сравнивать:

```text
успешно решенные задачи на сессию
```

или:

```text
стоимость токенов на решенную задачу
```

Тест должен учитывать повторные запросы одного пользователя.

# 22. Что именно реализовано в решении

## Функция `cpc_sample`

1. генерирует `n_samples` значений 0/1;
2. генерирует `n_samples` rewards;
3. перемножает массивы;
4. возвращает `np.ndarray`.

## Функция `t_test`

1. получает две независимые выборки;
2. запускает Welch t-test;
3. получает `p-value`;
4. сравнивает его с `alpha`;
5. возвращает:

```python
(is_significant, p_value)
```
