## 4. Детерминированная unit-проверка логики выбора

Случайные симуляции неудобны для проверки самого цикла. Поэтому можно временно подменить функции предсказуемыми результатами.

Предположим:

| n_samples | Type I | Type II |
|---:|---:|---:|
| 100 | 0.04 | 0.60 |
| 200 | 0.04 | 0.30 |
| 300 | 0.04 | 0.19 |
| 400 | 0.04 | 0.10 |

При `alpha=0.05` и `beta=0.20` ожидается:

```python
(300, 0.04, 0.19)
```

Функция не должна вернуть 400, потому что 300 — первый подходящий размер.

---

## 5. Проверка обоих условий

Пример:

| n_samples | Type I | Type II |
|---:|---:|---:|
| 100 | 0.08 | 0.10 |
| 200 | 0.04 | 0.18 |

Первый размер имеет хорошую мощность, но слишком высокую ошибку I рода.

Правильный результат:

```python
(200, 0.04, 0.18)
```

Если функция вернула 100, значит она проверяет только β.

---

## 6. Проверка границы

При:

```text
type_1_error = 0.05
type_2_error = 0.20
alpha = 0.05
beta = 0.20
```

значение должно считаться подходящим.

Это проверяет использование `<=`.

---

## 7. Проверка `select_mde`

Предположим:

| MDE | Type II |
|---:|---:|
| 0.01 | 0.90 |
| 0.05 | 0.50 |
| 0.10 | 0.21 |
| 0.20 | 0.18 |
| 0.30 | 0.05 |

При `beta=0.20` ожидается:

```python
(0.20, 0.18)
```

## 8. Проверка `RuntimeError`

Если ни один размер не подходит:

```python
try:
    select_sample_size(...)
except RuntimeError as error:
    print(error)
```

Функция не должна молча возвращать неподходящий последний размер.

То же относится к `select_mde`.

---

## 9. Пустая сетка

Решение использует диагностические переменные:

```text
Last sample size: None
Last MDE: None
```

Поэтому пустая сетка приводит к понятному `RuntimeError`, а не к `UnboundLocalError`.


## 12. Воспроизводимость

```python
np.random.seed(123)
result_1 = select_mde(...)

np.random.seed(123)
result_2 = select_mde(...)

assert result_1 == result_2
```

Это работает, потому что решение использует `np.random.binomial` и `np.random.normal`, а не отдельный `default_rng`.

---

## 13. Порядок сетки

Функция не сортирует значения.

Если передать:

```python
mde_grid=[0.30, 0.10, 0.20]
```

проверка начнется с 0.30.

Пользователь должен передавать:

```python
mde_grid=[0.10, 0.20, 0.30]
```

---

## 14. Слишком маленькое `n_simulations`

При 20 симуляциях доля меняется шагом:

```text
1 / 20 = 0.05
```

Одна дополнительная ошибка может изменить оценку с 0.20 до 0.25.

Поэтому слишком маленькое число симуляций делает подбор нестабильным.

---

## 15. Почему sample size иногда не выбирается из-за Type I

Даже корректный тест может случайно получить:

```text
type_1_error = 0.052
```

при `alpha=0.05`.

Условие учебной задачи отклонит такой размер.

Для более стабильного результата увеличивают `n_simulations`.

---

## 16. Ограничение вероятности

В A/B:

```python
test_conversion_rate = conversion_rate * (1 + mde)
```

Результат должен быть не больше единицы.

Некорректный пример:

```text
conversion_rate = 0.8
mde = 0.5
результат = 1.2
```

NumPy не принимает вероятность больше 1.

Clipping не добавляется, потому что он изменил бы фактический MDE и скрывал ошибку входа.

---

## 17. MDE = 0

При `mde=0` A/B превращается в A/A по распределению.

`ab_test` будет считать почти все незначимые результаты ошибками II рода, поэтому β получится около `1 - alpha`.

Концептуально это не дизайн с существующим эффектом, поэтому MDE-сетка должна содержать положительные бизнес-эффекты.

---

## 18. Проверка отсутствия `default_rng`

В файле не должно быть:

```python

```

