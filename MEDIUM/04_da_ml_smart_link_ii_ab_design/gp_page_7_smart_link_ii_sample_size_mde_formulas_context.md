# 1. Что требуется сделать

На этой странице нужно реализовать две независимые аналитические функции:

```python
calculate_sample_size(...)
calculate_mde(...)
```

В отличие от прошлых шагов здесь не требуются:

- генерация CPC;
- циклы симуляций;
- A/A-тест;
- A/B-тест;
- перебор сеток;
- `np.random`.

Используется готовая формула для двух независимых групп одинакового размера при допущении:

```text
дисперсии групп одинаковы;
анализируемая метрика достаточно близка к нормальной;
используется двусторонняя проверка гипотезы;
размер A и B одинаков.
```

# 2. Зачем нужны аналитические формулы

Симуляционный подход универсален, но дорог по вычислениям.

На предыдущих страницах для каждого параметра выполнялось:

```text
сетка параметров
    × число симуляций
    × две группы
    × генерация выборок
    × статистический тест.
```

Если:

```text
10 размеров выборки;
10 000 симуляций;
100 000 наблюдений в каждой группе,
```

объем вычислений становится огромным.

Аналитическая формула дает ответ почти мгновенно.

Но скорость достигается за счет более сильных предположений о данных.

---

# 3. Формула размера выборки

Для каждой группы:

```text
                  2 × σ² × (z_(1-α/2) + z_(1-β))²
n = -------------------------------------------------------
                         MDE_absolute²
```

Где:

```text
n             — размер каждой группы;
σ             — стандартное отклонение метрики;
σ²            — дисперсия;
α             — ошибка I рода;
β             — ошибка II рода;
1 - β         — мощность;
MDE_absolute  — эффект в абсолютных единицах метрики;
z             — квантиль стандартного нормального распределения.
```

---

# 4. Почему в формуле появляется коэффициент 2

Сравниваются два независимых средних:

```text
mean(B) - mean(A)
```

Если обе группы имеют одинаковую дисперсию `σ²` и одинаковый размер `n`, то дисперсия разницы средних:

```text
Var(mean(B) - mean(A))
= σ² / n + σ² / n
= 2σ² / n
```

Отсюда появляется:

```text
2 × σ²
```

---

# 5. Что такое квантиль z

Стандартное нормальное распределение:

```text
Z ~ Normal(0, 1)
```

`stats.norm.ppf(probability)` возвращает такое значение `z`, что:

```text
P(Z <= z) = probability
```

Пример:

```python
stats.norm.ppf(0.975)
```

дает примерно:

```text
1.96
```

Потому что около 97.5% стандартного нормального распределения находится левее 1.96.

---

# 6. Почему используется `1 - alpha / 2`

Альтернатива двусторонняя:

```text
H1: средние различаются
```

Ошибка `alpha` распределяется между двумя хвостами:

```text
alpha / 2 слева;
alpha / 2 справа.
```

Поэтому квантиль:

```python
stats.norm.ppf(1 - alpha / 2)
```

Для:

```text
alpha = 0.05
```

получаем:

```text
z_(1-0.05/2)
= z_0.975
≈ 1.96
```

---

# 7. Почему используется `1 - beta`

Мощность:

```text
power = 1 - beta
```

Поэтому второй квантиль:

```python
stats.norm.ppf(1 - beta)
```

При:

```text
beta = 0.20
```

мощность:

```text
80%
```

и:

```text
z_0.80 ≈ 0.84
```

Сумма квантилей:

```text
1.96 + 0.84 ≈ 2.80
```

---

# 8. Относительный и абсолютный MDE

Это центральный момент задания.

В `calculate_sample_size` параметр `mde` передается относительно.

Пример:

```text
reward_avg = 100
mde = 0.10
```

Это означает рост на 10% относительно текущего среднего.

Абсолютный эффект:

```text
absolute_mde = 100 × 0.10 = 10
```

Именно число `10` должно попасть в формулу.

В коде:

```python
absolute_mde = reward_avg * mde
```

---

# 9. Почему нельзя подставлять относительный MDE напрямую

Неправильно:

```text
reward_avg = 100
mde = 0.10
```

и в знаменатель подставить:

```text
0.10²
```

Тогда единицы измерения не совпадут.

Числитель содержит дисперсию метрики:

```text
денежные единицы²
```

Знаменатель должен также быть:

```text
денежные единицы²
```

Поэтому нужен:

```text
absolute_mde²
```

---

# 10. Почему sample size округляется вверх

Формула может дать:

```text
784.31
```

Невозможно набрать 0.31 наблюдения.

Округление вниз:

```text
784
```

может дать немного меньшую мощность, чем запланировано.

Поэтому используется:

```python
np.ceil(...)
```

Результат:

```text
785
```

Это минимальное целое число, не меньше рассчитанного.

---

# 11. Обратная формула для MDE

Из формулы размера выборки выражаем MDE:

```text
MDE_absolute
=
(z_(1-α/2) + z_(1-β))
× sqrt(2σ² / n)
```

Эквивалентно:

```text
MDE_absolute
=
(z_(1-α/2) + z_(1-β))
× sqrt(2)
× σ
/ sqrt(n)
```

Именно эта форма реализована в коде.

---

# 12. Почему `calculate_mde` возвращает абсолютный эффект

Условие отдельно говорит:

```text
результат должен быть в тех же единицах,
в которых измеряется исходная метрика.
```

Если reward измеряется в евро, MDE возвращается в евро.

Пример:

```text
calculate_mde(...) = 2.5
```

означает:

```text
тест способен обнаружить изменение среднего примерно на 2.5 евро.
```

Чтобы получить относительный MDE, нужен `reward_avg`:

```text
relative_mde = absolute_mde / reward_avg
```

Но `reward_avg` не входит в API `calculate_mde`, поэтому функция возвращает абсолютную величину.

# 13. Бизнесовый пример

Пусть:

```text
средний reward = 100 евро;
стандартное отклонение = 30 евро;
желаемый относительный MDE = 10%;
alpha = 5%;
beta = 20%;
```

Абсолютный MDE:

```text
100 × 0.10 = 10 евро
```

Квантили:

```text
z_(0.975) ≈ 1.96
z_(0.80)  ≈ 0.84
```

Размер каждой группы:

```text
n
≈ 2 × 30² × (1.96 + 0.84)² / 10²
≈ 141.3
```

После округления вверх:

```text
142 наблюдения на группу.
```

Всего при сплите 50/50:

```text
284 наблюдения.
```

---

# 14. Как интерпретировать расчет

Результат не означает:

```text
142 наблюдения гарантируют успех конкретного теста.
```

Он означает:

> При выполнении допущений, истинном эффекте заданного размера, выбранных alpha и beta такой дизайн имеет расчетную мощность примерно `1 - beta`.

Конкретный эксперимент все равно может:

- обнаружить эффект;
- не обнаружить эффект;
- дать случайно более сильную или слабую оценку.

# 15. Как параметры влияют на sample size

## Стандартное отклонение растет

```text
шум больше → выборка больше.
```

Причем:

```text
n пропорционально σ².
```

Если `reward_std` увеличить в два раза:

```text
sample size увеличится примерно в четыре раза.
```

## MDE уменьшается

```text
хотим ловить меньший эффект → выборка растет.
```

Причем:

```text
n пропорционально 1 / MDE².
```

Если MDE уменьшить в два раза:

```text
выборка вырастет примерно в четыре раза.
```

## Alpha уменьшается

Требование становится строже.

```text
меньше ложных побед → больше выборка.
```

## Beta уменьшается

Требуется более высокая мощность.

```text
меньше пропущенных эффектов → больше выборка.
```

---

# 16. Как параметры влияют на MDE

Формула:

```text
MDE ∝ σ / sqrt(n)
```

## `reward_std` растет

Обнаружимый эффект становится больше.

## `sample_size` растет

Обнаружимый эффект уменьшается.

Но уменьшается как:

```text
1 / sqrt(n)
```

Чтобы уменьшить MDE в два раза, размер группы нужно увеличить примерно в четыре раза.

---

# 17. Почему формула может плохо работать для CPC

CPC из прошлых страниц:

```text
Action × Reward
```

имеет:

- много нулей;
- асимметрию;
- смесь распределений;
- иногда тяжелый хвост;
- возможные отрицательные значения при нормальном reward;
- дисперсию, зависящую от CVR.

Такое распределение не похоже на нормальное.

При больших выборках среднее может приближаться к нормальному по центральной предельной теореме, но точность аналитической формулы все равно нужно проверять симуляциями.

Лучший практический подход:

```text
1. Быстро получить первоначальную оценку по формуле.
2. Проверить ее симуляцией на реалистичном распределении.
3. Добавить запас на реальные проблемы данных.
```

---

# 18. Почему формула предполагает одинаковые дисперсии

Используется:

```text
2σ² / n
```

Это верно, если:

```text
σ_A² = σ_B² = σ²
```

Если дисперсии сильно различаются, нужен более общий расчет:

```text
σ_A² / n_A + σ_B² / n_B
```

или симуляционный подход.

# 19. Где это применяется в бизнесе

## Финтех

Расчет для:

- среднего дохода на клиента;
- средней маржи;
- суммы транзакций;
- времени обработки;
- среднего баланса.

Но для бинарных метрик, например approval rate, чаще используют формулы для пропорций.

## E-commerce

Метрики:

- average order value;
- revenue per visitor;
- margin per user;
- среднее время доставки.

Денежные метрики часто имеют тяжелые хвосты, поэтому формулу нужно проверять.

## Marketplace

Метрики:

- комиссия на пользователя;
- средний чек;
- time to match;
- доход продавца.

Нужно учитывать кластеризацию и interference.

## AI-продукты

Метрики:

- средняя стоимость ответа;
- среднее время ответа;
- средний пользовательский score.

Повторные запросы одного пользователя могут быть зависимы.

## Ритейл

Метрики:

- средняя выручка магазина;
- продажи SKU;
- средний чек;
- потери.

При рандомизации магазинов единицей становится магазин, а не транзакция.

# 20. Формула и продуктовая экономика

MDE нельзя выбирать только потому, что его удобно обнаружить.

Нужно спросить:

```text
Какой эффект окупает внедрение?
```

Пример:

```text
новая инфраструктура стоит 100 000 евро в год;
эффект 0.1% приносит только 20 000 евро;
эффект 1% приносит 200 000 евро.
```

Тогда минимально бизнес-полезный MDE может быть около 1%, а не 0.1%.

---

# 21. Что реализовано в решении

## `calculate_sample_size`

1. проверяет `mde > 0`;
2. переводит относительный MDE в абсолютный;
3. получает два нормальных квантиля;
4. применяет формулу;
5. округляет вверх;
6. возвращает `int`.

## `calculate_mde`

1. получает два нормальных квантиля;
2. применяет обратную формулу;
3. возвращает абсолютный MDE как `float`.

# 22. Главный вывод

Аналитические формулы быстрые, но основаны на допущениях.

Симуляции медленнее, но гибче.

В реальной работе:

```text
формула дает стартовую оценку;
симуляция валидирует дизайн;
бизнес определяет полезный MDE;
экспериментальная платформа проверяет дизайн A/A-тестом.
```
