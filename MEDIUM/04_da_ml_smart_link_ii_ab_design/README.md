# Smart-Link II: A/B Design — страница 1

## Что находится в комплекте

Эта страница содержит вводную теорию. Поэтому здесь **нет придуманного программного решения** и нет файла `.py`.

Файлы:

1. `gp_page_1_smart_link_ii_ab_design_context.md`  
   Большой разбор темы: бизнес-задача, устройство Smart-Link, логика A/B-теста, гипотеза, метрики, сплитование, MDE, мощность, ошибки I/II рода, размер выборки, мониторинг, анализ, риски и примеры из разных индустрий.

2. `gp_page_1_smart_link_ii_ab_design_product_and_system_design_interview.md`  
   Как отвечать на продуктовом, аналитическом и system design интервью по теме A/B-тестов и рекомендательных систем.

3. `gp_page_1_smart_link_ii_ab_design_practical_checklist.md`  
   Практический шаблон дизайна эксперимента, который можно заполнять перед запуском теста.

## Главная цель страницы

Понять не отдельную формулу, а весь путь принятия решения:

> бизнес-проблема → гипотеза → метрики → единица рандомизации → дизайн → размер выборки → запуск → мониторинг → статистический анализ → продуктовое решение.

## Что важно запомнить

A/B-тест — это не просто сравнение двух средних и не просто вызов статистического теста.

Хороший эксперимент должен:

- действительно измерять причинный эффект новой версии;
- не смешивать пользователей между группами;
- иметь заранее выбранную главную метрику;
- иметь достаточно наблюдений;
- не останавливаться после первого красивого `p-value`;
- учитывать качество данных и работу инфраструктуры;
- завершаться бизнес-решением, а не только статистическим выводом.
