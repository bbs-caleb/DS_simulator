# 7. Проверка, что возвращается β, а не мощность

Для очень сильного эффекта тест почти всегда значим.

Тогда:

```text
β близка к 0;
Power близка к 1.
```

Если функция возвращает около 1, вероятно, разработчик вернул мощность вместо ошибки II рода.

# 9. Проверка MDE = 0

```python
np.random.seed(42)

beta = ab_test(
    5_000,
    1_000,
    0.10,
    0.0,
    10.0,
    2.0,
    alpha=0.05,
)

print(beta)
```

При `mde=0` реального эффекта нет.

Функция все равно считает отсутствие значимости ошибкой II рода по своей механике.

Поэтому значение будет примерно:

```text
1 - alpha ≈ 0.95
```

Но концептуально это уже A/A-режим, и термин «ошибка II рода» здесь неприменим, потому что H0 истинна.

---

# 10. Ограничение conversion_rate

Нужно, чтобы:

```text
conversion_rate × (1 + mde) <= 1
```

Иначе NumPy получит недопустимую вероятность.

Пример некорректного дизайна:

```text
conversion_rate = 0.80
mde = 0.50
```

Тогда:

```text
0.80 × 1.50 = 1.20
```

Вероятность не может быть больше 1.

Условие предполагает корректные параметры.

В решение не добавляется clipping, потому что он скрывал бы ошибку пользователя и менял бы MDE.

# 11. Комплексная проверка

```python
import numpy as np

from solution import ab_test, cpc_sample, t_test


def run_checks():
    np.random.seed(42)

    cpc_a = cpc_sample(
        20_000,
        0.10,
        10.0,
        2.0,
    )
    cpc_b = cpc_sample(
        20_000,
        0.15,
        10.0,
        2.0,
    )

    is_significant, p_value = t_test(
        cpc_a,
        cpc_b,
    )

    assert type(is_significant) is bool
    assert type(p_value) is float
    assert is_significant

    np.random.seed(42)

    beta = ab_test(
        1_000,
        5_000,
        0.10,
        0.50,
        10.0,
        2.0,
    )

    assert type(beta) is float
    assert 0.0 <= beta <= 1.0
    assert beta < 0.10


run_checks()
print("All checks passed.")
```

# 12. Распределение p-value в A/A

При истинной H0:

```text
p-values приблизительно Uniform(0, 1)
```

Отсюда:

```text
около 5% ниже 0.05.
```

---

# 13. Распределение p-value в A/B

При реальном эффекте p-value смещаются к нулю.

Чем выше мощность:

- тем больше плотность около 0;
- тем больше доля ниже alpha;
- тем меньше β.

---

# 14. Численная связь p-value, power и β

После A/B-симуляции:

```python
power = np.mean(ab_p_values < alpha)
beta = np.mean(ab_p_values >= alpha)
```

Поэтому:

```python
power + beta = 1
```

с точностью до особенностей `nan`.

# 16. Почему A/B распределение не обязано полностью находиться слева от 0.05

Даже при реальном эффекте часть экспериментов дает:

```text
p-value >= 0.05
```

Именно эти случаи формируют ошибку II рода.

Если все p-value ниже alpha, оцененная мощность равна 100%.

---

# 17. Что происходит при маленьком MDE

A/B p-value начинают выглядеть ближе к A/A.

Причина:

```text
эффект мал относительно шума.
```

Тесту трудно отличить группы.

---

# 18. Что происходит при большом n_samples

A/B p-value сильнее концентрируются около нуля.

Потому что средние оцениваются точнее, а стандартная ошибка уменьшается.

---

# 19. Возможные `nan`

При очень маленьких выборках, низком CVR или нулевой дисперсии SciPy может вернуть `nan`.

Сравнение:

```python
nan < alpha
```

дает `False`.

Тогда итерация будет считаться ошибкой II рода.

В реальном дизайне большое число `nan` означает, что выбранный тест или размер выборки непригоден.
