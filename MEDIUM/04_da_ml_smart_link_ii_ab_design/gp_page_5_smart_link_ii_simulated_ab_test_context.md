# 1. Что требуется сделать

На предыдущих страницах были реализованы:

```python
cpc_sample(...)
t_test(...)
```

На этой странице их API и логика должны сохраниться.

Нужно добавить новую функцию:

```python
def ab_test(
    n_simulations: int,
    n_samples: int,
    conversion_rate: float,
    mde: float,
    reward_avg: float,
    reward_std: float,
    alpha: float = 0.05,
) -> float:
```

Она должна много раз симулировать A/B-эксперимент, в котором:

```text
группа A:
conversion_rate_A = conversion_rate

группа B:
conversion_rate_B = conversion_rate × (1 + mde)
```

Реальный эффект специально заложен в данные.

Функция должна вернуть долю симуляций, в которых t-test не обнаружил этот существующий эффект.

Эта доля является оценкой ошибки II рода:

```text
β
```

---

# 2. Чем эта страница отличается от A/A-теста

## A/A-тест

Обе группы создаются одинаково:

```text
CVR_A = CVR_B
```

Реального эффекта нет.

Ошибка:

```text
тест нашел различие, хотя различия нет.
```

Это ошибка I рода.

## A/B-тест

Группы создаются по-разному:

```text
CVR_B > CVR_A
```

Реальный эффект есть.

Ошибка:

```text
тест не нашел различие, хотя различие существует.
```

Это ошибка II рода.

---

# 3. Бизнесовый смысл ошибки II рода

Представим, что новый Smart-Link действительно повышает CPC.

Но проведенный эксперимент оказался недостаточно чувствительным.

Аналитик получает:

```text
p-value >= 0.05
```

и делает вывод:

```text
статистически значимого улучшения не обнаружено.
```

Компания может:

- отказаться от полезного алгоритма;
- не раскатить прибыльное изменение;
- потерять дополнительную выручку;
- потратить время команды без результата;
- ошибочно решить, что гипотеза не работает.

Это и есть бизнесовая цена ошибки II рода.

---

# 4. Пример из OrangeChicken

Пусть контроль имеет:

```text
conversion_rate = 0.10
reward_avg = 10
```

Ожидаемый CPC:

```text
0.10 × 10 = 1.00
```

Новый Smart-Link повышает конверсию на 10% относительно.

```text
mde = 0.10
```

Тогда:

```text
conversion_rate_B
= 0.10 × (1 + 0.10)
= 0.11
```

Ожидаемый CPC группы B:

```text
0.11 × 10 = 1.10
```

Реальный относительный рост ожидаемого CPC:

```text
10%
```

Но конкретные выборки шумные:

- многие клики приносят 0;
- число конверсий случайно;
- reward имеет разброс;
- средние A и B могут случайно оказаться близкими.

Поэтому отдельный тест иногда не обнаружит эффект.

---

# 5. Что такое MDE в этой задаче

MDE означает:

```text
minimal detectable effect
```

В условии он задается как относительное изменение.

Формула:

```text
новое значение = базовое значение × (1 + MDE)
```

Пример:

```text
conversion_rate = 0.20
mde = 0.10
```

Тогда:

```text
conversion_rate_B
= 0.20 × 1.10
= 0.22
```

Абсолютный рост:

```text
0.22 - 0.20 = 0.02
```

То есть:

```text
+2 процентных пункта
```

Относительный рост:

```text
0.02 / 0.20 = 10%
```

Нельзя путать:

```text
10% относительного роста
```

и:

```text
10 процентных пунктов.
```

---

# 6. Важная точность по условию

В одном предложении текста упоминается изменение `reward_avg`.

Но шаблон и комментарии функции однозначно требуют менять именно:

```text
conversion_rate
```

Правильная реализация:

```python
test_conversion_rate = conversion_rate * (1.0 + mde)
```

`reward_avg` и `reward_std` остаются одинаковыми в A и B.

---

# 7. Что такое ошибка II рода

Нулевая гипотеза:

```text
H0: средние CPC групп A и B равны.
```

Альтернативная:

```text
H1: средние CPC различаются.
```

В симуляции мы специально создаем группы с разными CVR.

Значит, H0 фактически неверна.

Если t-test возвращает:

```python
False
```

то есть:

```text
p-value >= alpha
```

тест не смог отвергнуть ложную H0.

Это ошибка II рода.

Обозначение:

```text
β
```

---

# 8. Как кодируетcя ошибка II рода

`t_test` возвращает:

```python
is_significant
```

Если:

```python
is_significant is True
```

эффект обнаружен правильно.

Тогда ошибки II рода нет:

```text
0
```

Если:

```python
is_significant is False
```

реальный эффект пропущен.

Тогда:

```text
type II error = 1
```

В коде:

```python
type_2_errors[simulation_index] = not is_significant
```

---

# 9. Что такое мощность теста

Мощность:

```text
Power = 1 - β
```

Она показывает вероятность обнаружить заложенный эффект.

Пример:

```text
β = 0.20
```

Тогда:

```text
Power = 1 - 0.20 = 0.80
```

То есть тест обнаруживает такой эффект примерно в 80% повторных экспериментов.

---

# 10. Интерпретация мощности 80%

Мощность 80% не означает:

```text
результат текущего теста верен с вероятностью 80%.
```

Она означает:

> Если реальный эффект равен заданному MDE и все предпосылки соблюдены, то при многократном повторении такого дизайна примерно 80% экспериментов дадут статистически значимый результат.

Оставшиеся примерно 20%:

```text
ошибка II рода.
```

---

# 11. Связь MDE и β

Чем больше реальный эффект, тем легче его отличить от шума.

## Маленький MDE

Средние A и B близки.

Распределения сильно перекрываются.

Тест часто не обнаруживает разницу.

```text
β высокий
Power низкая
```

## Большой MDE

Средние дальше друг от друга.

Тест чаще получает маленький p-value.

```text
β низкий
Power высокая
```

---

# 12. Связь размера выборки и β

При большем `n_samples` средние оцениваются точнее.

Стандартная ошибка среднего уменьшается примерно как:

```text
1 / sqrt(n)
```

Поэтому одинаковый эффект легче обнаружить.

```text
n_samples растет
→ β уменьшается
→ Power растет
```

Это основа расчета размера выборки перед A/B-тестом.

---

# 13. Почему маленький эффект требует много данных

Упрощенно размер выборки масштабируется как:

```text
n ∝ 1 / effect²
```

Если нужно обнаруживать эффект в два раза меньше, данных может потребоваться примерно в четыре раза больше.

Пример:

```text
MDE 10% → условно 10 000 наблюдений
MDE 5%  → условно около 40 000 наблюдений
```

Точное число зависит от:

- baseline;
- дисперсии;
- alpha;
- требуемой мощности;
- распределения метрики;
- распределения трафика между группами.

---

# 14. Связь alpha и β

При фиксированных данных и размере выборки увеличение `alpha` делает критерий менее строгим.

Например:

```text
alpha = 0.01
```

требует очень маленький p-value.

```text
alpha = 0.10
```

позволяет объявить значимость легче.

Поэтому при увеличении alpha:

```text
ошибка I рода растет;
ошибка II рода обычно уменьшается;
мощность растет.
```

Это компромисс.

Нельзя повышать alpha только ради красивой мощности, не учитывая цену ложноположительного решения.

---

# 15. Почему нельзя максимизировать мощность любой ценой

Мощность можно увеличить:

- увеличив выборку;
- увеличив alpha;
- снизив дисперсию;
- выбрав более сильный эффект;
- улучшив статистический метод.

Но повышение alpha увеличивает ложные победы.

Очень большой тест требует:

- времени;
- трафика;
- денег;
- удержания части пользователей в контроле;
- инженерной поддержки.

Дизайн — это баланс между:

```text
ошибкой I рода;
ошибкой II рода;
стоимостью;
скоростью;
минимально полезным эффектом.
```

---

# 16. Алгоритм функции `ab_test`

Для каждой симуляции:

```text
1. Создать CPC_A с базовым CVR.
2. Создать CPC_B с CVR × (1 + MDE).
3. Сравнить выборки t-test.
4. Если значимости нет, записать 1.
5. Если значимость есть, записать 0.
```

После цикла:

```text
β_hat = среднее массива ошибок
```

---

# 17. Почему обе выборки независимы

Нельзя использовать одинаковый массив или общие actions.

Правильно:

```python
cpc_a = cpc_sample(...)
cpc_b = cpc_sample(...)
```

Два независимых вызова создают независимый случайный шум.

Это соответствует двум независимым экспериментальным группам.

---

# 18. Почему reward одинаковый в обеих группах

Задача хочет изолировать эффект изменения CVR.

Поэтому:

```text
reward_avg_A = reward_avg_B
reward_std_A = reward_std_B
```

Если одновременно изменить CVR и reward, будет непонятно:

- какой фактор создал эффект;
- как интерпретировать MDE;
- соответствует ли симуляция условию.

---

# 19. Ожидаемый CPC групп

Для A:

```text
E[CPC_A] = conversion_rate × reward_avg
```

Для B:

```text
E[CPC_B]
= conversion_rate × (1 + mde) × reward_avg
```

Следовательно:

```text
E[CPC_B] / E[CPC_A] - 1 = mde
```

При неизменном reward относительное изменение ожидаемого CPC совпадает с MDE по conversion rate.

---

# 20. Пример расчета

Пусть:

```text
conversion_rate = 0.08
mde = 0.25
reward_avg = 20
```

Группа A:

```text
CVR_A = 0.08
E[CPC_A] = 0.08 × 20 = 1.60
```

Группа B:

```text
CVR_B = 0.08 × 1.25 = 0.10
E[CPC_B] = 0.10 × 20 = 2.00
```

Относительный рост:

```text
2.00 / 1.60 - 1 = 25%
```

---

# 21. Что показывает распределение p-value в A/B

В A/A при истинной H0 p-value приблизительно равномерны на `[0, 1]`.

В A/B H0 неверна.

Если тест имеет мощность, p-value чаще оказываются близко к нулю.

Чем сильнее эффект и больше выборка:

- тем выше пик около нуля;
- тем больше доля `p < alpha`;
- тем выше мощность;
- тем ниже β.

---

# 22. Сравнение A/A и A/B p-value

## A/A

```text
приблизительно Uniform(0, 1)
```

Гистограмма относительно плоская.

## A/B

Распределение не равномерное.

Оно смещено к нулю.

Количество p-value ниже alpha равно приблизительной мощности теста:

```text
Power ≈ mean(p_value < alpha)
```

Количество p-value выше или равно alpha:

```text
β ≈ mean(p_value >= alpha)
```

---

# 23. Почему A/B p-value не имеют одной универсальной формы

Форма зависит от:

- MDE;
- n_samples;
- baseline CVR;
- reward_std;
- alpha не меняет сами p-value, но меняет границу решения;
- выбранного теста;
- структуры данных.

При слабом тесте распределение A/B p-value может быть почти похоже на A/A.

При очень мощном тесте почти все p-value будут около нуля.

---

# 24. Индустриальный смысл мощности

## Финтех

Слабый тест может пропустить небольшое, но прибыльное улучшение:

- конверсии заявки;
- активации карты;
- repayment;
- cross-sell.

Но нужно учитывать guardrails:

- default;
- fraud;
- complaints;
- fairness.

## E-commerce

Можно пропустить реальный рост:

- revenue per visitor;
- conversion;
- gross profit;
- retention.

При огромном трафике даже 1% может иметь большую денежную ценность.

## Marketplace

Маленький эффект на GMV может быть важен, но нужно учитывать:

- seller exposure;
- отмены;
- delivery;
- liquidity;
- долгосрочное предложение.

## AI-продукты

Можно пропустить улучшение task success или снижение стоимости.

Но высокая мощность не должна достигаться за счет игнорирования:

- hallucination;
- safety;
- latency;
- token cost.

## Dating

Небольшой рост meaningful conversations может быть ценным, но нельзя оптимизировать только объем взаимодействий без safety и fairness.

# 25. Почему «не значимо» не означает «эффекта нет»

Допустим, тест вернул:

```text
p-value = 0.12
```

Нельзя автоматически сказать:

```text
новый алгоритм не работает.
```

Возможны два сценария:

1. эффект действительно близок к нулю;
2. эффект есть, но тест недостаточно мощный.

Чтобы различить их, смотрят:

- доверительный интервал;
- MDE;
- рассчитанную мощность;
- фактический размер выборки;
- дисперсию;
- качество данных.

# 28. Главный вывод страницы

```text
Ошибка II рода β
= вероятность пропустить существующий эффект.

Мощность
= 1 - β
= вероятность обнаружить эффект заданного размера.
```

Мощность растет, когда:

- MDE увеличивается;
- n_samples увеличивается;
- дисперсия снижается;
- alpha становится менее строгим;
- статистический метод лучше использует данные.

В хорошем дизайне MDE выбирается не ради красивой статистики, а через минимально полезный бизнес-эффект.
