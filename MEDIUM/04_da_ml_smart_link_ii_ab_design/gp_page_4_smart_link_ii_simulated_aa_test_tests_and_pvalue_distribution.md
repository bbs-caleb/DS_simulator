# 4. Почему seed повторяется перед каждым вызовом

Мы хотим сравнить влияние порога `alpha` на максимально сопоставимых случайных данных.

Повторная установка одного seed воспроизводит одинаковую последовательность выборок.

Меняется только порог значимости.

# 7. Проверка независимости двух выборок

Правильная реализация дважды вызывает:

```python
cpc_sample(...)
```

Неправильно:

```python
cpc_b = cpc_a
```

Иначе p-value почти всегда равен единице и оценка ошибки I рода будет близка к нулю.

# 8. Полная самостоятельная проверка

```python
import numpy as np

from solution import aa_test, cpc_sample, t_test


def run_checks():
    np.random.seed(42)

    sample = cpc_sample(
        10_000,
        0.10,
        10.0,
        2.0,
    )

    assert sample.shape == (10_000,)

    is_significant, p_value = t_test(
        sample,
        sample.copy(),
    )

    assert is_significant is False
    assert p_value == 1.0

    np.random.seed(42)

    error_rate = aa_test(
        2_000,
        1_000,
        0.10,
        10.0,
        2.0,
        alpha=0.05,
    )

    assert type(error_rate) is float
    assert 0.03 < error_rate < 0.07


run_checks()
print("All checks passed.")
```

# 9. Дополнительное задание: форма p-value

Для корректного A/A-теста ожидается:

```text
p-value ~ Uniform(0, 1)
```

То есть приблизительно равномерное распределение.

Не нормальное.

Не биномиальное.

Не экспоненциальное.

Не распределение Стьюдента.

Сам t-статистический критерий использует t-распределение для вычисления вероятности, но преобразованные p-value при истинной H0 должны быть приблизительно равномерными.

---

# 10. Практическая проверка равномерности

Можно разбить `[0, 1]` на десять интервалов:

```text
[0.0, 0.1)
[0.1, 0.2)
...
[0.9, 1.0]
```

При 10 000 симуляций ожидается примерно:

```text
около 1 000 p-value в каждом интервале.
```

Конечно, значения не будут ровно одинаковыми.

---

# 11. Доля p-value ниже порогов

Для Uniform(0, 1):

```text
P(p < 0.01) = 0.01
P(p < 0.05) = 0.05
P(p < 0.10) = 0.10
P(p < 0.50) = 0.50
```

Проверка:

```python
print(np.mean(p_values < 0.01))
print(np.mean(p_values < 0.05))
print(np.mean(p_values < 0.10))
print(np.mean(p_values < 0.50))
```

# 13. Почему график не идеально плоский

Причины:

- конечное число симуляций;
- случайный шум;
- ненормальность CPC;
- дискретность числа конверсий;
- приближенное поведение t-test;
- возможные редкие вырожденные выборки.

Равномерность ожидается приблизительно, а не идеально.

---

# 14. Что означает сильный пик около нуля

Если p-value подозрительно часто близки к нулю, возможны:

1. группы реально различаются;
2. в A и B переданы разные параметры;
3. наблюдения зависимы;
4. стандартная ошибка занижена;
5. нарушена рандомизация;
6. происходит p-hacking;
7. тест не соответствует данным.

---

# 15. Что означает пик около единицы

Возможны:

1. одна выборка копируется в другую;
2. данные чрезмерно одинаковы;
3. тест слишком консервативен;
4. выборки вырождены;
5. большая доля `nan` была некорректно заменена единицей.

# 16. Граничный случай: `n_simulations=0`

Тогда:

```python
np.zeros(0).mean()
```

дает `nan` и предупреждение.

Условие предполагает положительное число симуляций.

В решение не добавлена лишняя валидация, чтобы сохранить минимальный ожидаемый API.

---

# 17. Граничный случай: все CPC равны нулю

При:

```text
conversion_rate = 0
```

обе выборки состоят из нулей.

Welch t-test не может оценить стандартную ошибку и может вернуть:

```text
p-value = nan
```

Сравнение:

```python
nan < alpha
```

дает `False`.

В результате такие итерации не считаются ошибками I рода.

Но это не полноценная статистическая проверка — это вырожденные данные.

---

# 18. Почему не обрабатывается `nan` вручную

Условие не задает специального поведения.

Самовольная замена:

```python
nan → 1.0
```

или:

```python
nan → 0.0
```

может скрыть проблему и изменить ожидаемые автотесты.

В учебном решении сохраняется стандартное поведение SciPy.

---

# 19. Производительность

При:

```text
10 000 симуляций;
1 000 наблюдений в каждой группе;
```

генерируется:

```text
20 миллионов CPC.
```

Кроме генерации, 10 000 раз вызывается SciPy t-test.

Поэтому симуляция может занять заметное время.

Векторизовать абсолютно весь процесс можно, но:

- это усложнит код;
- потребует много памяти;
- не соответствует предложенному шаблону;
- ухудшит понятность для новичка.

Цикл является правильным решением для задания.
