# Функции потерь и их градиенты — MSE и MAE

## 1. Бизнесовый контекст

Мы продолжаем строить градиентный бустинг для прогноза просрочки кредита. На предыдущем шаге первое "дерево" ансамбля предсказывало всем клиентам одну и ту же константу — среднюю просрочку по выборке. Это грубое приближение, и каждое следующее дерево должно его улучшать.

Но чтобы улучшать — нужно понимать, **в какую сторону** двигать прогноз для каждого конкретного клиента. Для этого и нужна функция потерь: она формально определяет, что мы считаем "плохим" прогнозом, а её градиент говорит, куда именно нужно сдвинуть прогноз, чтобы уменьшить ошибку. Банку важно верно выбрать функцию потерь: MSE сильнее штрафует за большие ошибки (например, если предсказали просрочку в 5 дней, а по факту 60 — это очень плохо), MAE устойчивее к редким "выбросам" в данных (например, к одному клиенту с аномально долгой просрочкой).

## 2. Технический контекст

### Что такое градиент в этой задаче

У нас есть вектор истинных значений `y_true` (реальная просрочка каждого клиента) и вектор предсказаний `y_pred` (то, что выдал ансамбль деревьев на текущий момент). Функция потерь — это одно число, усредняющее "ошибку" по всем объектам. Но чтобы понять, как поменять предсказание **для каждого отдельного объекта**, нам нужна не сама функция потерь (одно число), а её **производная по каждому предсказанию `y_pred_i`** — то есть вектор той же длины, что и `y_true`/`y_pred`. Это и есть градиент.

### Вывод формулы для MSE

MSE считается так:

```
MSE = (1/n) * Σ (y_pred_i - y_true_i)²
```

Берём производную одного слагаемого `(y_pred_i - y_true_i)²` по `y_pred_i` (по правилу дифференцирования сложной функции: производная `u²` равна `2u`, где `u = y_pred_i - y_true_i`):

```
d/d(y_pred_i) [(y_pred_i - y_true_i)²] = 2 * (y_pred_i - y_true_i)
```

Константу `2` мы, как явно указано в задании, можем отбросить — она всё равно компенсируется размером шага градиентного спуска на следующих этапах. Получаем:

```
grad_i = y_pred_i - y_true_i
```

То есть для MSE градиент — это просто обычный остаток (разница между предсказанием и истиной), поэтому многие путают "приближение ошибки" и "приближение градиента ошибки" — для MSE они совпадают с точностью до множителя 2, но это не общее правило (для MAE — уже не так).

### Вывод формулы для MAE

MAE считается так:

```
MAE = (1/n) * Σ |y_pred_i - y_true_i|
```

Производная модуля `|u|` по `u` равна `+1`, если `u > 0`, и `-1`, если `u < 0` (в нуле производная не определена, но на практике библиотеки NumPy в этой точке возвращают 0 — это не критично, так как вероятность точного совпадения `y_pred_i == y_true_i` для непрерывных величин практически нулевая). Подставляя `u = y_pred_i - y_true_i`, получаем:

```
grad_i = sign(y_pred_i - y_true_i)
```

где `sign` — функция знака.

## 3. Разбор кода по шагам

Файл целиком: `ml_gradient_boosting_02.py`.

```python
"""Loss functions for gradient boosting: MSE and MAE with their gradients."""

from typing import Tuple

import numpy as np
```

- Docstring модуля описывает назначение файла.
- `from typing import Tuple` — импортируем тип `Tuple` для аннотации возвращаемого значения (кортеж из float и np.ndarray), как того требует шаблон API.
- `import numpy as np` — нужен для векторных операций (`**`, `np.mean`, `np.abs`, `np.sign`).

### Функция `mse`

```python
def mse(y_true: np.ndarray, y_pred: np.ndarray) -> Tuple[float, np.ndarray]:
    """Mean squared error loss function and gradient."""
    diff = y_pred - y_true  # residuals: prediction minus true value
    loss = np.mean(diff ** 2)  # average of squared residuals
    grad = diff  # gradient d(diff^2)/d(y_pred) = 2 * diff, constant 2 dropped
    return loss, grad
```

Пошагово:
1. `diff = y_pred - y_true` — считаем разницу между предсказанием и истинным значением **для каждого объекта одновременно** (NumPy сам применяет операцию поэлементно ко всему вектору — это называется векторизацией, отдельный цикл `for` не нужен).
2. `loss = np.mean(diff ** 2)` — возводим каждый элемент `diff` в квадрат (`diff ** 2`, тоже поэлементно), затем берём среднее по всем элементам (`np.mean`) — это в точности формула MSE.
3. `grad = diff` — как мы вывели выше, градиент MSE (с точностью до отброшенной константы 2) равен просто `diff`, то есть вектору обычных остатков.
4. `return loss, grad` — возвращаем кортеж: сначала число (значение функции потерь), затем вектор (градиент) — именно в этом порядке требует шаблон.

### Функция `mae`

```python
def mae(y_true: np.ndarray, y_pred: np.ndarray) -> Tuple[float, np.ndarray]:
    """Mean absolute error loss function and gradient."""
    diff = y_pred - y_true  # residuals: prediction minus true value
    loss = np.mean(np.abs(diff))  # average of absolute residuals
    grad = np.sign(diff)  # gradient is +1, -1, or 0 depending on diff's sign
    return loss, grad
```

Пошагово:
1. `diff = y_pred - y_true` — та же разница, что и в `mse`.
2. `loss = np.mean(np.abs(diff))` — `np.abs` берёт модуль каждого элемента вектора `diff`, затем `np.mean` усредняет — получаем MAE.
3. `grad = np.sign(diff)` — функция `np.sign` возвращает поэлементно `+1`, если элемент положительный, `-1`, если отрицательный, и `0`, если равен нулю. Это ровно вектор градиента, который мы вывели: `+1` там, где предсказание завышено, `-1` — где занижено.
4. `return loss, grad` — возвращаем кортеж в том же порядке.

## 4. Проверка работоспособности (пример для отладки)

```python
import numpy as np
from ml_gradient_boosting_02 import mse, mae

y_true = np.array([3.0, 5.0, 2.5, 7.0])
y_pred = np.array([2.5, 5.0, 4.0, 8.0])

loss, grad = mse(y_true, y_pred)
print(loss)  # 0.875
print(grad)  # [-0.5  0.   1.5  1. ]

loss, grad = mae(y_true, y_pred)
print(loss)  # 0.75
print(grad)  # [-1.  0.  1.  1.]
```

Разберём вручную первый объект: `y_true=3.0`, `y_pred=2.5`, значит `diff = 2.5 - 3.0 = -0.5`. Для MSE вклад в градиент — это сам `diff`, то есть `-0.5` (модель предсказала меньше истины — градиент отрицательный, значит нужно "подвинуть" предсказание вверх). Для MAE вклад — это `sign(-0.5) = -1.0`. Я прогнал этот пример на реальном коде — результат совпал с ручным расчётом.
