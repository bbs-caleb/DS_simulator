# 2. Полный код

```python
import numpy as np
import uvicorn
from fastapi import FastAPI

app = FastAPI()


@app.get("/sample/")
def sample(offer_ids: str) -> dict:
    """Sample random offer"""
    # Parse offer IDs
    offers_ids = [int(offer) for offer in offer_ids.split(",")]

    # Sample random offer ID
    offer_id = int(np.random.choice(offers_ids))

    # Prepare response
    response = {
        "offer_id": offer_id,
    }

    return response


def main() -> None:
    """Run application"""
    uvicorn.run("gp_page_2_random_sampler_app:app", host="localhost")


if __name__ == "__main__":
    main()
```

# 7. Парсинг входной строки

```python
offers_ids = [int(offer) for offer in offer_ids.split(",")]
```

Это наиболее насыщенная строка программы. Разберём её справа налево.

## Шаг 1. Исходная строка

Допустим:

```python
offer_ids = "2,5,1"
```

Обратите внимание: это одна строка, а не список.

## Шаг 2. Разделение строки

```python
offer_ids.split(",")
```

Метод `.split(",")` разрезает строку в местах, где встречается запятая.

Получаем:

```python
["2", "5", "1"]
```

Это уже список, но его элементы всё ещё являются строками.

## Шаг 3. Цикл внутри list comprehension

```python
for offer in offer_ids.split(",")
```

Python по очереди берёт каждый элемент:

```text
"2"
"5"
"1"
```

и временно сохраняет его в переменной `offer`.

## Шаг 4. Преобразование в число

```python
int(offer)
```

Функция `int()` превращает текстовое число в целое число.

```python
int("2")  # 2
int("5")  # 5
int("1")  # 1
```

## Шаг 5. Создание нового списка

Квадратные скобки собирают результаты:

```python
[2, 5, 1]
```

Именно этот список сохраняется в `offers_ids`.

---

# 8. Случайный выбор

```python
offer_id = int(np.random.choice(offers_ids))
```

## `np.random`

Модуль случайных операций библиотеки NumPy.

## `.choice(offers_ids)`

Функция получает список:

```python
[2, 5, 1]
```

и выбирает из него один элемент.

В одном вызове может вернуться:

```python
2
```

В другом:

```python
5
```

В следующем:

```python
1
```

## Почему не используется сортировка

Сортировка не создаёт случайность.

Код:

```python
sorted([2, 5, 1])[0]
```

всегда возвращает `1`.

Код:

```python
np.random.choice([2, 5, 1])
```

может вернуть любой элемент списка.

## Почему добавлен `int`

NumPy иногда возвращает собственный числовой тип:

```text
numpy.int64
```

Для JSON безопаснее использовать обычный Python `int`.

Поэтому:

```python
int(...)
```

превращает результат в стандартное целое число.

# 17. Ручная проверка случайности

Одного вызова недостаточно. Даже случайный алгоритм может случайно несколько раз подряд вернуть одно значение.

Можно вызвать функцию много раз:

```python
from collections import Counter

from gp_page_2_random_sampler_app import sample

results = [
    sample("2,5,1")["offer_id"]
    for _ in range(10_000)
]

print(Counter(results))
```

Пример результата:

```text
Counter({2: 3371, 1: 3338, 5: 3291})
```

Числа не обязаны совпадать идеально.

При случайном выборе нормальны небольшие отклонения.

Главное:

- встречаются все допустимые офферы;
- значения находятся только в `{1, 2, 5}`;
- один и тот же оффер не выбирается всегда;
- при большом числе испытаний частоты близки.

# 18. Какую вероятность имеет каждый оффер

Если кандидатов три:

```python
[2, 5, 1]
```

и `np.random.choice` вызывается без параметра вероятностей, выбор равномерный.

Для каждого оффера:

```text
P = 1 / 3
```

То есть приблизительно:

```text
33.33%
```

Если кандидатов десять:

```text
P = 1 / 10 = 10%
```

Если кандидат один:

```python
[7]
```

результат всегда будет `7`, и это правильно: выбирать больше не из чего.

# 19. Возможные неправильные решения

## Неправильно: первый элемент

```python
offer_id = offers_ids[0]
```

Всегда выбирается первый элемент.

## Неправильно: последний элемент

```python
offer_id = offers_ids[-1]
```

Всегда выбирается последний элемент.

## Неправильно: минимальный элемент

```python
offer_id = min(offers_ids)
```

Всегда выбирается минимальный ID.

## Неправильно: максимальный элемент

```python
offer_id = max(offers_ids)
```

Всегда выбирается максимальный ID.

## Неправильно: случайное число между минимумом и максимумом

```python
offer_id = np.random.randint(min(offers_ids), max(offers_ids) + 1)
```

Так можно вернуть ID, которого нет среди кандидатов.

Например, для:

```python
[2, 5]
```

может быть возвращено `3` или `4`.

Это нарушает условие.

## Правильно: выбор из самих кандидатов

```python
offer_id = int(np.random.choice(offers_ids))
```

Результат всегда принадлежит входному списку.

offer_id = int(np.random.choice(offers_ids))

# 21. Итоговый алгоритм словами

Для каждого HTTP-запроса:

1. получить строку с идентификаторами;
2. разделить её по запятым;
3. преобразовать идентификаторы в целые числа;
4. случайно выбрать один элемент списка;
5. привести его к обычному `int`;
6. поместить его в словарь;
7. вернуть JSON-ответ.

Это полностью соответствует условию Random Sampler.
