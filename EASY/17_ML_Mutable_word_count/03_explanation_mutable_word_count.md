# Подробное объяснение с нуля

Разберем код:

```python
{True: 'yes', 1: 'no', 1.0: 'probably'}
```

На первый взгляд кажется, что в словаре будет три пары:

```python
True -> 'yes'
1 -> 'no'
1.0 -> 'probably'
```

Но в Python это не так.

## 1. Что такое словарь

Словарь — это структура данных вида:

```python
ключ -> значение
```

Пример:

```python
person = {
    "name": "Nodir",
    "age": 25
}
```

Здесь:

```python
"name"
```

это ключ, а:

```python
"Nodir"
```

это значение.

## 2. Что важно для ключей словаря

Ключи словаря должны быть хэшируемыми.

То есть Python должен уметь взять от ключа хэш:

```python
hash(key)
```

Пример:

```python
hash("name")
hash(1)
hash(True)
```

Это работает.

А вот так нельзя:

```python
hash([1, 2, 3])
```

Потому что список — изменяемый объект.

## 3. Почему изменяемые объекты нельзя использовать как ключи

Представь, что список можно было бы использовать как ключ:

```python
my_dict = {
    [1, 2, 3]: "value"
}
```

Потом список изменился бы:

```python
[1, 2, 3] -> [1, 2, 3, 4]
```

И Python уже не смог бы надежно найти этот ключ в словаре.

Поэтому ключами словаря могут быть только объекты, которые не меняются: строки, числа, кортежи, булевы значения.

## 4. Что такое хэш простыми словами

Хэш — это числовой идентификатор объекта.

Например:

```python
hash(True)
```

вернет:

```python
1
```

```python
hash(1)
```

тоже вернет:

```python
1
```

```python
hash(1.0)
```

тоже вернет:

```python
1
```

То есть:

```python
hash(True) == hash(1) == hash(1.0)
```

это правда.

## 5. Почему `True`, `1` и `1.0` считаются одинаковыми

В Python:

```python
True == 1
```

дает:

```python
True
```

Потому что исторически `bool` является наследником `int`.

То есть `True` ведет себя как `1`, а `False` ведет себя как `0`.

Также:

```python
1 == 1.0
```

дает:

```python
True
```

Потому что математически целое число `1` равно вещественному числу `1.0`.

## 6. Как Python строит словарь по шагам

Код:

```python
{True: 'yes', 1: 'no', 1.0: 'probably'}
```

создается слева направо.

### Шаг 1

Python видит первую пару:

```python
True: 'yes'
```

В словаре появляется:

```python
{True: 'yes'}
```

Ключ — `True`.

Значение — `'yes'`.

### Шаг 2

Python видит вторую пару:

```python
1: 'no'
```

Он проверяет: есть ли уже такой ключ?

Для этого Python смотрит:

```python
hash(1)
```

Получает:

```python
1
```

Но у `True` хэш тоже:

```python
1
```

Потом Python проверяет равенство:

```python
True == 1
```

Получает:

```python
True
```

Значит, для словаря это один и тот же ключ.

Поэтому новый ключ не добавляется.

Просто обновляется значение:

```python
{True: 'no'}
```

Обрати внимание: ключ остался `True`, но значение стало `'no'`.

### Шаг 3

Python видит третью пару:

```python
1.0: 'probably'
```

Он снова проверяет:

```python
hash(1.0)
```

Получает:

```python
1
```

И проверяет равенство:

```python
True == 1.0
```

Получает:

```python
True
```

Значит, это снова тот же самый ключ.

Python не добавляет новый ключ.

Он только обновляет значение:

```python
{True: 'probably'}
```

## 7. Почему ключ остался `True`, а не стал `1.0`

Это самый важный момент.

Когда словарь впервые создал ключ `True`, он сохранил именно объект `True` как ключ.

Потом `1` и `1.0` не создавали новый ключ, а только обновляли значение у уже существующего ключа.

Поэтому ключ остался первым:

```python
True
```

А значение стало последним:

```python
'probably'
```

## 8. Итог

Финальный результат:

```python
{True: 'probably'}
```

## 9. Как это запомнить

Правило:

Если в словаре несколько ключей равны друг другу и имеют одинаковый хэш, то:

- ключ остается от первого добавления;
- значение берется от последнего добавления.

В нашем случае:

```python
{True: 'yes', 1: 'no', 1.0: 'probably'}
```

первый ключ:

```python
True
```

последнее значение:

```python
'probably'
```

поэтому ответ:

```python
{True: 'probably'}
```
