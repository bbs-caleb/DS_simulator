# 17 урок — ML: MUTABLE WORD COUNT

## 1. Контекст задачи

Эта тема про одно из самых важных свойств объектов в Python — **изменяемость**.

В Python объект может быть:

- **изменяемым** — его содержимое можно менять прямо внутри того же объекта;
- **неизменяемым** — его нельзя изменить на месте, можно только создать новый объект.

## Бизнесовый контекст

В реальных ML/DS-задачах мы постоянно работаем со структурами данных:

- словарями с признаками;
- списками товаров;
- наборами пользователей;
- датафреймами;
- массивами NumPy;
- кэшами результатов;
- счетчиками слов, событий, кликов, заказов.

Например, в NLP-задаче нужно посчитать, сколько раз каждое слово встретилось в тексте.

Результат обычно удобно хранить в словаре:

```python
{
    "apple": 3,
    "banana": 1,
    "milk": 5
}
```

Но словарь — это **изменяемый объект**. Если неправильно использовать словари, особенно как значения по умолчанию в функциях, можно получить очень неприятные баги: данные из прошлого вызова функции неожиданно останутся в следующем вызове.

Для будущего DS/MLE это важно, потому что такие ошибки могут ломать:

- препроцессинг текста;
- подсчет статистик;
- feature engineering;
- кэширование;
- пайплайны обучения моделей;
- онлайн-сервисы с предсказаниями.

## Технический контекст

В этой части урока разбираются:

1. изменяемые и неизменяемые типы;
2. хэширование;
3. почему не все объекты могут быть ключами словаря;
4. почему `True`, `1` и `1.0` в словаре ведут себя неожиданно;
5. почему у них совпадает хэш;
6. почему ключ остается первым, а значение перезаписывается последним.

Главный пример из задания:

```python
{True: 'yes', 1: 'no', 1.0: 'probably'}
```
