# Causal Impact Analysis — обе задачи. Общий обзор

Два задания урока DA/ML по методу Causal Impact. Оба сданы.

| Задача | Категория | Метрика | Ответ | Статус |
|---|---|---|---|---|
| 1. Реклама приставок | `console_sales` | Cumulative Absolute Effect | **-350** | 100/100 |
| 2. Кешбэк SigmaBank | `power_tools_sales` | Relative Average Effect | **-0.03** | сдано (≥80) |

## Главные уроки, которые стоят обеих задач

### 1. Библиотека имеет значение

Задание требует **`causalimpact`** (порт jamalsenouci на statsmodels), НЕ
`tfcausalimpact`.

- `tfcausalimpact` — байесовская, сэмплирует, результат **не воспроизводим**
  (прыгал от −49 до −110 на одних данных). Из-за этого невозможно стабильно
  попасть в узкий диапазон эталона.
- `causalimpact` — MLE-порт, **детерминированный**. Требует `impact.run()` перед
  `summary()`. Именно он дал точные ответы.

Установка окружения:
```bash
pip install "pandas==2.2.3" "numpy>=2,<2.3" causalimpact==0.2.6
```
(в коде есть monkey-patch совместимости со свежим pandas — старый порт зовёт
удалённую функцию `is_datetime_or_timedelta_dtype`).

### 2. Отбор контрольных рядов — три критерия

Отбрасываем ряд, если он:
1. **слишком сильно** коррелирует с целью И затронут вмешательством напрямую
   (games для приставок);
2. имеет **обратную** сезонность (sport_goods; home_goods);
3. **не имеет** сезонности вообще — шум (books; office_supplies).

### 3. Знак эффекта задаёт относительное поведение контролей

Во 2-й задаче знак «перевернулся» в зависимости от набора: относительно самого
тесно связанного ряда (`hand_tools`) цель просела сильнее → эффект отрицательный.
Подсказка грейдера «more related» указывала именно на это.

### 4. Off-by-one в индексе вмешательства

«N-я неделя» (счёт с 1) = строка с индексом **N−1** (счёт с 0 в Python).
Задача 1: 84 → индекс 83. Задача 2: 96 → индекс 95.

## Как воспроизвести

1. Поставь окружение (см. выше).
2. Положи рядом нужный CSV (`console_campaing.csv` / `power_tools_promo.csv`).
3. Запусти `python console_causal_impact.py` или `python sigmabank_causal_impact.py`.
4. Число из вывода = ответ (в solution-файлах оно уже зафиксировано).
