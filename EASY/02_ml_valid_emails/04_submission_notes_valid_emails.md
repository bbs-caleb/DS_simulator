# Этап 4. Что сдавать в систему проверки

## Основной файл

Сдавать нужно файл:

```text
valid_emails.py
```

Внутри должен быть такой код:

```python
import re
from typing import List


VALID_EMAIL_REGEX = re.compile(
    r"^[a-zA-Z0-9_.+-]+@[a-zA-Z0-9-]+\.[a-zA-Z]{2,}$"
)


def valid_emails(strings: List[str]) -> List[str]:
    """Take list of potential emails and return only valid ones."""
    return [email for email in strings if VALID_EMAIL_REGEX.fullmatch(email)]
```

---

## Почему не нужно менять регулярное выражение

В условии сказано, что исходный код уже правильно валидирует email-адреса.

Поэтому рискованно менять само регулярное выражение.

Если изменить правило валидации и начать принимать или отклонять другие email, можно получить 0 баллов за корректность.

Здесь мы сохраняем исходный regex полностью:

```python
r"^[a-zA-Z0-9_.+-]+@[a-zA-Z0-9-]+\.[a-zA-Z]{2,}$"
```

Меняется только способ его использования.

---

## Что даёт баллы

### Скорость

Основное ускорение даёт эта строка:

```python
VALID_EMAIL_REGEX = re.compile(...)
```

и использование:

```python
VALID_EMAIL_REGEX.fullmatch(email)
```

### Качество кода

Код соответствует базовым требованиям PEP8:

1. импорты сверху;
2. константа названа большими буквами;
3. между импортами, константой и функцией есть пустые строки;
4. функция короткая и понятная;
5. есть docstring;
6. нет лишней вложенной функции.

---

## Короткий вывод

Это решение должно пройти проверку, потому что оно:

1. возвращает те же валидные email, что и исходный код;
2. работает быстрее за счёт `re.compile()`;
3. написано чисто и просто;
4. не усложняет задачу лишней логикой.
