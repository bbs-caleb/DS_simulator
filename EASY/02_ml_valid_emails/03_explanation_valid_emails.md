# Этап 3. Пошаговое объяснение кода с нуля

## Полный код

```python
import re
from typing import List


VALID_EMAIL_REGEX = re.compile(
    r"^[a-zA-Z0-9_.+-]+@[a-zA-Z0-9-]+\.[a-zA-Z]{2,}$"
)


def valid_emails(strings: List[str]) -> List[str]:
    """Take list of potential emails and return only valid ones."""
    return [email for email in strings if VALID_EMAIL_REGEX.fullmatch(email)]
```

---

## Шаг 1. Импортируем модуль `re`

```python
import re
```

`re` — это стандартный модуль Python для работы с регулярными выражениями.

Регулярное выражение — это правило, по которому мы проверяем текст.

В этой задаче мы проверяем, похожа ли строка на email.

---

## Шаг 2. Импортируем `List`

```python
from typing import List
```

`List` нужен только для подсказки типов.

Вот эта запись:

```python
def valid_emails(strings: List[str]) -> List[str]:
```

означает:

- на вход функция получает список строк;
- на выход функция возвращает список строк.

Пример:

```python
["user@mail.com", "bad_email", "admin@test.org"]
```

Это список строк.

---

## Шаг 3. Создаём скомпилированное регулярное выражение

```python
VALID_EMAIL_REGEX = re.compile(
    r"^[a-zA-Z0-9_.+-]+@[a-zA-Z0-9-]+\.[a-zA-Z]{2,}$"
)
```

Здесь создаётся переменная `VALID_EMAIL_REGEX`.

Она хранит не обычную строку, а заранее подготовленный объект регулярного выражения.

Почему название большими буквами:

```python
VALID_EMAIL_REGEX
```

Потому что это константа.

Константа — это значение, которое мы создали один раз и дальше не меняем.

Так принято писать по PEP8.

---

## Шаг 4. Разбираем регулярное выражение

Регулярное выражение выглядит так:

```python
r"^[a-zA-Z0-9_.+-]+@[a-zA-Z0-9-]+\.[a-zA-Z]{2,}$"
```

Разберём его по частям.

### `^`

```python
^
```

Означает начало строки.

То есть проверка должна начинаться с самого первого символа.

---

### `[a-zA-Z0-9_.+-]+`

```python
[a-zA-Z0-9_.+-]+
```

Это первая часть email до символа `@`.

Она разрешает:

- маленькие латинские буквы: `a-z`;
- большие латинские буквы: `A-Z`;
- цифры: `0-9`;
- символ `_`;
- точку `.`;
- плюс `+`;
- минус `-`.

Знак `+` после квадратных скобок означает: один или больше таких символов.

Примеры подходящих первых частей:

```text
user
john.doe
user_123
test+promo
```

---

### `@`

```python
@
```

Означает, что в email обязательно должен быть символ `@`.

Пример:

```text
user@mail.com
```

---

### `[a-zA-Z0-9-]+`

```python
[a-zA-Z0-9-]+
```

Это домен после `@`, но до точки.

Например в email:

```text
user@gmail.com
```

часть `gmail` проверяется именно этим фрагментом.

Разрешены:

- латинские буквы;
- цифры;
- дефис `-`.

---

### `\.`

```python
\.
```

Означает обычную точку.

Почему не просто `.`?

Потому что в регулярных выражениях точка — специальный символ. Она означает почти любой символ.

Чтобы сказать Python: «мне нужна именно точка», мы экранируем её через обратный слэш:

```python
\.
```

---

### `[a-zA-Z]{2,}`

```python
[a-zA-Z]{2,}
```

Это доменная зона.

Например:

```text
com
ru
org
uz
```

`[a-zA-Z]` означает латинскую букву.

`{2,}` означает минимум 2 символа.

То есть доменная зона должна быть длиной хотя бы 2 буквы.

---

### `$`

```python
$
```

Означает конец строки.

То есть после доменной зоны не должно быть лишних символов.

---

## Шаг 5. Объявляем функцию

```python
def valid_emails(strings: List[str]) -> List[str]:
```

Здесь мы создаём функцию `valid_emails`.

Она принимает один аргумент:

```python
strings
```

`strings` — это список строк, где каждая строка может быть email, а может быть мусором.

Пример входа:

```python
[
    "user@mail.com",
    "not_email",
    "admin@test.org",
]
```

---

## Шаг 6. Докстринг

```python
"""Take list of potential emails and return only valid ones."""
```

Это короткое описание функции.

Оно говорит: функция получает список потенциальных email и возвращает только валидные.

---

## Шаг 7. Возвращаем только валидные email

```python
return [email for email in strings if VALID_EMAIL_REGEX.fullmatch(email)]
```

Это list comprehension.

Проще говоря, это короткая запись цикла.

Обычный вариант выглядел бы так:

```python
emails = []

for email in strings:
    if VALID_EMAIL_REGEX.fullmatch(email):
        emails.append(email)

return emails
```

А короткий вариант:

```python
return [email for email in strings if VALID_EMAIL_REGEX.fullmatch(email)]
```

Оба варианта делают одно и то же.

---

## Шаг 8. Что делает `.fullmatch()`

```python
VALID_EMAIL_REGEX.fullmatch(email)
```

Метод `.fullmatch()` проверяет, что вся строка полностью подходит под регулярное выражение.

Например:

```python
VALID_EMAIL_REGEX.fullmatch("user@mail.com")
```

Вернёт объект `Match`, потому что строка валидная.

А вот так:

```python
VALID_EMAIL_REGEX.fullmatch("not_email")
```

вернёт `None`, потому что строка не похожа на email.

---

## Шаг 9. Почему решение быстрее

В старом коде проверка была такой:

```python
re.fullmatch(valid_email_regex, email)
```

То есть на каждой проверке мы обращались к функции модуля `re` и передавали туда строку с регулярным выражением.

В новом коде регулярное выражение подготовлено заранее:

```python
VALID_EMAIL_REGEX = re.compile(...)
```

А внутри проверки используется уже готовый объект:

```python
VALID_EMAIL_REGEX.fullmatch(email)
```

Это быстрее, потому что Python не должен каждый раз заново проходить весь путь работы с обычной строкой-паттерном.

## Шаг 10. Проверка на примере

```python
emails = [
    "good@mail.com",
    "bad_email",
    "user.name@test.org",
    "wrong@site",
    "admin@test.co",
]

print(valid_emails(emails))
```

Результат:

```python
[
    "good@mail.com",
    "user.name@test.org",
    "admin@test.co",
]
```

Почему остальные не прошли:

```text
bad_email
```

Нет символа `@` и домена.

```text
wrong@site
```

Нет точки и доменной зоны после точки.

---

## Финальная идея

Мы не меняли бизнес-логику и не меняли правило проверки email.

Мы только сделали проверку быстрее:

1. вынесли регулярное выражение в константу;
2. заранее скомпилировали его через `re.compile()`;
3. используем метод `.fullmatch()` у готового regex-объекта;
4. вернули результат через понятный list comprehension.
