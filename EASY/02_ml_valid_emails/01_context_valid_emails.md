# Этап 1. Контекст задачи: VALID EMAILS

## 1. Бизнесовый контекст

Компания разрабатывает сервис, который должен автоматически отлавливать ботов и помогать защищаться от DDoS-атак.

Один из первых фильтров при регистрации пользователя — проверка email-адреса.

Зачем это нужно бизнесу:

1. **Меньше мусорных регистраций**  
   Если бот отправляет случайные строки вместо email, сервис может сразу отсеять часть таких заявок.

2. **Меньше нагрузки на систему**  
   Во время DDoS или массовой бот-атаки важно быстро обрабатывать входящие данные. Даже маленькая функция валидации email может вызываться тысячи или миллионы раз.

3. **Лучшее качество данных**  
   В базе не будут копиться явно некорректные email-адреса.

4. **Меньше ошибок дальше по продукту**  
   Если email нужен для подтверждения аккаунта, уведомлений или восстановления пароля, некорректный email ломает дальнейшую коммуникацию с пользователем.

Главная бизнес-идея: функция должна не просто правильно проверять email, но и делать это быстро, потому что она стоит на входе в систему регистрации.

---

## 2. Технический контекст

У нас есть функция:

```python
import re
from typing import List


def valid_emails(strings: List[str]) -> List[str]:
    """Take list of potential emails and returns only valid ones"""

    valid_email_regex = r"^[a-zA-Z0-9_.+-]+@[a-zA-Z0-9-]+\.[a-zA-Z]{2,}$"

    def is_valid_email(email: str) -> bool:
        return bool(re.fullmatch(valid_email_regex, email))

    emails = []
    for email in strings:
        if is_valid_email(email):
            emails.append(email)

    return emails
```

Она делает правильную проверку, но работает медленнее, чем могла бы.

Причина в этой строке:

```python
re.fullmatch(valid_email_regex, email)
```

Здесь мы каждый раз вызываем `re.fullmatch` через модуль `re` и передаём туда строку с регулярным выражением.

Python умеет кешировать регулярные выражения, но при массовой проверке большого списка email быстрее заранее скомпилировать регулярное выражение через `re.compile()` и потом использовать уже готовый объект.

---

## 3. Что такое компиляция регулярного выражения

Регулярное выражение — это строка с правилом поиска.

Например:

```python
r"^[a-zA-Z0-9_.+-]+@[a-zA-Z0-9-]+\.[a-zA-Z]{2,}$"
```

Но компьютеру неудобно каждый раз работать именно со строкой. Ему нужно разобрать это правило и подготовить его к проверке.

`re.compile()` делает эту подготовку заранее:

```python
pattern = re.compile(r"...")
```

После этого можно быстро проверять строки так:

```python
pattern.fullmatch(email)
```

То есть вместо того, чтобы каждый раз обращаться к модулю `re`, мы используем уже готовый скомпилированный объект.

---

## 4. Что нужно изменить

Нужно поменять основную идею работы с regex:

Было:

```python
valid_email_regex = r"^[a-zA-Z0-9_.+-]+@[a-zA-Z0-9-]+\.[a-zA-Z]{2,}$"
re.fullmatch(valid_email_regex, email)
```

Стало:

```python
VALID_EMAIL_REGEX = re.compile(
    r"^[a-zA-Z0-9_.+-]+@[a-zA-Z0-9-]+\.[a-zA-Z]{2,}$"
)
VALID_EMAIL_REGEX.fullmatch(email)
```

Сама логика проверки email не меняется. Меняется только способ использования регулярного выражения.

---

## 5. Почему это решение подходит на 100 баллов

Решение сохраняет исходное правило валидации email, поэтому функция должна возвращать те же корректные email-адреса, что и старая версия.

При этом оно быстрее, потому что:

1. регулярное выражение компилируется один раз;
2. внутри цикла используется готовый regex-объект;
3. код становится короче за счёт list comprehension;
4. код соответствует PEP8: константа написана заглавными буквами, есть нормальные отступы, читаемая структура и докстринг.
