## Короткая суть задания

Есть два списка одинаковой длины:

```python
labels = [1, 0, 1, 0]
scores = [0.95, 0.80, 0.60, 0.10]
```

`labels` содержит историческую информацию о релевантности:

- `1` — изображение релевантно или по нему был положительный сигнал;
- `0` — изображение нерелевантно.

`scores` содержит оценки новой модели:

- чем больше score, тем выше модель хочет поставить изображение.

Для вычисления метрики нужно:

1. Отсортировать изображения по `scores` по убыванию.
2. Взять первые `K` изображений.
3. Считать их предсказанным положительным классом.
4. Все остальные считать предсказанным отрицательным классом.
5. Построить TP, FP, TN и FN.
6. Вычислить четыре метрики.

---

## Формулы

```text
Recall@K = TP / (TP + FN)
```

Какая доля всех релевантных объектов попала в top-K.

```text
Precision@K = TP / (TP + FP)
```

Какая доля объектов внутри top-K действительно релевантна.

```text
Specificity@K = TN / (TN + FP)
```

Какая доля нерелевантных объектов была правильно оставлена за пределами top-K.

```text
F1@K = 2 × TP / (2 × TP + FP + FN)
```

Баланс Precision@K и Recall@K.

---

## Важное различие

`score` не является готовым ответом `0` или `1`.

Модель только говорит:

> «Этот объект кажется мне более релевантным, чем другой».

Решение о том, какие объекты считать положительным предсказанием, принимает правило top-K.

---

## Проверочный пример

```python
labels = [1, 0, 1, 0, 1, 0, 0, 1]
scores = [0.95, 0.85, 0.80, 0.75, 0.70, 0.60, 0.40, 0.20]
k = 5
```

В top-5 окажутся метки:

```text
[1, 0, 1, 0, 1]
```

Получаем:

```text
TP = 3
FP = 2
FN = 1
TN = 2
```

Следовательно:

```text
Recall@5 = 3 / 4 = 0.75
Precision@5 = 3 / 5 = 0.60
Specificity@5 = 2 / 4 = 0.50
F1@5 = 2 × 3 / (2 × 3 + 2 + 1) = 6 / 9 ≈ 0.6667
```

## Практический смысл урока

Это не просто учебная реализация формул. Такой код является маленькой частью реального offline evaluation pipeline для:

- поисковой системы;
- рекомендательной системы;
- ранжирования товаров;
- подбора кандидатов;
- антифрод-модели с ручной проверкой top-K операций;
- приоритизации лидов;
- модерации контента;
- поиска похожих документов;
- retrieval-части RAG-системы.

В промышленной системе одна такая функция обычно применяется к миллионам запросов, после чего метрики агрегируются по пользователям, сегментам, странам, типам запросов и временным периодам.
