## Важное уточнение к формулировке урока

Учебное описание упрощает MCP до «единой спецификации API в JSON/YAML». Для первого знакомства это может быть полезной аналогией, но технически точнее:

> MCP — открытый протокол взаимодействия AI-приложений с внешними источниками данных, инструментами и workflow. Он использует JSON-RPC 2.0, определяет жизненный цикл соединения и позволяет сторонам согласовывать поддерживаемые возможности.

MCP не следует путать с OpenAPI:

- OpenAPI документирует HTTP API;
- MCP описывает протокольное взаимодействие AI-host/client/server;
- MCP может внутри инструмента вызывать обычный REST API;
- MCP-сервер часто является адаптером между AI-приложением и существующими системами.

## Актуальность

Материалы подготовлены по состоянию на **12 июля 2026 года** с опорой на официальную спецификацию MCP `2025-11-25`, архитектурную документацию, Python SDK, Security Best Practices и MCP Registry.

## Главная мысль

**MCP не делает модель умнее сам по себе. Он делает доступ к возможностям внешнего мира стандартизированным, управляемым и переиспользуемым.**

Качество системы всё равно зависит от корректности инструментов, схем, прав доступа, бизнес-ограничений, подтверждений пользователя, наблюдаемости и тестирования.
