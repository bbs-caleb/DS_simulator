# Страница 4. Визуальный разбор матрицы Вагнера–Фишера

## 1. Что означает каждая ячейка

Главная формула понимания:

```text
D[i][j]
```

— это не сравнение двух отдельных символов.

Это минимальное расстояние между префиксами:

```python
text[:i]
ocr_text[:j]
```

Например:

```text
D[2][3]
```

для строк:

```text
phony
money
```

означает расстояние между:

```text
"ph"
"mon"
```

---

## 2. Пример `phony → money`

Строки:

```text
phony
money
```

Обе имеют длину `5`.

С пустыми префиксами получается матрица `6 × 6`.

Заголовки:

```text
      ""  m  o  n  e  y
"" 
p
h
o
n
y
```

---

## 3. Границы матрицы

Первая строка:

```text
0 1 2 3 4 5
```

Она показывает, сколько вставок нужно, чтобы получить префикс `money`
из пустой строки.

Первый столбец:

```text
0
1
2
3
4
5
```

Он показывает, сколько удалений нужно, чтобы превратить префикс `phony`
в пустую строку.

После инициализации:

```text
      ""  m  o  n  e  y
""     0  1  2  3  4  5
p      1
h      2
o      3
n      4
y      5
```

## 4. Ячейка `p` против `m`

Сравниваются:

```text
"p"
"m"
```

### Вставка

Ячейка слева:

```text
1
```

Плюс одна операция:

```text
2
```

### Удаление

Ячейка сверху:

```text
1
```

Плюс одна операция:

```text
2
```

### Замена

Диагональная ячейка:

```text
0
```

`p` и `m` различаются, поэтому:

```text
0 + 1 = 1
```

Минимум:

```text
1
```

---

## 5. Ячейка `p` против `mo`

Сравниваются префиксы:

```text
"p"
"mo"
```

Стоимость вставки:

```text
1 + 1 = 2
```

Стоимость удаления:

```text
2 + 1 = 3
```

Стоимость замены:

```text
1 + 1 = 2
```

Минимум:

```text
2
```

---

## 6. Полная матрица

```text
      ""  m  o  n  e  y
""     0  1  2  3  4  5
p      1  1  2  3  4  5
h      2  2  2  3  4  5
o      3  3  2  3  4  5
n      4  4  3  2  3  4
y      5  5  4  3  3  3
```

Правый нижний элемент:

```text
3
```

Значит, минимально нужны три редакторские операции.

---

## 7. Один из возможных путей

```text
phony
mony
money
```

Один из вариантов:

1. заменить `p` на `m`;
2. удалить `h`;
3. вставить `e`.

Расстояние равно `3`.

---

## 8. Как направление связано с операцией

Если восстанавливать путь по матрице:

- движение влево соответствует вставке;
- движение вверх соответствует удалению;
- движение по диагонали соответствует совпадению или замене.

В текущем задании путь восстанавливать не нужно.

Нужно только значение расстояния.

## 9. Мини-пример полного совпадения

```text
a → a
```

Матрица:

```text
      ""  a
""     0  1
a      1  0
```

Правый нижний элемент `0`.

---

## 10. Мини-пример замены

```text
a → b
```

Матрица:

```text
      ""  b
""     0  1
a      1  1
```

Нужна одна замена.

---

## 11. Мини-пример вставки

```text
a → ab
```

Матрица:

```text
      ""  a  b
""     0  1  2
a      1  0  1
```

Нужно вставить `b`.

---

## 12. Мини-пример удаления

```text
ab → a
```

Матрица:

```text
      ""  a
""     0  1
a      1  0
b      2  1
```

Нужно удалить `b`.

---

## 13. Почему алгоритм работает

Каждая большая задача зависит только от трёх уже решённых меньших задач:

```text
слева
сверху
по диагонали
```

Алгоритм не перебирает все возможные последовательности операций.

Он один раз вычисляет каждую пару префиксов и сохраняет результат.

Именно поэтому динамическое программирование превращает потенциально очень
дорогой перебор в алгоритм со сложностью:

\[
O(nm)
\]

---

## 14. Главная мысль

Каждая ячейка отвечает на маленький вопрос:

> Сколько минимум операций нужно, чтобы преобразовать первые `i` символов
> первой строки в первые `j` символов второй?

Когда заполнены все маленькие ответы, правый нижний элемент автоматически
отвечает на вопрос для полных строк.
