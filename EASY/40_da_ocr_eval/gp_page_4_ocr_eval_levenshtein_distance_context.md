# Страница 4. OCR Eval: Levenshtein Distance

## 1. Бизнес-контекст

Smart Tutor принимает фотографию или скриншот задания, извлекает текст через OCR,
а затем передаёт этот текст следующему компоненту — диалоговому помощнику.

Полная цепочка выглядит так:

```text
изображение
→ OCR
→ распознанный текст
→ понимание задания
→ решение
→ ответ ученику
```

Ошибка на OCR-этапе распространяется дальше. Например:

```text
5x + 3 = 18
```

может превратиться в:

```text
Sx + 3 = 18
```

А:

```text
9.8 м/с²
```

может быть распознано как:

```text
98 м/с²
```

Формально изменилась одна позиция, но смысл задачи изменился очень сильно.

Поэтому команде нужен инструмент, который не просто говорит, похожи ли тексты,
а оценивает минимальное число конкретных исправлений.

---

## 2. Почему предыдущие подходы недостаточны

### Прямое сравнение символов

Наивный алгоритм сравнивает:

```python
text[i] != ocr_text[i]
```

Если один символ пропущен, все следующие символы сдвигаются.

Пример:

```text
Эталон: abcdef
OCR:    abdef
```

Реально произошла одна ошибка — удаление `c`.

Но прямое сравнение после пропуска теряет синхронизацию и начинает считать
несовпадающими несколько следующих позиций.

### Jaccard Similarity

Jaccard сравнивает множества слов. Он устойчивее к сдвигам, но теряет:

- порядок слов;
- повторения;
- внутреннюю похожесть слов;
- пунктуационную структуру.

Например:

```text
home work
work home
```

получают нулевую ошибку.

То же самое происходит для:

```text
home work home
work home
```

потому что множества уникальных слов одинаковы.

## 3. Что такое расстояние Левенштейна

Расстояние Левенштейна — минимальное число редакторских операций,
необходимых для преобразования одной строки в другую.

Разрешены три операции:

1. вставка символа;
2. удаление символа;
3. замена символа.

Примеры:

```text
cat → cut
```

Одна замена, расстояние равно `1`.

```text
cat → cart
```

Одна вставка, расстояние равно `1`.

```text
house → hose
```

Одно удаление, расстояние равно `1`.

Если строки совпадают:

```text
hello → hello
```

расстояние равно `0`.

---

## 4. Почему эта метрика лучше подходит для OCR

Для примера:

```text
abcdef → abdef
```

алгоритм находит минимальное преобразование:

```text
удалить c
```

И возвращает:

```text
1
```

В отличие от прямого сравнения, вставка или удаление не приводит к ошибочной
оценке всех последующих символов.

---

## 5. Алгоритм Вагнера–Фишера

Расстояние Левенштейна вычисляется через динамическое программирование.

Создаётся матрица:

```text
(len(text) + 1) × (len(ocr_text) + 1)
```

Каждая ячейка:

```text
D[i][j]
```

хранит расстояние между префиксами:

```python
text[:i]
ocr_text[:j]
```

### Базовые случаи

```text
D[0][0] = 0
```

Две пустые строки совпадают.

```text
D[i][0] = i
```

Чтобы превратить строку длины `i` в пустую, нужно `i` удалений.

```text
D[0][j] = j
```

Чтобы превратить пустую строку в строку длины `j`, нужно `j` вставок.

### Рекуррентная формула

Для каждой внутренней ячейки берётся минимум:

\[
D(i,j)=\min
\begin{cases}
D(i,j-1)+1 \\
D(i-1,j)+1 \\
D(i-1,j-1)+cost
\end{cases}
\]

где:

```text
cost = 0, если символы равны
cost = 1, если символы различаются
```

Три варианта означают:

- слева + 1 — вставка;
- сверху + 1 — удаление;
- по диагонали + 0/1 — совпадение или замена.

---

## 6. Динамическое программирование простыми словами

Большая задача разбивается на маленькие:

> Как преобразовать первые `i` символов первой строки
> в первые `j` символов второй строки?

Каждый ответ сохраняется в матрице.

Когда алгоритм доходит до правого нижнего угла, там находится ответ
для полных строк.

## 7. Бизнес-применение

### EdTech

OCR распознаёт:

- условия задач;
- формулы;
- учебники;
- рукописные ответы;
- тесты.

Особенно критичны:

- числа;
- знаки операций;
- степени;
- десятичные разделители;
- единицы измерения;
- отрицания.

### Финтех

OCR используется для:

- паспортов;
- справок;
- выписок;
- договоров;
- реквизитов;
- чеков.

Для номера счёта одна ошибка может быть критичнее десятка ошибок в описании.
Поэтому Levenshtein Distance полезен как baseline, но критические поля требуют
отдельной точной проверки.

### E-commerce и маркетплейсы

OCR извлекает:

- названия товаров;
- бренды;
- цены;
- объёмы;
- состав;
- серийные номера;
- маркировку.

Ошибка:

```text
500 мл → 5000 мл
```

имеет небольшое расстояние, но большое бизнес-влияние.

### iGaming, crypto, dating

OCR может использоваться в KYC, проверке возраста, документах и антифроде.

Для криптокошелька один неверный символ означает другой адрес, поэтому
нужны дополнительные проверки формата и контрольных сумм.

### Big Tech и промышленность

В системах уровня Google, Meta, Anthropic, SpaceX или BP OCR может читать:

- технические схемы;
- приборные панели;
- маркировки;
- журналы;
- инструкции;
- документы.

Одна ошибка в давлении, номере детали или коде может быть опаснее десятка
ошибок в обычном абзаце.

## 8. Ограничения обычного Levenshtein Distance

### Зависимость от длины

Два текста могут иметь расстояние `2`.

Но:

```text
2 ошибки в 20 символах
```

и:

```text
2 ошибки в 200 символах
```

не одинаковы по качеству.

Поэтому на следующем шаге расстояние обычно нормируют на длину строки.

### Все операции имеют одинаковый вес

Обычная версия считает одинаковыми:

```text
а → о
```

и:

```text
O → 0
```

Но для конкретной OCR-системы путаница `O/0` может быть особенно важна.

### Нет понимания смысла

```text
да → нет
```

имеет небольшое расстояние, но противоположный смысл.

### Нет понимания структуры документа

Ошибка в поясняющем тексте и ошибка в формуле получают одинаковый вес.

## 9. Как рассуждать на продуктовом интервью

Перед выбором OCR-метрики нужно спросить:

1. Что распознаётся: текст, формула, таблица, документ или номер?
2. Какова единица оценки: символ, слово, поле или документ?
3. Какие ошибки критичны для бизнеса?
4. Нужна ли нормализация по длине?
5. Все ли замены должны иметь одинаковую стоимость?
6. Как качество отличается по языкам, фонам и устройствам?
7. Что делать при низкой уверенности?
8. Как offline-метрика связана с успешным ответом пользователю?

Хороший production-набор метрик часто включает:

- Levenshtein Distance;
- Character Error Rate;
- Word Error Rate;
- exact match;
- field-level accuracy;
- weighted error rate;
- downstream task success.

## 10. Итог

Levenshtein Distance решает главный недостаток прямого сравнения:
вставки и удаления больше не искажают оценку всего оставшегося текста.

Алгоритм:

1. создаёт матрицу;
2. заполняет первую строку и столбец;
3. рассматривает вставку, удаление и замену;
4. выбирает минимальную стоимость;
5. возвращает правый нижний элемент.

Это фундаментальная метрика для OCR, ASR, проверки орфографии,
нечёткого поиска и сравнения строк.
