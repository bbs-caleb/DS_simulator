# Пошаговое объяснение решения с абсолютного нуля

## 1. Что такое класс

Класс — это описание объекта и его поведения.

В задании нужен класс:

```python
class BaseEval:
```

Из него создаётся объект:

```python
ocr_eval = BaseEval("data.json")
```

В момент создания объекта Python автоматически вызывает метод `__init__`.

---

## 2. Импорты

```python
import json
import os
from typing import List, Tuple
```

Разберём каждый импорт.

### `json`

Модуль `json` нужен для чтения JSON-файла.

JSON — текстовый формат хранения структурированных данных.

Пример:

```json
[
    {
        "text": "Привет",
        "ocr_text": "Привeт"
    }
]
```

Функция:

```python
json.load(file)
```

читает JSON из открытого файла и преобразует его в Python-объекты.

В данном случае:

- JSON-массив превращается в `list`;
- JSON-объект превращается в `dict`;
- JSON-строка превращается в `str`.

### `os`

Модуль `os` используется для проверки файла:

```python
os.path.isfile(path)
```

Функция возвращает:

- `True`, если по пути существует обычный файл;
- `False`, если файла нет или путь ведёт не к файлу.

### `List` и `Tuple`

Это типы для аннотаций.

```python
List[Tuple[str, str]]
```

означает:

> список, в котором каждый элемент является кортежем из двух строк.

Пример подходящего значения:

```python
[
    ("hello", "he11o"),
    ("world", "wor1d"),
]
```

Аннотации помогают:

- читать код;
- понимать контракт функции;
- находить ошибки в IDE;
- проходить статические проверки;
- поддерживать проект командой.

Они сами по себе не запрещают Python передать другое значение во время выполнения.

## 3. Конструктор `__init__`

```python
class BaseEval:
    def __init__(self, documents_path: str):
        self.documents = self.load_documents(documents_path)

        if not self.documents:
            raise ValueError("The documents list is empty.")
```

### Что такое `self`

`self` — ссылка на конкретный созданный объект.

Когда мы пишем:

```python
ocr_eval = BaseEval("data.json")
```

внутри `__init__` переменная `self` указывает на объект `ocr_eval`.

### Что делает эта строка

```python
self.documents = self.load_documents(documents_path)
```

Справа вызывается метод загрузки:

```python
self.load_documents(documents_path)
```

Он возвращает список документов.

Слева результат сохраняется в атрибут объекта:

```python
self.documents
```

После создания объекта можно получить данные:

```python
print(ocr_eval.documents)
```

### Проверка пустого списка

```python
if not self.documents:
```

Пустой список в логическом контексте считается `False`:

```python
bool([])  # False
```

Непустой список считается `True`:

```python
bool([1])  # True
```

Поэтому конструкция означает:

> если список документов пустой.

Тогда выбрасывается исключение:

```python
raise ValueError("The documents list is empty.")
```

`raise` останавливает обычное выполнение и сообщает вызывающему коду об ошибке.

## 4. Метод `load_documents`

Сигнатура:

```python
def load_documents(self, path: str) -> List[Tuple[str, str]]:
```

Она сообщает:

- метод получает `path` — строку с путём;
- метод возвращает список кортежей из двух строк.

## 5. Проверка существования файла

```python
if not os.path.isfile(path):
    raise FileNotFoundError(f"File does not exist: {path}")
```

### Почему не нужно сразу вызывать `open`

`open` и сам выбросит `FileNotFoundError`, если файла нет. Однако явная проверка делает поведение понятным и позволяет контролировать сообщение.

### Что такое f-строка

```python
f"File does not exist: {path}"
```

Значение переменной `path` вставляется внутрь строки.

Если:

```python
path = "missing.json"
```

результат будет:

```text
File does not exist: missing.json
```

---

## 6. Открытие файла

```python
with open(path, "r", encoding="utf-8") as file:
    documents = json.load(file)
```

### Режим `"r"`

Буква `r` означает read — открыть для чтения.

### Почему `encoding="utf-8"`

В данных есть русский текст. Явное указание UTF-8 делает чтение предсказуемым на разных операционных системах.

Без явной кодировки код может работать на одном компьютере и падать на другом.

### Что делает `with`

`with` создаёт контекстный менеджер.

После выхода из блока файл автоматически закрывается, даже если во время чтения возникнет ошибка.

Это безопаснее, чем:

```python
file = open(path)
documents = json.load(file)
file.close()
```

Во втором варианте при исключении строка `file.close()` может не выполниться.

## 7. Обработка некорректного JSON

```python
try:
    with open(path, "r", encoding="utf-8") as file:
        documents = json.load(file)
except json.JSONDecodeError as error:
    raise ValueError("The file does not contain valid JSON.") from error
```

### Что такое `try`

В блок `try` помещают код, который потенциально может завершиться исключением.

### Что такое `except`

`except` перехватывает конкретный тип ошибки.

Если файл содержит, например:

```text
это не JSON
```

`json.load` выбросит:

```python
json.JSONDecodeError
```

Мы преобразуем его в `ValueError`, поскольку именно такой тип ошибки указан в контракте метода.

### Что означает `from error`

```python
raise ValueError(...) from error
```

Это сохраняет цепочку причин:

- внешняя понятная ошибка — `ValueError`;
- исходная техническая причина — `JSONDecodeError`.

Такой подход полезен при отладке.

## 8. Проверка корневой структуры JSON

```python
if not isinstance(documents, list):
    raise ValueError("The JSON file must contain a list of documents.")
```

### Что делает `isinstance`

```python
isinstance(value, list)
```

проверяет, является ли `value` списком.

Корректный файл:

```json
[
    {"text": "a", "ocr_text": "b"}
]
```

Некорректный для этого задания файл:

```json
{
    "text": "a",
    "ocr_text": "b"
}
```

Второй JSON синтаксически валиден, но содержит один словарь вместо списка документов.

Это две разные проверки:

1. является ли содержимое валидным JSON;
2. соответствует ли структура требованиям задачи.

## 9. Валидация каждого документа

```python
return [self.validate_document(document) for document in documents]
```

Это list comprehension — генерация списка.

Эквивалентный развёрнутый код:

```python
validated_documents = []

for document in documents:
    validated_document = self.validate_document(document)
    validated_documents.append(validated_document)

return validated_documents
```

Короткая версия делает то же самое.

Каждый словарь преобразуется в кортеж:

```python
(text, ocr_text)
```

## 11. Проверка типа документа

```python
if not isinstance(doc, dict):
    raise ValueError("Each document must be a dictionary.")
```

Корректный элемент:

```python
{
    "text": "Привет",
    "ocr_text": "Привeт",
}
```

Некорректные элементы:

```python
"Привет"
123
["Привет", "Привeт"]
None
```

Даже если список JSON существует, каждый его элемент должен иметь ожидаемую структуру.

## 12. Проверка обязательных ключей

```python
if "text" not in doc:
    raise ValueError("The document does not contain the 'text' field.")
```

Оператор `in` для словаря проверяет наличие ключа.

```python
"text" in {"text": "hello"}  # True
```

Далее аналогично проверяется `ocr_text`:

```python
if "ocr_text" not in doc:
    raise ValueError(
        "The document does not contain the 'ocr_text' field."
    )
```

Строка разделена на несколько строк кода для соблюдения ограничения длины строки и читаемости.

## 13. Возврат кортежа

```python
return doc["text"], doc["ocr_text"]
```

Python автоматически создаёт кортеж.

Это равнозначно:

```python
return (doc["text"], doc["ocr_text"])
```

Порядок критически важен:

```python
(text, ocr_text)
```

Нельзя поменять местами, потому что следующие методы ожидают именно такой контракт.

## 14. Метод `_eval_func`

```python
def _eval_func(self, text: str, ocr_text: str) -> float:
```

Подчёркивание в начале имени:

```python
_eval_func
```

означает внутренний метод класса по соглашению Python.

Это не строгая защита. Метод всё ещё можно вызвать напрямую:

```python
ocr_eval._eval_func("abc", "adc")
```

Но имя подсказывает разработчику, что метод предназначен прежде всего для внутреннего использования.

## 15. Как работает `zip`

```python
zip(text, ocr_text)
```

Создаёт пары элементов с одинаковыми индексами.

Пример:

```python
text = "cat"
ocr_text = "cut"
```

Пары:

```python
("c", "c")
("a", "u")
("t", "t")
```

В цикле:

```python
for source_char, ocr_char in zip(text, ocr_text):
```

на каждой итерации две переменные получают очередную пару символов.

---

## 16. Сравнение символов

```python
source_char != ocr_char
```

Оператор `!=` означает «не равно».

Он возвращает логическое значение:

```python
"a" != "a"  # False
"a" != "b"  # True
```

В Python логические значения участвуют в арифметике:

```python
int(False)  # 0
int(True)   # 1
```

Поэтому функция `sum` может посчитать количество `True`.

## 17. Подсчёт несовпадений

```python
different_characters = sum(
    source_char != ocr_char
    for source_char, ocr_char in zip(text, ocr_text)
)
```

Разберём изнутри наружу.

### Шаг 1

```python
zip(text, ocr_text)
```

создаёт пары символов.

### Шаг 2

```python
for source_char, ocr_char in ...
```

перебирает пары.

### Шаг 3

```python
source_char != ocr_char
```

для каждой пары создаёт `True` или `False`.

### Шаг 4

```python
sum(...)
```

складывает единицы и нули.

Пример:

```python
text = "cat"
ocr_text = "cut"
```

Получается:

```python
False, True, False
```

В числах:

```python
0, 1, 0
```

Сумма:

```python
1
```

## 18. Почему нужна разница длин

`zip` останавливается по короткой строке.

Пример:

```python
text = "abcd"
ocr_text = "ab"
```

`zip` сравнит только:

```text
a с a
b с b
```

Несовпадений внутри пар будет ноль.

Но символы `c` и `d` отсутствуют в OCR, значит ошибки есть.

Поэтому добавляется:

```python
length_difference = abs(len(text) - len(ocr_text))
```

### `len`

Возвращает длину строки:

```python
len("abcd")  # 4
len("ab")    # 2
```

### `abs`

Возвращает абсолютное значение:

```python
abs(4 - 2)  # 2
abs(2 - 4)  # 2
```

Неважно, какая строка длиннее: число лишних или недостающих символов должно быть положительным.

## 19. Итоговый score

```python
return different_characters + length_difference
```

Пример:

```python
text = "abcd"
ocr_text = "ax"
```

Попарно:

```text
a == a
b != x
```

`different_characters = 1`.

Разница длин:

```text
4 - 2 = 2
```

Итог:

```text
1 + 2 = 3
```

## 20. Почему аннотация говорит `float`, а получается `int`

Сигнатура задания требует:

```python
-> float
```

Но количество различающихся символов естественным образом является целым числом:

```text
0, 1, 2, 3, ...
```

Выражения `sum` и `abs(len(...) - len(...))` возвращают `int`.

В Python:

```python
67 == 67.0
```

даёт `True`.

При этом ожидаемый вывод задания показывает:

```text
score: 67
```

а не `67.0`. Поэтому сохранять целочисленный результат здесь разумно и соответствует примеру.

## 22. Формирование результатов

```python
results = [
    (self._eval_func(text, ocr_text), text, ocr_text)
    for text, ocr_text in self.documents
]
```

Каждый элемент `self.documents` выглядит так:

```python
(text, ocr_text)
```

Python распаковывает кортеж в две переменные:

```python
for text, ocr_text in self.documents
```

Для каждой пары вызывается:

```python
self._eval_func(text, ocr_text)
```

После этого создаётся новый кортеж:

```python
(score, text, ocr_text)
```

В результате:

```python
results = [
    (3, "hello", "he11o"),
    (1, "world", "wor1d"),
]
```

## 23. Сортировка

```python
results.sort(key=lambda result: result[0], reverse=True)
```

### Метод `.sort`

Изменяет существующий список на месте.

До:

```python
[(3, ...), (8, ...), (1, ...)]
```

После:

```python
[(8, ...), (3, ...), (1, ...)]
```

### `key`

Параметр `key` сообщает, по какому значению сортировать.

```python
lambda result: result[0]
```

означает:

> для каждого кортежа взять элемент с индексом 0.

Индекс 0 — score.

Кортеж:

```python
(score, text, ocr_text)
```

Индексы:

```text
0 → score
1 → text
2 → ocr_text
```

### `reverse=True`

По умолчанию сортировка идёт по возрастанию:

```text
1, 3, 8
```

Нам нужны документы с максимальными ошибками первыми:

```text
8, 3, 1
```

Поэтому используется `reverse=True`.

---

## 24. Почему сортировка стабильная

Python гарантирует стабильность сортировки.

Исходно:

```python
[
    (5, "first", "..."),
    (5, "second", "..."),
]
```

Ключи одинаковые:

```text
5 == 5
```

После сортировки `first` останется раньше `second`.

Это именно то, что требует условие.

Критически важно, что сортировка получает ключ:

```python
result[0]
```

Если ключ не указать, Python будет сравнивать весь кортеж и при равных score перейдёт к сравнению текста. Тогда исходный порядок может измениться.

---

## 25. Применение `limit`

```python
if limit is None:
    return results

return results[:limit]
```

### Когда `limit` не передан

Возвращается весь список.

### Когда `limit=3`

Используется срез:

```python
results[:3]
```

Он берёт элементы с начала до индекса 3, не включая 3:

```text
индексы 0, 1, 2
```

То есть первые три результата.

### Почему limit применяется после сортировки

Если сначала взять первые три исходных документа, а потом отсортировать только их, мы не получим три самых ошибочных документа во всём наборе.

Правильный порядок:

```text
оценить все документы
        ↓
отсортировать все документы
        ↓
взять первые limit документов
```

## 26. Полный поток выполнения программы

При вызове:

```python
ocr_eval = BaseEval("data.json")
```

происходит:

```text
1. Python создаёт объект BaseEval
2. Вызывает __init__
3. __init__ вызывает load_documents
4. Проверяется существование файла
5. JSON читается с диска
6. Проверяется, что корень JSON — список
7. Каждый документ передаётся в validate_document
8. Формируется список (text, ocr_text)
9. Список сохраняется в self.documents
10. Проверяется, что список не пустой
```

При вызове:

```python
result = ocr_eval.evaluate(limit=3)
```

происходит:

```text
1. Берётся первый документ
2. _eval_func считает score
3. Создаётся (score, text, ocr_text)
4. Процесс повторяется для всех документов
5. Все результаты сортируются по score по убыванию
6. При равных score сохраняется исходный порядок
7. Возвращаются первые три результата
```

## 29. Типичные неправильные решения

### Ошибка 1. Сравнивать только через `zip`

```python
return sum(a != b for a, b in zip(text, ocr_text))
```

Проблема: не учитываются лишние символы в конце более длинной строки.

### Ошибка 2. Сортировать весь кортеж

```python
results.sort(reverse=True)
```

Проблема: при равных score начинают участвовать `text` и `ocr_text`. Нарушается требование стабильности относительно исходного порядка.

### Ошибка 3. Сортировать по возрастанию

```python
results.sort(key=lambda result: result[0])
```

Проблема: сначала будут лучшие документы, а нужны документы с максимальным количеством ошибок.

### Ошибка 4. Применять `limit` до сортировки

```python
results = results[:limit]
results.sort(...)
```

Проблема: выбираются первые документы, а не самые ошибочные.

### Ошибка 5. Вернуть словари вместо кортежей

```python
return documents
```

Проблема: нарушается API. Ожидается:

```python
[(text, ocr_text), ...]
```

### Ошибка 6. Поменять порядок полей

```python
return doc["ocr_text"], doc["text"]
```

Проблема: оригинал и результат OCR меняются местами.

### Ошибка 7. Не использовать UTF-8

На некоторых системах русский текст может читаться некорректно.

### Ошибка 8. Перехватывать вообще все исключения

```python
except Exception:
    raise ValueError
```

Проблема: скрываются реальные ошибки, включая ошибки программирования и доступа к файлу.

Лучше перехватывать конкретно:

```python
json.JSONDecodeError
```

### Ошибка 9. Изменить имена методов или класса

Скрытые тесты импортируют и вызывают конкретный API:

```python
BaseEval
load_documents
validate_document
_eval_func
evaluate
```

Изменение имён приводит к нулю за функциональные тесты.

## 30. Сложность алгоритма

Пусть:

- `n` — количество документов;
- `m` — средняя длина текста.

### Загрузка

Чтение файла зависит от общего объёма данных.

### Оценка

Каждая пара строк сравнивается за время порядка `O(m)`.

Для всех документов:

```text
O(n × m)
```

### Сортировка

Сортировка `n` результатов:

```text
O(n log n)
```

### Общая оценка

```text
O(n × m + n log n)
```

Для учебного набора и большинства небольших evaluation-наборов этого более чем достаточно.

## 31. Почему метод технически простой, но методологически слабый

Алгоритм корректно реализует именно заданную формулу. Однако формула не умеет выравнивать строки.

Один пропуск вызывает сдвиг:

```text
original: abcdef
ocr:      abdef
```

После `ab` все позиции расходятся.

Поэтому большой score не всегда означает большое количество реальных OCR-операций.

Важно разделять:

- корректность кода относительно задания;
- пригодность метрики для реальной системы.

Код может быть реализован идеально, но сама метрика может иметь известные ограничения. Это нормальная ситуация в аналитике и ML.

## 32. Как объяснить решение на интервью

Можно ответить так:

> Я реализовал baseline для OCR evaluation. Данные загружаются из JSON и валидируются по контракту. Для каждой пары строк я считаю число несовпадений на одинаковых позициях через zip и добавляю абсолютную разницу длин, потому что zip обрезает сравнение по короткой строке. Затем формирую кортежи score, original text и OCR text и выполняю стабильную сортировку только по score по убыванию. Это быстрый и интерпретируемый baseline, но он завышает ошибку при вставках и удалениях из-за сдвига индексов. Для production-оценки следующим шагом я бы использовал edit distance, CER и сегментный анализ ошибок.

Такой ответ показывает одновременно:

- знание Python;
- понимание API;
- понимание алгоритма;
- понимание ограничений;
- продуктовое мышление.
