# DA/ML: QUANTILE T-TEST — Bootstrap для сравнения квантилей

## 1. Что изменилось относительно предыдущей страницы

На предыдущей странице мы сравнивали **средние ошибки** контрольной и экспериментальной групп.

Использовался обычный независимый двухвыборочный t-тест:

```python
ttest_ind(control, experiment)
```

Он отвечал на вопрос:

> Отличается ли средняя ошибка старой ETA-модели от средней ошибки новой ETA-модели?

Но исходная бизнес-задача была другой.

Команду интересовали самые плохие случаи — заказы, где модель сильно недооценила время доставки.

Например:

- приложение обещало 20 минут;
- заказ приехал через 60 минут;
- ошибка составила 40 минут.

Такие редкие, но тяжёлые ошибки находятся в **правом хвосте распределения**.

Поэтому теперь нужно сравнивать не средние, а высокий квантиль, например 95-й персентиль.

---

# 2. Что такое квантиль

Квантиль уровня `q` — это такое значение, ниже которого находится приблизительно доля `q` наблюдений.

Например:

```text
q = 0.95
```

означает 95-й персентиль.

Упрощённая интерпретация:

> Около 95% ошибок не превышают это значение, а около 5% ошибок больше него.

Если 95-й квантиль ошибки равен 18 минутам, то примерно для 95% заказов ошибка не превосходит 18 минут, а худшие 5% заказов имеют ещё более крупную ошибку.

---

# 3. Почему высокий квантиль важен для ETA

Средняя ошибка может быть небольшой, хотя отдельные клиенты получают катастрофически неверные обещания.

Пример:

```text
[0, 1, 1, 2, 2, 3, 3, 4, 5, 60]
```

Большинство ошибок небольшие, но один клиент получил ошибку 60 минут.

Для бизнеса этот хвост особенно дорог:

- клиент может отменить заказ;
- служба поддержки получает обращение;
- компания выдаёт компенсацию;
- падает доверие;
- снижается повторная покупка;
- ухудшается репутация ETA.

Высокий квантиль помогает сфокусироваться на худших сценариях.

---

# 4. Точечная оценка квантиля

Для одной выборки можно вычислить квантиль:

```python
np.quantile(control, 0.95)
```

Допустим:

```text
95-й квантиль A = 20 минут
95-й квантиль B = 16 минут
```

Точечная разница:

```text
16 - 20 = -4 минуты
```

На первый взгляд новая модель лучше на 4 минуты.

Но возникает вопрос:

> Является ли эта разница устойчивым эффектом или она могла появиться из-за случайного состава выборки?

У нас есть только одно значение квантиля для каждой группы.

Для статистического анализа нужна оценка неопределённости.

---

# 5. Почему один квантиль — это недостаточно

Квантиль, вычисленный по конечной выборке, является случайной величиной.

Если бы мы:

1. снова провели эксперимент;
2. получили других пользователей и другие заказы;
3. снова вычислили 95-й квантиль,

результат немного изменился бы.

Например:

```text
эксперимент 1: q95 = 20.0
эксперимент 2: q95 = 19.2
эксперимент 3: q95 = 21.1
эксперимент 4: q95 = 18.8
```

Нас интересует не только точка, но и то, насколько эта точка нестабильна.

---

# 6. Идея bootstrap

Bootstrap позволяет приблизительно воспроизвести многократное проведение эксперимента, используя уже собранные данные.

Для исходной выборки размера `N` мы:

1. случайно выбираем `N` элементов;
2. выбираем **с возвращением**;
3. считаем нужную статистику;
4. повторяем процесс много раз.

Если:

```python
x = [10, 20, 30]
```

то bootstrap-выборка размера 3 может быть:

```text
[10, 10, 30]
```

или:

```text
[20, 20, 20]
```

или:

```text
[30, 10, 20]
```

Некоторые элементы могут:

- попасть один раз;
- попасть несколько раз;
- не попасть совсем.

---

# 7. Почему выборка делается с возвращением

Параметр:

```python
replace=True
```

означает, что после выбора элемента он остаётся доступным для следующего выбора.

Без возвращения при размере новой выборки, равном размеру исходной, мы получили бы только перестановку исходных данных.

Например:

```text
исходная выборка: [1, 2, 3]
без возвращения:  [3, 1, 2]
```

Статистика не изменилась бы.

С возвращением возможны новые комбинации:

```text
[1, 1, 3]
[2, 2, 2]
[3, 1, 3]
```

Именно это создаёт вариативность bootstrap-оценок.

---

# 8. Размер bootstrap-выборки

Если исходная контрольная выборка содержит:

```text
len(control) = 1000
```

каждая bootstrap-выборка control тоже должна содержать 1000 элементов:

```python
size=len(control)
```

Для экспериментальной группы используется её собственный размер:

```python
size=len(experiment)
```

Группы могут иметь разное количество наблюдений.

---

# 9. Что мы делаем в этой задаче

Для каждой итерации:

1. создаём bootstrap-выборку из control;
2. считаем её квантиль;
3. сохраняем квантиль;
4. создаём bootstrap-выборку из experiment;
5. считаем её квантиль;
6. сохраняем квантиль.

После `n_bootstraps=1000` получаем:

```text
1000 квантилей control
1000 квантилей experiment
```

То есть вместо двух точечных оценок появляются два распределения bootstrap-квантилей.

---

# 10. Пример распределений

Допустим, bootstrap дал:

```text
control_quantiles:
[19.8, 20.4, 19.6, 21.0, 20.1, ...]

experiment_quantiles:
[15.9, 16.3, 15.5, 17.0, 16.1, ...]
```

Теперь можно сравнить эти два набора значений.

В рамках учебной задачи к ним применяется:

```python
ttest_ind(control_quantiles, experiment_quantiles)
```

---

# 11. Нулевая и альтернативная гипотезы

В логике этой задачи:

## Нулевая гипотеза

```text
H0: средние bootstrap-распределений квантилей равны
```

То есть нет статистически значимого различия между соответствующими квантилями групп.

## Альтернативная гипотеза

```text
H1: средние bootstrap-распределений квантилей различаются
```

По умолчанию `ttest_ind` использует двухстороннюю альтернативу.

Это означает, что тест обнаруживает:

- уменьшение квантиля;
- увеличение квантиля.

Направление эффекта отдельно определяется сравнением самих оценок.

---

# 12. Почему `quantile=0.95`

Параметр функции:

```python
quantile: float = 0.95
```

означает, что по умолчанию сравнивается 95-й квантиль.

Однако автопроверка также передаёт:

```python
quantile=0.05
```

Поэтому нельзя жёстко прописывать `0.95` внутри функции.

Неправильно:

```python
np.quantile(sample, 0.95)
```

Правильно:

```python
np.quantile(sample, quantile)
```

---

# 13. Что означает `quantile=0.05`

Это 5-й персентиль.

Он находится в нижнем хвосте распределения.

В зависимости от определения ошибки нижний хвост может описывать:

- сильный перепрогноз;
- слишком консервативное ETA;
- редкие отрицательные значения;
- другую бизнес-проблему.

Функция должна быть универсальной и работать с любым допустимым `quantile`.

---

# 14. Что означает `n_bootstraps=1000`

Это количество повторений bootstrap-процедуры.

Чем больше повторений:

- тем стабильнее симулированное распределение;
- тем меньше случайный Monte Carlo noise;
- тем больше вычислительное время.

В условии значение по умолчанию:

```python
n_bootstraps=1000
```

Нельзя игнорировать этот параметр.

Неправильно:

```python
for _ in range(10_000):
```

Правильно:

```python
for _ in range(n_bootstraps):
```

---

# 15. Почему используется `np.random.choice`

Пример из урока использует:

```python
np.random.choice(
    x,
    size=len(x),
    replace=True,
)
```

Эта функция:

- случайно выбирает элементы из `x`;
- создаёт выборку нужного размера;
- позволяет повторения.

Для соответствия постановке и возможным тестам нужно использовать глобальный генератор `np.random`.

Если автотест перед вызовом задаёт:

```python
np.random.seed(42)
```

то результаты будут воспроизводимыми.

Использование отдельного `np.random.default_rng()` могло бы не учитывать такой seed.

---

# 16. Почему контроль и эксперимент bootstrap-ятся отдельно

Группы A и B являются независимыми выборками.

Поэтому для каждой группы формируется собственная bootstrap-выборка:

```python
control_sample = np.random.choice(...)
experiment_sample = np.random.choice(...)
```

Нельзя объединять группы до bootstrap, потому что это уничтожит наблюдаемое различие между ними.

---

# 17. Почему размеры групп могут быть разными

Пусть:

```text
len(control) = 5000
len(experiment) = 3000
```

Тогда:

- bootstrap control имеет размер 5000;
- bootstrap experiment имеет размер 3000.

Так сохраняется структура исходного эксперимента.

`ttest_ind` допускает разные размеры входных выборок.

---

# 18. Что возвращает функция

Функция должна вернуть:

```python
p_value, result
```

Где:

- `p_value` — p-value t-теста;
- `result` — `True`, если различие статистически значимо;
- `result` — `False`, если оснований отвергать H0 нет.

Правило:

```python
result = p_value < alpha
```

---

# 19. Параметр alpha

По умолчанию:

```python
alpha = 0.05
```

Если:

```text
p_value < 0.05
```

результат значим.

Автотест отдельно проверяет:

```python
alpha=0.01
```

Поэтому сравнение должно использовать переменную:

```python
p_value < alpha
```

а не фиксированное число.

---

# 20. Полный алгоритм

```text
Вход:
control
experiment
alpha
quantile
n_bootstraps

1. Создать пустой список control_quantiles.
2. Создать пустой список experiment_quantiles.
3. Повторить n_bootstraps раз:
   3.1. Получить bootstrap-выборку control.
   3.2. Посчитать её quantile.
   3.3. Добавить значение в control_quantiles.
   3.4. Получить bootstrap-выборку experiment.
   3.5. Посчитать её quantile.
   3.6. Добавить значение в experiment_quantiles.
4. Выполнить ttest_ind для двух списков квантилей.
5. Сравнить p_value с alpha.
6. Вернуть p_value и bool-вердикт.
```

# 21. Бизнес-интерпретация результата

Допустим:

```text
q95_control = 25 минут
q95_experiment = 18 минут
p_value = 0.002
result = True
```

Можно сказать:

> В bootstrap-анализе 95-й квантиль ошибки новой модели статистически отличается от старой. Точечная оценка снизилась с 25 до 18 минут, то есть худшие сценарии стали лучше.

Но одного `result=True` недостаточно.

Нужно также смотреть:

- направление изменения;
- абсолютный эффект;
- относительный эффект;
- доверительный интервал;
- бизнес-ценность;
- guardrail-метрики.

---

# 22. Статистическая и практическая значимость

При большом числе наблюдений даже маленькое изменение может стать статистически значимым.

Например:

```text
q95 A = 20.0
q95 B = 19.9
```

Разница:

```text
-0.1 минуты = -6 секунд
```

Она может быть статистически значимой, но бизнес-ценность может быть нулевой.

До эксперимента нужно определить минимально полезное изменение:

```text
MDE = 2 минуты
```

Тогда результат должен удовлетворять двум требованиям:

1. статистическая значимость;
2. практически значимый размер эффекта.

# 24. Важное методологическое замечание

В рамках курса требуется:

1. построить bootstrap-распределения квантилей;
2. применить к ним `ttest_ind`.

Именно это нужно реализовать для получения баллов.

В профессиональном экспериментальном анализе часто предпочитают:

- bootstrap доверительный интервал **разницы квантилей**;
- percentile bootstrap;
- BCa bootstrap;
- пермутационный тест разницы квантилей;
- quantile regression.

Причина: bootstrap-реплики являются симуляциями из одних исходных данных, а не независимыми новыми экспериментами. Обычный t-тест на всех bootstrap-репликах может создавать слишком маленький p-value при увеличении `n_bootstraps`.

Но учебное решение должно буквально соответствовать постановке.

---

# 25. Почему нельзя самовольно заменить метод

Даже если альтернативный статистический метод кажется более строгим, автопроверка ожидает конкретный алгоритм.

Нельзя вместо задания использовать:

- Mann–Whitney;
- Kolmogorov–Smirnov;
- permutation test;
- bootstrap CI без t-теста;
- Welch t-test с `equal_var=False`;
- ручную формулу.

Нужно использовать:

```python
ttest_ind(control_quantiles, experiment_quantiles)
```

с дефолтными параметрами SciPy.

---

# 26. Финальное решение

```python
from typing import List, Tuple

import numpy as np
from scipy.stats import ttest_ind


def quantile_ttest(
    control: List[float],
    experiment: List[float],
    alpha: float = 0.05,
    quantile: float = 0.95,
    n_bootstraps: int = 1000,
) -> Tuple[float, bool]:
    """
    Bootstrapped t-test for quantiles of two samples.
    """
    control_quantiles = []
    experiment_quantiles = []

    for _ in range(n_bootstraps):
        control_sample = np.random.choice(
            control,
            size=len(control),
            replace=True,
        )
        experiment_sample = np.random.choice(
            experiment,
            size=len(experiment),
            replace=True,
        )

        control_quantiles.append(
            np.quantile(control_sample, quantile)
        )
        experiment_quantiles.append(
            np.quantile(experiment_sample, quantile)
        )

    _, p_value = ttest_ind(
        control_quantiles,
        experiment_quantiles,
    )
    result = bool(p_value < alpha)

    return float(p_value), result
```

# 27. Главная мысль страницы

Bootstrap превращает одну точечную оценку квантиля в эмпирическое распределение оценок.

В рамках задания это позволяет:

```text
один квантиль A
и
один квантиль B
```

превратить в:

```text
распределение bootstrap-квантилей A
и
распределение bootstrap-квантилей B
```

после чего применить статистический критерий.
