# QUANTILE T-TEST — построчное объяснение кода с нуля

## 1. Полный код

```python
from typing import List, Tuple

from scipy.stats import ttest_ind


def ttest(
    control: List[float],
    experiment: List[float],
    alpha: float = 0.05,
) -> Tuple[float, bool]:
    """Two-sample t-test for the means of two independent samples."""
    _, p_value = ttest_ind(control, experiment)
    result = bool(p_value < alpha)

    return float(p_value), result
```

# 6. Импорт статистического теста

```python

```

---

# 7. Почему мы не пишем формулу вручную

Ручная реализация потребовала бы:

- средние;
- выборочные дисперсии;
- pooled variance;
- standard error;
- degrees of freedom;
- t-distribution CDF;
- корректное двустороннее p-value.

Это намного сложнее и создаёт риск ошибки.

Условие прямо направляет к SciPy, поэтому библиотечная функция — правильное решение.

# 20. Что такое p-value в коде

`p_value` — число, например:

```python

```

Маленькое p-value означает, что наблюдаемая разница плохо согласуется с H0 равных средних.

# 21. Создание result

```python
result = bool(p_value < alpha)
```

Сначала Python вычисляет сравнение.

Пример:

```python

```

получается:

```python

```

Другой пример:

```python

```

```python

```

---

# 22. Почему используется `<`

Правило статистической значимости в задаче:

```text
p_value < alpha
```

Если условие истинно, различие считается значимым.

Если p-value не меньше alpha, оснований отвергнуть H0 недостаточно.

# 23. Зачем `bool(...)`

Сравнение может вернуть NumPy-тип `numpy.bool_`.

Он похож на Python `bool`, но строгая проверка типов иногда различает их.

```python
bool(...)
```

гарантирует обычный Python `True` или `False`.

# 35. Ошибка: парный тест

Неправильно:

```python

```

Группы A и B в условии независимы, поэтому нужен `ttest_ind`.

# 43. Почему не нужен if/else

Можно написать:

```python

```

Но сравнение уже возвращает bool.

Поэтому лучше:

```python
result = bool(p_value < alpha)
```

# 44. Альтернативная компактная версия

```python
def ttest(
    control: List[float],
    experiment: List[float],
    alpha: float = 0.05,
) -> Tuple[float, bool]:
    _, p_value = ttest_ind(control, experiment)
    return float(p_value), bool(p_value < alpha)
```

Она корректна, но отдельная переменная `result` понятнее новичку и повторяет формулировку условия.
