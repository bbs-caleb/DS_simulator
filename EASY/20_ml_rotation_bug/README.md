```python
import numpy as np

import utils


def test_rotated_image():
    image = np.zeros((100, 200, 3), dtype=np.uint8)

    result = utils.rotated_image(image, angle=0)

    assert result.shape == image.shape
```

## Какой баг ловит тест

В NumPy цветное изображение обычно имеет форму:

```text
(height, width, channels)
```

Например:

```text
(100, 200, 3)
```

означает:

- высота — 100 пикселей;
- ширина — 200 пикселей;
- три цветовых канала.

В исходной функции написано:

```python
width, height, channels = image.shape
```

Поэтому высота и ширина получают неправильные имена.

Далее `cv2.warpAffine` ожидает размер результата в формате:

```text
(width, height)
```

Но ему передаются перепутанные значения. В итоге входная форма:

```text
(100, 200, 3)
```

становится:

```text
(200, 100, 3)
```

На квадратном изображении ошибка незаметна, поэтому тест использует прямоугольный массив.

## Почему угол равен 0°

При нулевом угле изображение не должно менять форму. Такой пример изолирует именно ошибку размерностей:

- нет реального поворота;
- нет сложной интерполяции;
- нет споров об обрезании углов;
- нет зависимости от конкретной картинки.
