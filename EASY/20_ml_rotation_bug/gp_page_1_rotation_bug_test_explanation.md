# Часть I. Что такое автоматический тест

## 2. Простое определение

Тест — это программа, которая:

1. создаёт известный вход;
2. вызывает проверяемую функцию;
3. получает фактический результат;
4. сравнивает его с ожидаемым;
5. падает, если результат неправильный.

Общая форма:

```python
def test_something():
    input_data = ...
    result = function(input_data)
    assert result == expected
```

## 3. Что делает pytest

`pytest` автоматически ищет:

- файлы, имя которых начинается с `test_`;
- функции, имя которых начинается с `test_`.

Поэтому:

```text
test_utils.py
```

и:

```python
def test_rotated_image():
```

будут найдены без дополнительной настройки.

## 4. Как работает assert

```python
assert condition
```

Если `condition` равно `True`, тест проходит.

Если `condition` равно `False`, Python выбрасывает `AssertionError`, и pytest помечает тест как упавший.

Пример:

```python
assert 2 + 2 == 4
```

проходит.

```python
assert 2 + 2 == 5
```

падает.

# Часть II. Разбор каждой строки решения

## 5. Импорт NumPy

```python
import numpy as np
```

NumPy — библиотека для работы с массивами.

Изображение в computer vision обычно представлено NumPy-массивом.

`as np` создаёт короткое имя:

```python
numpy.zeros(...)
```

можно писать как:

```python
np.zeros(...)
```

## 6. Импорт utils

```python
import utils
```

Проверяющая система предоставляет файл `utils.py` с функцией:

```python
rotated_image
```

После импорта вызываем её так:

```python
utils.rotated_image(...)
```

Это точно соответствует API из условия.

## 7. Объявление тестовой функции

```python
def test_rotated_image():
```

- `def` создаёт функцию;
- `test_` позволяет pytest найти её;
- круглые скобки пустые, потому что аргументы не нужны;
- двоеточие начинает тело функции;
- внутри используется отступ в четыре пробела.

## 8. Создание изображения

```python
image = np.zeros((100, 200, 3), dtype=np.uint8)
```

Разберём каждую часть.

### `np.zeros`

Создаёт массив, заполненный нулями.

Для обычного трёхканального изображения нули означают чёрные пиксели.

### `(100, 200, 3)`

Форма массива:

```text
height = 100
width = 200
channels = 3
```

Это прямоугольное цветное изображение.

### Почему размеры разные

Нам обязательно нужно:

```text
height != width
```

Потому что баг переставляет высоту и ширину местами.

Если взять квадрат:

```text
(100, 100, 3)
```

то перестановка останется незаметной.

### `dtype=np.uint8`

Каждый элемент — беззнаковое целое число из диапазона:

```text
0..255
```

Это стандартный тип обычных изображений.

`uint8` означает:

- `u` — unsigned;
- `int` — integer;
- `8` — 8 бит.

## 9. Вызов функции

```python
result = utils.rotated_image(image, angle=0)
```

Передаём:

- созданное изображение;
- угол `0°`.

Возвращаемое значение записываем в `result`.

## 10. Почему угол равен 0°

При нулевом угле изображение не должно реально поворачиваться.

Это идеальный диагностический случай:

- не возникает сложная интерполяция;
- не обрезаются углы;
- не появляются вопросы о border mode;
- ожидаемое поведение очевидно;
- форма должна сохраниться.

Так мы изолируем именно ошибку размерностей.

## 11. Проверка формы

```python
assert result.shape == image.shape
```

Исходная форма:

```text
image.shape == (100, 200, 3)
```

На дефектной реализации:

```text
result.shape == (200, 100, 3)
```

Сравнение:

```python
(200, 100, 3) == (100, 200, 3)
```

возвращает `False`.

Значит, тест падает.

После исправления:

```text
result.shape == (100, 200, 3)
```

и тест проходит.

# Часть III. Где именно находится баг

## 12. Как NumPy описывает изображение

Для цветного массива:

```text
(height, width, channels)
```

Пример:

```text
(100, 200, 3)
```

означает:

- 100 строк;
- 200 столбцов;
- три канала.

## 13. Ошибочное присваивание

В функции:

```python
width, height, channels = image.shape
```

При форме `(100, 200, 3)` получается:

```text
width = 100
height = 200
channels = 3
```

Но правильный смысл:

```text
height = 100
width = 200
channels = 3
```

## 14. Что ожидает warpAffine

`cv2.warpAffine` получает выходной размер в формате:

```text
(width, height)
```

Исходная функция передаёт:

```python
(width, height)
```

Но переменные содержат перепутанные значения.

Фактически передаётся:

```text
(100, 200)
```

вместо:

```text
(200, 100)
```

## 15. Почему выходная shape равна `(200, 100, 3)`

OpenCV понимает:

```text
output_width = 100
output_height = 200
```

Но NumPy показывает результат как:

```text
(output_height, output_width, channels)
```

Следовательно:

```text
(200, 100, 3)
```

# Часть IV. Почему центр поворота выглядит правильно

## 16. Строка с transform

```python
transform = cv2.getRotationMatrix2D(
    (height / 2, width / 2),
    angle,
    1,
)
```

OpenCV ждёт центр:

```text
(center_x, center_y)
```

то есть:

```text
(real_width / 2, real_height / 2)
```

После ошибочного присваивания:

```text
height variable = real_width
width variable = real_height
```

Поэтому выражение:

```text
(height / 2, width / 2)
```

случайно превращается в:

```text
(real_width / 2, real_height / 2)
```

Получается редкий случай, когда неправильные имена переменных в одной строке компенсируются перестановкой в другой строке.

Но в `warpAffine` компенсации уже нет, поэтому размер ломается.


- interpolation;
