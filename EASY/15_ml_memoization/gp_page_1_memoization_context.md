# Мемоизация: полный разбор с нуля

## 1. Зачем бизнесу вообще нужна эта тема

На первый взгляд задача выглядит как небольшое упражнение по Python:

```python
@memoize
def some_function(...):
    ...
```

Но в реальной работе мемоизация связана не просто с синтаксисом Python. Она связана с важнейшей инженерной задачей:

> Как сделать сервис достаточно быстрым и дешёвым, не выполняя повторно одинаковую работу?

Это актуально для:

- рекомендательных систем;
- антифрода;
- кредитного скоринга;
- динамического ценообразования;
- поиска;
- ранжирования;
- прогнозирования спроса;
- генеративных моделей;
- систем подбора контента;
- игровых платформ;
- рекламных систем;
- маркетплейсов;
- банковских приложений;
- сервисов знакомств;
- криптовалютных платформ;
- любых API с высокой нагрузкой.

Пользователю неважно, насколько сложная модель находится внутри. Пользователь видит только:

- быстро ли открылась страница;
- успел ли загрузиться список рекомендаций;
- прошла ли операция;
- появился ли ответ;
- не зависло ли приложение;
- не была ли отклонена транзакция из-за технической ошибки.

Поэтому скорость вычисления — это не «косметическая оптимизация». Она влияет на деньги.

# Часть I. Real-time сервисы и SLA

## 2. Что такое real-time сервис

Real-time сервис — это сервис, который должен дать ответ достаточно быстро, чтобы результат оставался полезным в текущем пользовательском или бизнес-процессе.

Термин `real-time` не всегда означает буквально одну микросекунду.

В разных задачах допустимое время ответа различается.

| Система | Что происходит | Типичная задержка |
|---|---|---:|
| Антифрод банковской карты | Проверка операции перед одобрением | десятки миллисекунд |
| Поисковое ранжирование | Формирование выдачи | десятки или сотни миллисекунд |
| Рекомендации главной страницы | Подбор товаров или видео | десятки или сотни миллисекунд |
| Чат-бот | Генерация первого ответа | секунды |
| Прогнозирование спроса ночью | Пакетный расчёт | минуты или часы |
| Ежемесячный отчёт | Аналитика | минуты |

Следовательно, real-time определяется не абсолютным числом, а требованиями процесса.

---

## 3. Что такое SLA

`SLA` — `Service Level Agreement`, соглашение об уровне сервиса.

Оно описывает обязательства перед клиентом, партнёром или внутренним заказчиком.

Примеры:

- 95% запросов должны обрабатываться быстрее 50 ms;
- сервис должен быть доступен 99.5% времени;
- сервис должен выдерживать 5000 запросов в секунду;
- доля ошибок не должна превышать 0.1%;
- данные должны обновляться не реже одного раза в 15 минут.

SLA — не просто техническая метрика, а обещание.

Нарушение SLA может привести к:

- плохому пользовательскому опыту;
- падению конверсии;
- финансовым потерям;
- штрафам;
- отказу партнёра;
- росту обращений в поддержку;
- репутационному ущербу;
- невозможности запустить продукт.

---

## 4. SLA, SLO и SLI

### SLI — Service Level Indicator

Фактически измеряемый показатель:

```text
Фактический p95 latency = 43 ms
```

### SLO — Service Level Objective

Внутренняя цель команды:

```text
p95 latency должен быть меньше 45 ms
```

### SLA — Service Level Agreement

Внешнее обязательство:

```text
p95 latency должен быть меньше 50 ms
```

Обычно SLO строже SLA:

```text
SLA: p95 < 50 ms
SLO: p95 < 40 ms
Фактический SLI: p95 = 36 ms
```

---

## 5. Что такое latency

`Latency` — время от получения запроса до выдачи ответа.

Если запрос пришёл в:

```text
10:00:00.000
```

а ответ отправлен в:

```text
10:00:00.042
```

latency равна:

```text
42 ms
```

Общая задержка может включать:

```text
сеть
+ чтение данных
+ подготовка признаков
+ вызов модели
+ постобработка
+ логирование
+ сериализация
```

Даже если модель работает за 10 ms, весь сервис может отвечать за 70 ms.

---

## 6. p50, p95 и p99

Среднее время ответа может скрывать медленные запросы.

Пример:

```text
10, 11, 11, 12, 12, 13, 13, 14, 15, 500 ms
```

- `p50` — медиана;
- `p95` — 95% запросов завершились не медленнее этого значения;
- `p99` — 99% запросов завершились не медленнее этого значения.

«Хвост» важен, потому что медленные запросы могут вызывать:

- закрытие страницы;
- таймаут платежа;
- повторные запросы;
- рост нагрузки;
- каскадный отказ.

---

## 7. Availability

`Availability` — доступность сервиса.

При SLA 99.5% приблизительный допустимый простой за 30 дней:

```text
30 × 24 × 60 = 43 200 минут
0.5% × 43 200 = 216 минут
```

То есть около 3 часов 36 минут.

---

## 8. Throughput и RPS

`RPS` — requests per second.

```text
5000 RPS
```

означает 5000 запросов в секунду.

Если каждый запрос запускает тяжёлую модель, растут:

- CPU/GPU;
- память;
- число инстансов;
- стоимость;
- очереди;
- latency;
- риск отказа.

Мемоизация уменьшает число реальных вычислений, если входы повторяются.

# Часть II. Почему ML-сервисам трудно соблюдать SLA

## 9. Типичный ML inference

```text
1. Получить запрос.
2. Провалидировать.
3. Найти пользователя.
4. Получить признаки.
5. Рассчитать признаки.
6. Преобразовать данные.
7. Запустить модель.
8. Применить бизнес-правила.
9. Записать лог.
10. Вернуть ответ.
```

Дорогой может быть не только модельный inference.

Пример:

```text
База: 15 ms
Feature Store: 20 ms
Модель: 8 ms
Правила: 5 ms
Сеть: 10 ms
Итого: 58 ms
```

При SLA 50 ms сервис уже не укладывается.

## 10. Золотое правило оптимизации

> Если что-то можно не делать, это нужно не делать.

Самая быстрая операция — та, которую не выполнили.

```text
Первый запрос:
user_id=100, item_id=500
→ вычислить
→ сохранить

Повторный запрос:
user_id=100, item_id=500
→ достать готовое
```

# Часть III. Кэширование и мемоизация

## 11. Кэш

Кэш — быстрое хранилище ранее полученных данных или результатов.

```text
ключ → значение
```

Например:

```text
"user:100:recommendations" → [42, 91, 17, 88]
```

Алгоритм:

```text
Есть значение?
Да → вернуть.
Нет → вычислить, сохранить, вернуть.
```

---

## 12. Cache hit и cache miss

`Cache hit` — нужный результат найден.

`Cache miss` — результата нет, поэтому его нужно вычислить.

---

## 13. Hit rate

```text
hit rate = cache hits / all cache lookups
```

Пример:

```text
1000 проверок
800 hits
200 misses
hit rate = 80%
```

Нужно учитывать не только hit rate, но и:

- цену исходного вычисления;
- цену чтения из кэша;
- свежесть;
- память;
- корректность ключа.

---

## 14. Мемоизация

Мемоизация сохраняет:

```text
аргументы функции → результат функции
```

Пример:

```python
square(5) → 25
```

Сохраняем:

```python
cache[(5,)] = 25
```

При следующем вызове `square(5)` берём `25` из словаря.

---

## 15. Отличие от общего кэширования

Кэшировать можно:

- страницу;
- SQL;
- API-ответ;
- признаки;
- изображение;
- модельный ответ;
- сессию.

Мемоизация относится к вызову функции:

```text
одинаковые аргументы → ранее сохранённый результат
```

---

## 16. Главное условие корректности

Результат должен определяться аргументами.

Хорошо:

```python
def add(x, y):
    return x + y
```

Опасно:

```python
def current_time():
    return time.time()
```

У функции нет аргументов, но результат меняется.

Опасно:

```python
def account_balance(user_id):
    return read_balance_from_database(user_id)
```

Баланс меняется при том же `user_id`.

---

## 17. Чистая функция

Чистая функция:

1. зависит только от аргументов;
2. не меняет внешнее состояние.

```python
def multiply(x, y):
    return x * y
```

Функция с побочным эффектом:

```python
def save_payment(payment):
    database.insert(payment)
```

Её нельзя бездумно мемоизировать, потому что повторный вызов должен выполнить действие.

# Часть IV. Декораторы

## 18. Функция в Python — объект

Функцию можно:

- сохранить в переменную;
- передать другой функции;
- вернуть;
- положить в коллекцию.

```python
def hello():
    return "Hello"

another_name = hello
print(another_name())
```

---

## 19. Функция высшего порядка

Она принимает или возвращает функцию.

Декоратор:

```python
def decorator(original_function):
    def new_function(...):
        result = original_function(...)
        return result

    return new_function
```

---

## 20. Смысл декоратора

Он добавляет поведение:

- замер времени;
- логирование;
- авторизацию;
- retries;
- валидацию;
- кэш;
- метрики.

---

## 21. Синтаксис `@decorator`

```python
@timer
def calculate(x, y):
    return x + y
```

эквивалентен:

```python
def calculate(x, y):
    return x + y

calculate = timer(calculate)
```

После этого имя `calculate` указывает на функцию-обёртку.

---

## 22. Wrapper

```python
def decorator(func):
    def wrapper(*args, **kwargs):
        print("До")
        result = func(*args, **kwargs)
        print("После")
        return result

    return wrapper
```

---

## 23. Замыкание

Внутренняя функция запоминает переменные внешней функции.

```python
def make_multiplier(multiplier):
    def multiply(value):
        return value * multiplier

    return multiply
```

`double = make_multiplier(2)` продолжает помнить `multiplier = 2`.

В `memoize` wrapper помнит:

- `func`;
- `cache`.

# Часть V. `*args` и `**kwargs`

## 24. Почему нельзя ограничиться `x, y`

```python
def wrapper(x, y):
    return func(x, y)
```

работает только с двумя аргументами.

Не подходит для:

```python
f()
f(1)
f(1, 2, 3)
f(name="Alex")
```

---

## 25. `*args`

Собирает позиционные аргументы в кортеж:

```python
def show(*args):
    print(args)

show(10, 20, 30)
```

Результат:

```text
(10, 20, 30)
```

---

## 26. `**kwargs`

Собирает именованные аргументы в словарь:

```python
def show(**kwargs):
    print(kwargs)

show(name="Nodir", age=25)
```

Результат:

```text
{"name": "Nodir", "age": 25}
```

---

## 27. Универсальная обёртка

```python
def wrapper(*args, **kwargs):
    return func(*args, **kwargs)
```

В объявлении звёздочки собирают аргументы, а при вызове — распаковывают.

---

# Часть VI. Ключ кэша

## 28. Что является ключом

```text
ключ = все аргументы вызова
значение = результат
```

Для `add(2, 3)`:

```python
key = ((2, 3), ())
value = 5
```

---

## 29. Почему `args` подходит

`args` — кортеж.

Кортеж из хешируемых элементов можно использовать как ключ.

---

## 30. Почему `kwargs` нельзя использовать напрямую

`kwargs` — словарь, а словарь не хешируется:

```text
TypeError: unhashable type: 'dict'
```

Нужно преобразовать:

```python
tuple(sorted(kwargs.items()))
```

---

## 31. Зачем сортировка

```python
f(x=1, y=2)
f(y=2, x=1)
```

должны приводить к одному ключу.

Сортировка приводит пары к каноническому порядку.

---

## 32. Итоговый ключ

```python
key = (args, tuple(sorted(kwargs.items())))
```

Пример:

```python
f(10, y=20, z=30)
```

```text
(
    (10,),
    (("y", 20), ("z", 30))
)
```

---

# Часть VII. Алгоритм

## 33. Пошагово

```text
1. Получить args и kwargs.
2. Создать key.
3. Проверить key в cache.
4. Если нет:
   - вызвать func;
   - сохранить результат.
5. Вернуть cache[key].
```

---

## 34. Первый вызов

```python
slow_add(2, 3)
```

Кэш пуст, функция выполняется, результат `5` сохраняется:

```python
{
    ((2, 3), ()): 5
}
```

---

## 35. Повторный вызов

Тот же ключ уже существует.

Исходная функция не вызывается.

# Часть VIII. `functools.wraps`

## 36. Проблема без `wraps`

После декорирования могут потеряться:

- `__name__`;
- `__doc__`;
- сведения для отладки;
- данные для документации.

---

## 37. Решение

```python
from functools import wraps


@wraps(func)
def wrapper(...):
    ...
```

Это стандарт хорошего декоратора.

# Часть IX. Бизнес-применения

## 38. Финтех

Можно временно кэшировать повторный скоринг только при корректном ключе:

```text
client_id
+ snapshot_of_features
+ model_version
+ product
```

Нельзя бездумно кэшировать:

- баланс;
- лимит;
- статус платежа;
- динамический риск.

---

## 39. Антифрод

Можно кэшировать:

- статические признаки устройства;
- BIN-справочник;
- нормализацию адреса;
- географию IP;
- неизменяемые преобразования.

Итоговое решение часто зависит от конкретной операции и времени.

---

## 40. E-commerce

Можно кэшировать:

- embeddings товаров;
- похожие товары;
- категории;
- тяжёлые преобразования каталога.

Цена и остаток требуют TTL или событийного инвалидирования.

---

## 41. Маркетплейс

Нужно учитывать:

- пользователя;
- продавца;
- цену;
- регион;
- доставку;
- остаток;
- модель;
- эксперимент.

---

## 42. Рекомендательная система

Уровни кэша:

```text
user → candidates
item → embedding
user → embedding
(user, context, model_version) → ranked list
```

---

## 43. Поиск

Ключ может включать:

- текст запроса;

- регион;
- фильтры;
- сортировку;
- персонализацию;
- права;

## 44. Генеративный AI

Можно кэшировать:

- embeddings;
- токенизацию;
- preprocessing;
- детерминированные классификаторы;
- одинаковый запрос при фиксированных параметрах.

Ключ должен учитывать:

- prompt;
- system prompt;
- model version;
- temperature;
- tools;
- контекст;
- policy version.

---

## 45. Космические и промышленные системы

Можно мемоизировать детерминированные расчёты при фиксированных:

- параметрах;
- алгоритме;
- единицах;
- версии;
- входном срезе.

---

## 46. Энергетика

Примеры:

- расчёт свойств смеси;
- геопространственные преобразования;
- повторные части симуляции;
- признаки для неизменившегося среза.

---

## 47. iGaming и casino

Допустимо кэшировать статические данные и детерминированные преобразования.

Нельзя:

- кэшировать новый случайный исход;
- смешивать пользователей;
- возвращать старый баланс;
- обходить корректный расчёт операции.

---

## 48. Crypto

Можно кэшировать данные неизменяемого блока.

Текущие баланс, цена, mempool и confirmations меняются.

Нужна привязка к номеру блока или хешу состояния.

---

## 49. Dating

Можно кэшировать embeddings и обработанные фотографии.

Перед показом нужны свежие проверки:

- профиль не удалён;
- нет блокировки;
- география допустима;
- правила безопасности соблюдены.

# Часть X. Когда мемоизация вредна

## 50. Меняющиеся данные

```python
def get_stock(product_id):
    return database.get_stock(product_id)
```

Остаток меняется.

---

## 51. Текущее время

```python
def current_time():
    return time.time()
```

---

## 52. Случайность

```python
def random_number():
    return random.random()
```

---

## 53. Побочный эффект

```python
def charge_card(payment):
    ...
```

---

## 54. Уникальные ключи

Если каждый запрос уникален, кэш занимает память, но почти не ускоряет.

---

## 55. Нехешируемые аргументы

Простой ключ не поддерживает списки, словари и множества.

```python
f([1, 2, 3])
```

может вызвать:

```text
TypeError: unhashable type: 'list'
```

В учебной задаче обычно предполагаются хешируемые аргументы.

## 62. Итог

Мемоизация меняет:

```text
вычисления
```

на:

```text
память + риск устаревания
```

Сильный специалист спрашивает не только «как сохранить?», но и:

- когда результат корректен;
- как долго;
- для кого;
- для какой версии модели;
- как доказать пользу.
