## 1. Что теперь требуется сделать

На первой странице был только общий бизнес-сценарий: KarpovExpress собирает HTML-страницы конкурентов, чтобы отслеживать цены, сопоставлять товары и принимать решения по динамическому ценообразованию.

Теперь появилось точное программное задание. Нужно реализовать функцию:

```python
def parse_product_info(html: str) -> dict:
```

Она принимает HTML одной товарной карточки и возвращает словарь:

```python
{
    "title": "...",
    "category": "...",
    "old_price": "...",
    "new_price": "...",
}
```

Это небольшой, но важный слой общего data pipeline: функция превращает HTML-разметку в обычные структурированные данные Python.

## 2. Бизнесовая задача

Представим, что KarpovExpress ежедневно получает тысячи карточек товаров конкурентов. Браузер видит разметку:

```html
<h1 class="product-title"><b>Телевизор</b> Ultra HD</h1>
<div class="product-category">Электроника / Телевизоры</div>
<span class="price-old"><strike>55 000 ₽</strike></span>
<span class="price-new">50 000 ₽</span>
```

Человеку понятно, что:

- товар называется «Телевизор Ultra HD»;
- категория — «Электроника / Телевизоры»;
- прежняя цена — `55 000 ₽`;
- актуальная цена — `50 000 ₽`.

Компьютеру нужно формально указать, откуда брать каждое поле. После обработки мы должны получить:

```python
{
    "title": "Телевизор Ultra HD",
    "category": "Электроника / Телевизоры",
    "old_price": "55 000 ₽",
    "new_price": "50 000 ₽",
}
```

Дальше результат можно:

- записать в таблицу конкурентных цен;
- преобразовать в строку pandas DataFrame;
- передать товарному матчингу;
- сравнить с нашей ценой;
- сохранить в историю;
- использовать в отчёте категорийного менеджера;
- передать в pricing engine после дополнительных проверок.

## 3. Почему нельзя просто найти все числа

Наивный вариант:

```python
re.findall(r"\d+", html)
```

плох, потому что HTML может содержать:

```html
<h1>Телевизор 4K 2026</h1>
<div>Рейтинг: 4.8</div>
<div>120 отзывов</div>
<span class="price-old">55 000 ₽</span>
<span class="price-new">50 000 ₽</span>
```

Тогда будут найдены числа:

```text
4, 2026, 4, 8, 120, 55, 000, 50, 000
```

Нельзя надёжно определить, где:

- модель;
- год;
- рейтинг;
- число отзывов;
- старая цена;
- новая цена.

Поэтому ищется не просто число, а содержимое конкретного HTML-тега с конкретным CSS-классом.

## 4. Контракт функции

### Вход

```python
html: str
```

Это строка с HTML-кодом.

### Выход

```python
dict
```

Словарь обязан содержать ровно нужные ключи:

```python
"title"
"category"
"old_price"
"new_price"
```

### Тип значений

Все значения остаются строками.

Это означает, что цена должна быть:

```python
"50 000 ₽"
```

а не:

```python
50000
```

На этой странице мы решаем только задачу извлечения текста. Нормализация денег в число — отдельная стадия.

## 5. Что извлекается

### Название

Источник:

```html
<h1 class="product-title">...</h1>
```

Возможен вложенный тег:

```html
<h1 class="product-title"><b>Телевизор</b> Ultra HD</h1>
```

Результат:

```text
Телевизор Ultra HD
```

### Категория

Источник:

```html
<div class="product-category">...</div>
```

Результат — только текст внутри блока.

### Старая цена

Источник:

```html
<span class="price-old">...</span>
```

Внутри может быть:

```html
<strike>55 000 ₽</strike>
```

В результат не должен попасть тег `strike`.

Если блок пуст:

```html
<span class="price-old"></span>
```

результат должен быть:

```python
""
```

### Новая цена

Источник:

```html
<span class="price-new">...</span>
```

Это актуальная цена товара.

---

## 6. Почему вложенные теги важны

После первого поиска мы можем получить:

```html
<b>Телевизор</b> Ultra HD
```

Но это ещё не чистый текст. Необходимо удалить:

```html
<b>
</b>
```

и оставить:

```text
Телевизор Ultra HD
```

То же относится к:

```html
<strike>55 000 ₽</strike>
```

После очистки должно остаться:

```text
55 000 ₽
```

---

## 7. Почему используются два уровня regex

Решение разделяет две задачи.

### Поиск внешнего блока

Используются шаблоны:

```python
TITLE_PATTERN
CATEGORY_PATTERN
OLD_PRICE_PATTERN
NEW_PRICE_PATTERN
```

Они находят блоки по тегу и CSS-классу.

### Удаление вложенных тегов

Используется:

```python
HTML_TAG_PATTERN
```

Он удаляет элементы вроде:

```html
<b>
</b>
<strike>
</strike>
<i>
</i>
```

Разделение логики делает решение понятнее и проще для тестирования.

---

## 8. Почему шаблоны компилируются заранее

Вместо постоянного повторения:

```python
re.search(pattern, html)
```

используются глобальные объекты:

```python
TITLE_PATTERN = re.compile(...)
```

Преимущества:

- шаблон создаётся один раз;
- меньше повторений;
- выше читаемость;
- проще менять;
- удобно вызывать для большого числа карточек;
- соответствует подсказке задания о глобальных переменных.

Глобальными константами в решении являются:

```python
TITLE_PATTERN
CATEGORY_PATTERN
OLD_PRICE_PATTERN
NEW_PRICE_PATTERN
HTML_TAG_PATTERN
WHITESPACE_PATTERN
```

## 9. Почему решение устойчивее самого минимального варианта

Условие показывает разметку:

```html
<h1 class="product-title">
```

Но даже в учебных тестах могут встречаться:

```html
<h1 id="main" class="product-title">
```

```html
<h1 class="headline product-title active">
```

```html
<h1 class='product-title'>
```

```html
<H1 CLASS="product-title">
```

Решение поддерживает:

- дополнительные атрибуты;
- несколько CSS-классов;
- одинарные и двойные кавычки;
- переносы строк;
- разный регистр HTML-тегов;
- HTML-сущности;
- пустую старую цену.

## 10. Зачем нужны флаги

### `re.DOTALL`

Обычная точка в regex не проходит через перенос строки. Но HTML может быть многострочным:

```html
<h1 class="product-title">
    <b>Телевизор</b>
    Ultra HD
</h1>
```

`re.DOTALL` позволяет `.` совпадать и с переносом строки.

### `re.IGNORECASE`

Позволяет одинаково обрабатывать:

```html
<h1>
```

и:

```html
<H1>
```

---

## 11. Зачем нужен `html.unescape`

HTML может содержать сущности:

```html
TV &amp; Video
50&nbsp;000 ₽
```

Без декодирования результат был бы:

```text
TV &amp; Video
50&nbsp;000 ₽
```

После `unescape`:

```text
TV & Video
50 000 ₽
```

Это делает результат ближе к тексту, который видит пользователь в браузере.

---

## 12. Нормализация пробелов

HTML может содержать:

- переносы строк;
- табуляции;
- несколько пробелов подряд;
- неразрывные пробелы;
- пробелы по краям.

Шаблон:

```python
WHITESPACE_PATTERN = re.compile(r"\s+")
```

заменяет любую последовательность пробельных символов одним пробелом.

Метод `.strip()` убирает пробелы в начале и конце.

Пример:

```text
"\n   Телевизор      Ultra HD   \n"
```

становится:

```text
"Телевизор Ultra HD"
```

## 13. Место функции в промышленной системе

```text
HTML Fetcher
    ↓
parse_product_info
    ↓
словарь с текстовыми полями
    ↓
нормализация цены
    ↓
товарный матчинг
    ↓
валидация
    ↓
Price History / DWH
    ↓
BI или Pricing Engine
```

Функция имеет одну ответственность:

> Извлечь четыре поля из одной HTML-карточки.

Она не должна одновременно:

- загружать страницу;
- рассчитывать скидку;
- переводить валюту;
- сопоставлять товары;
- менять нашу цену;
- сохранять данные в БД.

Такое разделение называется separation of concerns.

## 14. Бизнесовый эффект ошибки

Конкурент показывает:

```html
<span class="price-old"><strike>55 000 ₽</strike></span>
<span class="price-new">50 000 ₽</span>
```

Если парсер ошибочно сохранит `55 000 ₽` как текущую цену, KarpovExpress может решить, что собственная цена `53 000 ₽` выгоднее конкурента.

Фактически конкурент дешевле на `3 000 ₽`.

Возможные последствия:

- падение конверсии;
- потеря заказов;
- неправильный price index;
- неверное решение категорийного менеджера;
- ошибочное обучение модели;
- недоверие к отчётности.

Обратная ошибка тоже опасна: если парсер принимает нерелевантное низкое число за цену, pricing engine может снизить собственную цену и потерять маржу.

## 15. Почему не используется BeautifulSoup

Для сложного production HTML специализированный парсер часто надёжнее регулярных выражений.

Но здесь:

- структура фиксирована;
- блоки гарантированно есть;
- урок посвящён `re`;
- проверяющая система ожидает один небольшой Python-файл;
- внешние библиотеки могут отсутствовать.

Поэтому достаточно стандартной библиотеки Python.
