# Урок 25 — Generators: бизнесовый и технический контекст

## 1. Почему генераторы важны для DA, DS и ML Engineer

Генератор — это не просто «функция с `yield`». В реальной системе это инструмент управления:

- оперативной памятью;
- временем до первого результата;
- обработкой потоковых данных;
- размером батча;
- нагрузкой на базу данных;
- нагрузкой на сеть;
- связью между этапами data pipeline;
- возможностью остановить вычисление раньше времени.

Главная инженерная мысль:

> Не всегда нужно сначала получить все данные, положить их в огромный список и только потом начать обработку. Часто правильнее получать один элемент или одну небольшую порцию, сразу обрабатывать её и переходить дальше.

Такой подход называется **ленивой обработкой**.

# 2. Бизнесовая проблема

Представим маркетплейс, где накоплено 500 миллионов событий:

- просмотры карточек;
- поисковые запросы;
- добавления в корзину;
- покупки;
- клики по рекомендациям;
- изменения цен;
- остатки товаров.

Если попытаться загрузить все события в обычный Python-список, программа может:

- занять всю RAM;
- начать использовать swap;
- резко замедлиться;
- завершиться с `MemoryError`;
- создать слишком большую нагрузку на DWH;
- сорвать SLA;
- остановить важный pipeline.

Бизнесовые последствия:

- рекомендации обновились слишком поздно;
- antifraud-модель не получила свежие признаки;
- отчёт не сформировался к началу рабочего дня;
- прогноз спроса построен на устаревших данных;
- маркетинговая кампания ушла неправильной аудитории;
- система понесла инфраструктурные расходы;
- пользователь получил медленный сервис.

Генератор позволяет обрабатывать поток постепенно:

```python
for event in event_generator():

```

В каждый момент в памяти может находиться только один объект `event`, а не сотни миллионов объектов.

# 3. Три разных понятия: iterable, iterator, generator

## 3.1. Итерируемый объект

**Итерируемый объект** — объект, по которому можно пройти циклом `for`.

Примеры:

```python
numbers = [10, 20, 30]
text = "python"
coordinates = (55.75, 37.61)
```

Итерируемый объект умеет предоставить итератор. Обычно для этого используется метод `__iter__()`.

## 3.2. Итератор

**Итератор** — объект, который хранит текущее положение в последовательности и умеет отдавать следующий элемент.

Для него важны методы:

```python
__iter__()
__next__()
```

На практике используют встроенные функции:

```python
iterator = iter([10, 20, 30])

print(next(iterator))
print(next(iterator))
print(next(iterator))
```

Результат:

```text
10
20
30
```

Следующий вызов:

```python
next(iterator)
```

приведёт к исключению:

```text
StopIteration
```

Это означает: элементы закончились.

## 3.3. Как работает `for`

Код:

```python
for number in [10, 20, 30]:
    print(number)
```

концептуально похож на:

```python
iterator = iter([10, 20, 30])

while True:
    try:
        number = next(iterator)
        print(number)
    except StopIteration:
        break
```

Цикл `for` сам получает итератор, вызывает `next()` и завершает работу при `StopIteration`.

## 3.4. Генератор

**Генератор** — удобный способ создать итератор без написания отдельного класса с `__iter__()` и `__next__()`.

```python
def simple_generator():
    yield 1
    yield 2
    yield 3
```

Вызов:

```python
generator = simple_generator()
```

создаёт генераторный объект. Значения ещё не вычислены полностью.

# 4. Что делает `yield`

Первый вызов:

```python
next(generator)
```

запускает функцию до первого `yield` и возвращает `1`.

Второй `next()` продолжает выполнение после первого `yield` и возвращает `2`.

Третий возвращает `3`.

Четвёртый приводит к `StopIteration`.

Важно:

- функция временно приостанавливается;
- её локальные переменные сохраняются;
- позиция выполнения сохраняется;
- следующий вызов продолжает выполнение с того же места.

Пример:

```python
def counter():
    value = 1

    while value <= 3:
        yield value
        value += 1
```

После выдачи `1` переменная `value` не исчезает. Она продолжает существовать внутри состояния генератора.

# 5. `return` и `yield`

## Обычная функция

```python
def get_number():
    return 10
```

`return`:

- возвращает итог;
- полностью завершает функцию;
- после него выполнение функции не продолжается.

## Генераторная функция

```python
def get_numbers():
    yield 10
    yield 20
```

`yield`:

- возвращает очередной элемент;
- приостанавливает функцию;
- сохраняет состояние;
- позволяет продолжить работу позже.

# 6. Файл как поток строк

Плохой вариант для файла размером 50 ГБ:

```python
with open("events.txt", "r") as file:
    lines = file.readlines()

for line in lines:
    process(line)
```

`readlines()` пытается загрузить все строки в память.

Более безопасный вариант:

```python
with open("events.txt", "r") as file:
    for line in file:
        process(line)
```

Файл выдаёт строки постепенно.

Это применяется для:

- backend-логов;
- транзакций;
- JSONL-файлов;
- товарных фидов;
- телеметрии;
- игровых событий;
- событий рекламной платформы;
- выгрузок из DWH.

# 7. Генератор не является готовым списком

```python
def numbers():
    yield 1
    yield 2
    yield 3
```

```python
result = numbers()
print(result)
```

Вы увидите объект генератора, а не список.

Чтобы получить значения:

```python
for value in result:
    print(value)
```

или:

```python
values = list(result)
```

Но `list(...)` загружает все элементы в память и тем самым убирает основное преимущество ленивости.

# 8. Генератор одноразовый

```python
generator = numbers()

print(list(generator))
print(list(generator))
```

Результат:

```text
[1, 2, 3]
[]
```

После первого полного прохода генератор исчерпан.

Чтобы пройти заново:

```python
print(list(numbers()))
print(list(numbers()))
```

каждый раз создаётся новый объект.

Это критично в аналитике. Ошибка:

```python
events = load_events()

count = sum(1 for _ in events)
revenue = sum(event["revenue"] for event in events)
```

После подсчёта `count` генератор уже пуст, поэтому `revenue` будет неверной.

# 9. Почему генераторы экономят память

Список:

```python
numbers = [number ** 2 for number in range(10_000_000)]
```

создаёт и хранит десять миллионов результатов.

Генераторное выражение:

```python
numbers = (number ** 2 for number in range(10_000_000))
```

хранит в основном:

- правило вычисления следующего значения;
- текущее состояние;
- служебную информацию.

Очередной квадрат вычисляется только по запросу.

Но важно:

> Генератор не гарантирует, что программа всегда станет быстрее. Он прежде всего меняет модель потребления памяти и время появления первого результата.

# 10. Time-to-first-result

Списочный подход:

1. получить все записи;
2. преобразовать все записи;
3. вернуть готовый список;
4. начать использовать результат.

Генераторный подход:

1. получить первую запись;
2. преобразовать первую запись;
3. сразу передать потребителю;
4. продолжить со следующей.

Это важно для:

- стриминговых API;
- онлайн-скоринга;
- выдачи рекомендаций;
- ETL;
- мониторинга;
- экспорта больших файлов;
- последовательной обработки очереди.

# 11. Разбор задания: `username_generator`

Требуется функция:

```python
username_generator(n, first_names=None, last_names=None)
```

Она должна:

1. принять количество записей `n`;
2. использовать переданные списки, если они есть;
3. создать внутренние списки, если получен `None`;
4. сформировать словарь каждого пользователя;
5. отдать словари по одному через `yield`;
6. присвоить последовательные ID.

Форма словаря:

```python
{
    "id": 1,
    "first_name": "Max",
    "last_name": "Garcia",
}
```

## Почему ID начинается с 1

В условии и примере первый пользователь имеет ID `1`.

Поэтому нужен:

```python
range(1, n + 1)
```

Для `n = 3` получаем:

```text
1, 2, 3
```

## Почему проверяется `is None`

Правильно:

```python
if first_names is None:
```

Нежелательно:

```python
if not first_names:
```

Второй вариант считает отсутствием и `None`, и пустой список. Но это разные состояния:

- `None` — список не передали;
- `[]` — пользователь явно передал пустой список.

## Почему `random.choice`

```python
random.choice(first_names)
```

возвращает один случайный элемент списка.

Повторение имён допустимо. Уникальным по условию должен быть именно ID.

# 13. Генераторные выражения и comprehensions

## List comprehension

```python
even_numbers = [
    number
    for number in range(10)
    if number % 2 == 0
]
```

Создаёт список сразу.

## Set comprehension

```python
unique_lengths = {
    len(word)
    for word in ["cat", "dog", "python"]
}
```

Создаёт множество и удаляет дубли.

## Dict comprehension

```python
squares = {
    number: number ** 2
    for number in range(4)
}
```

Создаёт словарь.

## Generator expression

```python
squares = (
    number ** 2
    for number in range(4)
)
```

Создаёт ленивый поток значений.

Главное визуальное отличие:

- `[]` — список;
- `{}` — множество или словарь;
- `()` — генераторное выражение.

# 14. Когда использовать список, а когда генератор

Используйте список, когда:

- результат небольшой;
- нужен повторный проход;
- нужен доступ по индексу;
- требуется сортировка всего набора;
- элементы используются много раз;
- нужна длина без исчерпания.

Используйте генератор, когда:

- набор может быть очень большим;
- данные поступают потоком;
- нужен один проход;
- потребитель может остановиться раньше;
- важна экономия памяти;
- строится цепочка обработки.

# 15. Pipeline из генераторов

```python
def read_rows(file_name):
    with open(file_name, "r") as file:
        for line in file:
            yield line


def clean_rows(rows):
    for row in rows:
        yield row.strip()


def non_empty_rows(rows):
    for row in rows:
        if row:
            yield row
```

Использование:

```python
rows = read_rows("events.txt")
rows = clean_rows(rows)
rows = non_empty_rows(rows)

for row in rows:
    save(row)
```

Каждая строка проходит цепочку по одной:

```text
source -> cleaning -> validation -> sink
```

Это упрощённая модель production data pipeline.

# 17. Генераторы в ML

```python
def batch_generator(data, batch_size):
```

# 18. Ограничения генераторов

Генератор не является универсальной заменой коллекциям.

## Нет доступа по индексу

Нельзя написать:

```python
generator[100]
```

## Одноразовость

После полного прохода объект исчерпан.

## Ошибки возникают поздно

```python
def broken_generator():
    yield 1
    raise ValueError("Ошибка")
```

Создание объекта пройдёт успешно. Ошибка возникнет только при дальнейшем чтении.

## Нет автоматического параллелизма

Генератор сам по себе:

- не создаёт потоки;
- не создаёт процессы;
- не делает код асинхронным;
- не распределяет вычисления.

Он только задаёт ленивый интерфейс.

# 19. Частые ошибки в этом задании

## Использовать `return` вместо `yield`

```python
def data_generator(n):
    result = []

    for x in range(n):
        result.append((x, random.randint(0, 100)))

    return result
```

Значения похожи на правильные, но функция не является генератором.

## Начать ID с нуля

```python
for user_id in range(n):
```

Получатся `0, 1, 2`, а нужны `1, 2, 3`.

## Исключить значение 100

```python
random.randrange(0, 100)
```

Правая граница не включена, поэтому `100` никогда не появится.

## Печатать вместо выдачи

```python
print(user)
```

Автотесты ожидают элементы генератора, а не вывод в консоль.

## Изменить API

Нельзя переименовывать функции, параметры или ключи словаря.

# 20. Воспроизводимость и random seed

Для повторяемости можно задать:

```python
random.seed(42)
```

Но в отправляемое решение seed добавлять не нужно, потому что:

- этого нет в задании;
- он меняет глобальное состояние `random`;
- автотест может сам управлять случайностью;
- лишний побочный эффект повышает риск.

В production часто используют отдельный генератор случайных чисел:

```python
rng = random.Random(42)
```

```python
def combined_generator():

```

Это сокращённая форма последовательной выдачи элементов другого iterable.

В текущем задании `yield from` не нужен.

# 22. Важное уточнение про встроенные функции

`range`, `enumerate`, `zip`, `map`, `filter` работают лениво или возвращают итерируемые объекты/итераторы.

Но технически не все они являются объектами типа `generator`.

Например, `range` — специальный объект последовательности.

На собеседовании лучше говорить:

> Эти инструменты поддерживают ленивую итерацию, но не каждый из них является генератором в строгом смысле Python type system.

# 24. Итог темы

Нужно запомнить пять вещей:

1. `yield` превращает функцию в генераторную функцию.
2. Вызов создаёт генераторный объект, но не вычисляет весь результат.
3. Значения выдаются по одному через `next()` или `for`.
4. Состояние между выдачами сохраняется.
5. После полного прохода генератор исчерпан.

В учебном решении код намеренно минимальный. В production могли бы добавиться:

- type hints;
- валидация параметров;
- логирование;
- метрики;
- batch processing;
- контроль seed;
- retry;
- checkpoint;
- тесты на границы;
- управление ресурсами.

Но для автогрейдера правильнее строго выполнить заданный API без лишней сложности.
