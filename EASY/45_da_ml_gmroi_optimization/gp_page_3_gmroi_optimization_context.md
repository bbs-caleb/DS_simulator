# 2. Бизнесовая задача

KarpovExpress работает в двух ролях:

1. Площадка для сторонних продавцов.
2. Продавец собственного ассортимента.

Продавец с идентификатором `0` — сам KarpovExpress.

## `seller_gmroi`

Показывает эффективность каждого продавца по неделям:

```text
week | seller_id | gmroi
```

Бизнес использует таблицу, чтобы:

- сравнивать продавцов;
- находить устойчиво эффективных партнёров;
- выявлять продавцов с избыточным запасом;
- выбирать кандидатов для дополнительного поискового трафика.

## `category_gmroi`

Показывает эффективность категорий:

```text
week | category_id | gmroi
```

Она помогает:

- анализировать ассортимент;
- сравнивать категории;
- видеть ухудшение оборачиваемости;
- принимать решения о закупках и сокращении запасов.

## `purchases`

Показывает недельные вложения KarpovExpress в собственный ассортимент:

```text
week | category_1 | category_3 | ...
```

Каждое значение — количество закупленных единиц, умноженное на себестоимость единицы.

# 4. Составной ключ

Товары, продажи и остатки объединяются по:

```python
["week", "seller_id", "product_id"]
```

Все три поля обязательны.

Если соединить только по `product_id`, один товар из разных недель и от разных продавцов начнёт соединяться со всеми совпадениями. Это создаст many-to-many merge и многократно увеличит суммы.

---

# 5. Формулы

## Маржа

\[
margin =
(price - cost) \times qty_{sales}
\]

## Стоимость запасов

\[
avg\_inventory\_cost =
qty_{stocks} \times cost
\]

В этой задаче используется актуальный недельный снимок запасов, а не классическая полусумма начала и конца периода.

## Хранение

\[
storage\_cost =
qty_{stocks} \times unit\_storage\_cost
\]

## GMROI

\[
gmroi =
\frac{
margin - logistic\_costs - storage\_cost
}{
avg\_inventory\_cost
}
\]

Итог округляется до двух знаков.

---

# 6. Правильная агрегация

GMROI нельзя сначала рассчитать для каждого товара, а потом усреднить.

Правильная формула группы:

\[
GMROI_{group} =
\frac{
\sum margin -
\sum logistic\_costs -
\sum storage\_cost
}{
\sum avg\_inventory\_cost
}
\]

Сначала суммируются денежные компоненты, затем вычисляется отношение.

---

# 7. Логика закупок

`purchases.csv` не содержит `seller_id`, `category_id` и `cost`.

Поскольку покупки относятся только к KarpovExpress, сначала из `products.csv` выбираются строки:

```python
seller_id == 0
```

Затем покупки соединяются с ними по:

```python
["week", "product_id"]
```

Стоимость закупки:

\[
purchase\_cost = qty \times cost
\]

После группировки по неделе и категории выполняется `pivot`.
