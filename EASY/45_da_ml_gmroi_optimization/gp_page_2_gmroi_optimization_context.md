# 2. Бизнесовый смысл

KarpovExpress одновременно работает как:

- маркетплейс для сторонних продавцов;
- самостоятельный продавец собственного ассортимента.

Продавец с `seller_id = 0` — сам KarpovExpress.

## По продавцам

Таблица `seller_gmroi` отвечает на вопрос:

> Насколько эффективно каждый продавец превращает деньги, вложенные в запас, в прибыль после логистики и хранения?

Формат:

```text
week | seller_id | gmroi
```

## По категориям

Таблица `category_gmroi` отвечает на вопрос:

> Какие категории лучше используют товарные запасы?

Формат:

```text
week | category_id | gmroi
```

## По закупкам

Таблица `purchases` показывает:

> Сколько KarpovExpress потратил на закупку каждой категории в каждую неделю?

Формат:

```text
week | category_1 | category_3 | ...
```

# 4. Ключ объединения

Для таблиц товаров, продаж и остатков используется ключ:

```python
["week", "seller_id", "product_id"]
```

Почему нужны все три поля:

- один товар встречается в разных неделях;
- один товар продают разные продавцы;
- цена может отличаться у продавцов;
- цена и себестоимость могут меняться по неделям.

Merge только по `product_id` размножил бы строки и испортил суммы.

# 5. Формулы

## Маржа

\[
margin = (price - cost) \times sales\_qty
\]

## Стоимость запаса

По условию задачи:

\[
avg\_inventory\_cost = stock\_qty \times cost
\]

Это не классическая средняя начала и конца периода. Задание явно просит использовать актуальный недельный остаток.

## Хранение

\[
storage\_cost =
stock\_qty \times unit\_storage\_cost
\]

## GMROI

\[
gmroi =
\frac{
margin - logistic\_costs - storage\_cost
}{
avg\_inventory\_cost
}
\]

После расчёта значение округляется до двух знаков.

---

# 6. Почему GMROI считается после группировки

Неправильно:

```python
data["gmroi"] = ...
data.groupby(...).gmroi.mean()
```

Так маленький и крупный товар получили бы одинаковый вес.

Правильно:

1. Рассчитать компоненты по каждому товару.
2. Суммировать их по продавцу или категории.
3. Разделить суммарный числитель на суммарный знаменатель.

\[
GMROI_{group} =
\frac{
\sum margin -
\sum logistic\_costs -
\sum storage\_cost
}{
\sum avg\_inventory\_cost
}
\]

# 7. Особенность закупок

В `purchases.csv` нет:

- продавца;
- категории;
- себестоимости.

Поэтому покупки соединяются с товарами продавца `0`:

```python
products_df[products_df["seller_id"] == 0]
```

Фильтр обязателен. Без него одна покупка соединится с карточками того же товара всех продавцов.

Стоимость закупки:

\[
purchase\_cost = purchase\_qty \times cost
\]

После группировки по неделе и категории применяется `pivot`.
