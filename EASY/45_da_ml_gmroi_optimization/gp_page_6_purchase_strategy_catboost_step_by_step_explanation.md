# 1. Новый импорт

```python
from catboost import CatBoostRegressor
```

`CatBoostRegressor` — модель градиентного бустинга для задач регрессии.

Регрессия означает, что модель предсказывает число.

В нашей задаче это число:

```text
GMROI
```

---

Новый метод добавляется внутрь класса:

```python
class CategoryImpactEval(GMROI):
```

Название:


# 3. Определение признаков

```python
feature_columns = [
    column
    for column in self._eval_data.columns
    if column not in ["week", "gmroi"]
]
```

Таблица `_eval_data` содержит:

```text
week | 1 | 3 | 5 | 8 | 9 | gmroi
```

Из неё исключаются:

- `week`;
- `gmroi`.

Остаются:

```python
[1, 3, 5, 8, 9]
```

Это категории закупок.

---

# 4. Создание X

```python
features = self._eval_data[feature_columns]
```

`features` — таблица признаков.

Она выглядит примерно так:

```text
1        3        5        8        9
5129792  9231286  4241830  6895368  7395210
5335607  8854405  4293261  6515897  7403186

```

Каждая строка — неделя.

Каждый столбец — расходы на закупки категории.

# 5. Создание y

```python
target = self._eval_data["gmroi"]
```

`target` — правильные ответы для модели.

# 6. Создание модели

```python
model = CatBoostRegressor(
    random_seed=42,
    verbose=False,
)
```

## `random_seed=42`

Фиксирует случайность.

Это нужно, чтобы при повторном запуске модель давала одинаковый результат.

## `verbose=False`

Отключает вывод сотен строк обучения.

---

# 7. Почему нет StandardScaler

Для линейной регрессии использовалось:

```python
StandardScaler()
```

Для деревьев это не требуется.

Дерево ищет правила типа:

```text
если покупки категории 8 > 6 700 000
```

Изменение масштаба не меняет порядок значений.

Поэтому признаки передаются в CatBoost без стандартизации.

---

# 8. Обучение

```python
model.fit(features, target)
```

Модель получает:

- закупки по категориям;
- соответствующий GMROI.

Она строит множество деревьев решений.

---

# 9. Как дерево принимает решение

Упрощённое дерево может выглядеть так:

```text
Покупки категории 8 > 6.8 млн?
├── Да:
│   Покупки категории 9 > 7.4 млн?
│   ├── Да: прогноз 1.54
│   └── Нет: прогноз 1.50
└── Нет:
    прогноз 1.47
```

Реальный CatBoost строит много таких деревьев и складывает их прогнозы.

---

# 10. Получение прогнозов

```python
predictions = model.predict(features)
```

Модель предсказывает GMROI для тех же недель, на которых обучалась.

Это важно:

> Здесь оценивается качество на обучающих данных.

Именно поэтому R² может быть почти идеальным.

---

# 11. Расчёт R²

```python
r2_score(target, predictions)
```

R² показывает, какую долю вариации целевой переменной объясняет модель.

Результат:

```text
0.9998
```

Это примерно:

```text
99.98%
```

вариации обучающих значений.

# 15. Почему используется `float`

CatBoost может вернуть значения NumPy-типа.

```python
float(importance)
```

превращает их в обычный Python `float`.

Это полезно для:

- автотестов;
- сериализации;
- предсказуемого API.

---

# 16. Сортировка словаря

```python
sorted(
    importances.items(),
    key=lambda item: item[1],
    reverse=True,
)
```

## `importances.items()`

Возвращает пары:

```text
(category_id, importance)
```

## `key=lambda item: item[1]`

Сортирует по второму элементу пары — importance.

## `reverse=True`

Ставит самые большие значения первыми.

# 19. Что означает importance 23.2793

Это не означает:

- рост закупок на 1 рубль увеличивает GMROI на 23;
- категория приносит 23% прибыли;
- нужно увеличить закупки на 23%;
- эффект положительный.

Это означает:

> Внутри обученной модели категория 8 получила примерно 23.28% относительной важности.

---

# 20. Почему importance не показывает знак

Линейный коэффициент может быть:

- положительным;
- отрицательным.

Feature importance показывает только силу использования признака.

Чтобы понять направление, нужны:

- Partial Dependence Plot;
- SHAP values;
- сценарный прогноз;
- ALE plot.

---

# 21. Почему результат может быть переобучением

В выборке примерно 51 неделя и 5 признаков.

CatBoost по умолчанию строит много деревьев.

Он может очень хорошо запомнить маленький набор данных.

Поэтому:

```text
training R² = 0.9998
```

не гарантирует:

```text
test R² = 0.9998
```

На новых неделях качество может быть намного ниже.

---

# 22. Как правильно проверить в реальном проекте

Пример временного разделения:

```python
train = eval_data.iloc[:-10]
test = eval_data.iloc[-10:]
```

Модель обучается на `train`.

Качество считается на `test`.

Нельзя случайно перемешивать недели, потому что это временной ряд.
