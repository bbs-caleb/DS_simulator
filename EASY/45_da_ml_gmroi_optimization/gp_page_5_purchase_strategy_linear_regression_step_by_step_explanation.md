# 8. Создание StandardScaler

```python
scaler = StandardScaler()
```

Объект пока ничего не знает о данных.

---

# 9. Что делает `fit_transform`

```python
scaled_features = scaler.fit_transform(features)
```

Внутри происходят два действия.

## `fit`

Scaler вычисляет для каждой категории:

- среднее значение;
- стандартное отклонение.

## `transform`

Каждое значение преобразуется:

\[
z =
\frac{x - mean}{std}
\]

---

# 10. Зачем стандартизировать

Категории имеют разные объёмы закупок.

Например:

```text
категория 1 — около 5 млн
категория 3 — около 9 млн
```

Без стандартизации коэффициенты измеряли бы влияние одного рубля и получились бы очень маленькими.

После стандартизации изменение признака на `1` означает рост примерно на одно стандартное отклонение категории.

---

# 11. Создание модели

```python
model = LinearRegression()
```

Это объект линейной регрессии из sklearn.

---

# 12. Обучение

```python
model.fit(scaled_features, target)
```

Модель ищет коэффициенты, которые минимизируют сумму квадратов ошибок:

\[
\sum (y_i - \hat{y}_i)^2
\]

---

# 13. Прогноз

```python
predictions = model.predict(
    scaled_features
)
```

Модель рассчитывает прогноз GMROI для каждой недели.

---

# 14. R²

```python
r2_score(target, predictions)
```

Формула:

\[
R^2 =
1 -
\frac{
\sum (y_i - \hat{y}_i)^2
}{
\sum (y_i - \bar{y})^2
}
\]

## Интерпретация

- `1` — идеальное объяснение данных;
- `0` — модель не лучше среднего;
- меньше `0` — модель хуже среднего;
- `0.0402` — объяснено около 4.02% вариации.

---

# 15. Коэффициенты

```python
model.coef_
```

Возвращает массив коэффициентов в том же порядке, что и признаки.

Например:

```text
feature_columns = [1, 3, 5, 8, 9]
model.coef_    = [-0.002, -0.0017, ...]
```

---

# 16. Что делает `zip`

```python
zip(feature_columns, model.coef_)
```

Объединяет значения попарно:

```text
1 ↔ -0.002
3 ↔ -0.0017
5 ↔ 0.0038
8 ↔ -0.0001
9 ↔ -0.0051
```

---

# 17. Создание словаря коэффициентов

```python
coefficients = {
    column: round(float(coefficient), 4)
    for column, coefficient in zip(
        feature_columns,
        model.coef_,
    )
}
```

Получаем:

```python
{
    1: -0.002,
    3: -0.0017,
    5: 0.0038,
    8: -0.0001,
    9: -0.0051,
}
```

`float(...)` превращает NumPy-числа в обычные Python `float`.

---

# 18. Свободный член

```python
model.intercept_
```

Добавляется отдельно:

```python
coefficients["intercept"] = round(
    float(model.intercept_),
    4,
)
```

После стандартизации intercept приблизительно равен среднему GMROI.

---

# 19. Возвращаемый словарь

```python
return {
    "r2_score": ...,
    "coeffs": coefficients,
}
```

Структура должна быть точной, потому что автотесты проверяют API.

---

# 20. Почему коэффициенты близки к нулю

Это не обязательно означает, что закупки бесполезны.

Возможные причины:

- недельный GMROI слабо меняется;
- влияние закупок проявляется позже;
- связь нелинейная;
- признаки коррелируют друг с другом;
- GMROI зависит от множества других факторов;
- наблюдений мало.

---

# 21. Почему нельзя делать причинный вывод

Линейная регрессия на наблюдательных данных показывает ассоциацию.

Она не доказывает:

> Закупки категории 9 вызывают снижение GMROI.

Возможна обратная причинность:

- бизнес увеличивает закупки, когда ожидает изменения спроса;
- товар закупается сейчас, а продаётся позже;
- сезон одновременно влияет на закупки и GMROI.
