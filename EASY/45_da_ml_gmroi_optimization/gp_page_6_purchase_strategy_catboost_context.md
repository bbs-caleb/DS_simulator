# 2. Бизнесовая постановка

KarpovExpress хочет понять:

> Какие категории закупок сильнее всего связаны с изменением GMROI собственного ассортимента?

В модели используются недельные расходы на закупки категорий:

```text
1, 3, 5, 8, 9
```

Целевая переменная:

```text
gmroi
```

Идея:

- если закупки категории сильно помогают модели объяснять GMROI, её importance будет выше;
- если признак почти не используется моделью, importance будет ниже.

---

# 3. Почему CatBoost может работать лучше линейной регрессии

Линейная регрессия предполагает зависимость вида:

\[
y =
\beta_0 +
\beta_1 x_1 +
... +
\beta_n x_n
\]

То есть изменение признака имеет постоянный линейный эффект.

Но в реальном бизнесе зависимость может быть сложнее.

## Насыщение

Небольшое увеличение закупок может улучшать GMROI, но после определённого объёма дополнительный запас начинает замораживать капитал.

## Пороговый эффект

До определённого уровня запас недостаточен, а после достижения порога продажи резко растут.

## Взаимодействие категорий

Категория 1 может давать эффект только при достаточных закупках категории 8.

## Разные режимы

В сезон и вне сезона одинаковая сумма закупок может давать разный результат.

Деревья решений способны разбивать пространство признаков на участки и моделировать такие зависимости.

---

# 4. Что такое градиентный бустинг

Градиентный бустинг строит не одно дерево, а последовательность деревьев.

Логика:

1. Первое дерево делает грубый прогноз.
2. Вычисляются ошибки.
3. Следующее дерево учится исправлять ошибки первого.
4. Процесс повторяется много раз.
5. Итоговый прогноз — сумма вкладов всех деревьев.

Упрощённо:

\[
Prediction =
Tree_1 +
Tree_2 +
... +
Tree_m
\]

Каждое новое дерево исправляет ошибки предыдущих.

---

# 5. Что такое CatBoost

CatBoost — библиотека градиентного бустинга от Яндекса.

Она особенно известна хорошей работой с категориальными признаками, но также применяется к обычным числовым данным.

В этой задаче все признаки числовые:

```text
расходы на закупки категорий
```

Используется:

```python
CatBoostRegressor(
    random_seed=42,
    verbose=False,
)
```

Все остальные параметры остаются значениями по умолчанию.

---

# 6. Почему задаётся random_seed=42

Обучение бустинга включает случайные элементы.

Без фиксации seed:

- результат может немного меняться;
- feature importances могут отличаться;
- автотесты могут быть нестабильными.

Поэтому используется:

```python
random_seed=42
```

Это обеспечивает воспроизводимость.

---

# 7. Почему используется verbose=False

CatBoost по умолчанию печатает процесс обучения:

```text
0: learn ...
1: learn ...
2: learn ...
```

Для учебного решения этот вывод не нужен.

Поэтому добавляется:

```python
verbose=False
```

Это не меняет модель, а только отключает лишние сообщения.

# 9. Почему StandardScaler не используется для CatBoost

Деревья решений не зависят от масштаба признаков так, как линейная регрессия.

Дерево сравнивает значения с порогами:

```text
purchase_category_8 > threshold
```

Поэтому стандартизация не обязательна.

Это отличие от линейной регрессии, где масштаб сильно влияет на коэффициенты.

---

# 10. Что такое feature importance

После обучения CatBoost возвращает:

```python
model.feature_importances_
```

Это относительный вклад каждого признака в построение модели.

В этой задаче сумма importance примерно равна:

```text
100%
```

Результат:

```text
category 8 -> 23.2793
category 9 -> 22.1107
category 3 -> 18.8606
category 1 -> 18.7533
category 5 -> 16.9962
```

---

# 11. Бизнесовая интерпретация

Наиболее важные категории:

```text
8 и 9
```

Их суммарная важность:

\[
23.2793 + 22.1107 = 45.39\%
\]

Это означает, что модель чаще использовала закупки этих категорий при объяснении недельного GMROI.

Но feature importance не показывает направление эффекта.

Она не отвечает:

- увеличение закупок повышает GMROI;
- увеличение закупок снижает GMROI.

Она показывает только:

> Насколько признак важен для предсказаний модели.

---

# 12. Почему нельзя напрямую принимать закупочное решение

Нельзя сказать:

> Категория 8 самая важная, поэтому нужно увеличить её закупки.

Причины:

- importance не показывает знак эффекта;
- модель оценена на тех же данных, на которых обучалась;
- возможен сильный overfitting;
- всего около 51 недели;
- не учтены лаги;
- важность не доказывает причинность.

Правильный бизнесовый вывод:

> Категории 8 и 9 являются наиболее значимыми для предсказаний модели и требуют приоритетного дальнейшего анализа, но решение об увеличении или сокращении закупок нельзя принимать только по feature importance.

---

# 13. Критически важная оговорка про R² = 0.9998

Очень высокое значение выглядит впечатляюще:

```text
R² = 0.9998
```

Но модель:

- обучалась на всей выборке;
- прогнозировала ту же выборку;
- не проверялась на будущих неделях.

Поэтому это training R², а не честная оценка качества на новых данных.

CatBoost достаточно гибкий, чтобы почти запомнить маленький набор из примерно 51 строки.

В production нужно использовать временное разделение:

```text
train = ранние недели
test = последние недели
```

И оценивать R² на test.
