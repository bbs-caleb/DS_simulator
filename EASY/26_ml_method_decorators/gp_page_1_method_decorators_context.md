## 3. Какие бизнесовые вопросы решает такой модуль

### 3.1 Какая ставка действует сейчас

Банк должен знать текущую ставку продукта:

```text
15%
```

После решения продуктового или финансового комитета ставка может измениться:

```text
16%
```

### 3.2 Каков средний баланс клиентов

Средний баланс помогает ответить на вопросы:

- сколько денег в среднем хранит клиент;
- как меняется поведение клиентов;
- растёт ли депозитная база;
- насколько продукт интересен аудитории;
- какова стоимость привлечённых средств;
- какие сегменты наиболее ценны;
- как ставка влияет на объём средств.

### 3.3 Сколько процентов нужно начислить клиенту

Для каждого счёта нужен расчёт:

```text
проценты = баланс × ставка / 100
```

В задаче формула намеренно упрощена.

## 5. Что такое класс

Класс — описание будущих объектов.

```python
class BankMetrics:
    ...
```

Можно представить класс как чертёж.

Он говорит:

- какие данные будут у объектов;
- какие общие данные будут у класса;
- какие операции доступны;
- как создаётся новый объект.

Сам класс не является конкретным счётом Tom или Jerry.

---

## 6. Что такое экземпляр

Экземпляр — конкретный объект, созданный по классу.

```python
account1 = BankMetrics("Tom", 15000)
```

Здесь:

- `BankMetrics` — класс;
- `account1` — объект;
- `"Tom"` — имя владельца;
- `15000` — баланс.

Второй объект:

```python
account2 = BankMetrics("Jerry", 20000)
```

имеет собственное имя и собственный баланс.

---

## 7. Атрибуты экземпляра

В конструкторе записываются:

```python
self.name = name
self.balance = balance
```

Эти значения принадлежат конкретному объекту.

Для Tom:

```text
name    = "Tom"
balance = 15000
```

Для Jerry:

```text
name    = "Jerry"
balance = 20000
```

Если изменить баланс Tom, баланс Jerry не должен измениться.

---

## 8. Атрибуты класса

Внутри тела класса объявлены:

```python
global_bank_rate: float = 15.0
accounts: List[BankMetrics] = []
```

Это атрибуты класса.

### `global_bank_rate`

Общая банковская ставка.

Она относится ко всему классу `BankMetrics`.

### `accounts`

Общий список созданных объектов.

После создания трёх счетов в нём находятся ссылки на:

- Tom;
- Jerry;
- Spike.

---

## 9. Что такое `self`

`self` — ссылка на конкретный объект.

Когда выполняется:

```python
BankMetrics("Tom", 15000)
```

Python создаёт объект и передаёт его в `__init__` как `self`.

Упрощённо:

```text
self    → новый объект Tom
name    → "Tom"
balance → 15000
```

Строка:

```python
self.balance = balance
```

записывает баланс именно в текущий объект.

---

## 10. Что такое `cls`

`cls` — ссылка на класс.

Метод класса объявляется так:

```python
@classmethod
def method(cls):
    ...
```

При вызове:

```python
BankMetrics.method()
```

Python автоматически передаёт:

```text
cls → BankMetrics
```

Через `cls` можно обращаться к:

```python
cls.accounts
cls.global_bank_rate
```

---

## 11. Обычный метод экземпляра

Обычный метод первым аргументом получает `self`.

Пример:

```python
def deposit(self, amount: float) -> None:
    self.balance += amount
```

Этот метод меняет один конкретный счёт.

В нашей задаче обычным методом является конструктор:

```python
__init__
```

---

## 12. Что делает `@staticmethod`

Статический метод:

- не получает `self`;
- не получает `cls`;
- получает только явно переданные аргументы.

Пример:

```python
@staticmethod
def adjust_global_bank_rate(new_rate: float) -> None:
    BankMetrics.global_bank_rate = new_rate
```

В метод попадает только:

```text
new_rate
```

Он меняет общую ставку через явное имя класса.

---

## 13. Почему изменение ставки — статический метод

Точный контракт задания требует:

```python
@staticmethod
def adjust_global_bank_rate(new_rate: float) -> None:
```

Метод не использует `self` или `cls`.

Он выполняет одно действие:

```python
BankMetrics.global_bank_rate = new_rate
```

Это хороший пример того, почему при решении задач нужно учитывать не только теоретически возможный дизайн, но и точный API.

В production-коде такую операцию часто можно было бы оформить и как `@classmethod`, однако в данной задаче проверяется конкретная структура.

---

## 14. Что делает `@classmethod`

Метод класса автоматически получает `cls`.

```python
@classmethod
def calculate_avg_balance(cls) -> float:
    ...
```

Это позволяет работать с общими данными:

```python
cls.accounts
```

---

## 15. Почему средний баланс — метод класса

Средний баланс относится не к одному объекту.

Он вычисляется по всем счетам:

```python
cls.accounts
```

Поэтому метод получает класс и читает общий реестр.

Формула:

```text
средний баланс =
сумма всех балансов / количество счетов
```

---

## 16. Почему расчёт процентов — метод класса

Метод получает:

- класс через `cls`;
- конкретный счёт через `account`.

```python
@classmethod
def calculate_interest(
    cls,
    account: BankMetrics,
) -> float:
```

Ставка читается так:

```python
cls.global_bank_rate
```

Баланс читается так:

```python
account.balance
```

В результате метод объединяет:

- общее состояние класса;
- состояние переданного объекта.

---

## 17. Точная схема методов

| Метод | Декоратор | Автоматический аргумент | Что использует |
|---|---|---|---|
| `__init__` | нет | `self` | данные одного счёта |
| `adjust_global_bank_rate` | `@staticmethod` | нет | новое значение ставки |
| `calculate_avg_balance` | `@classmethod` | `cls` | общий список счетов |
| `calculate_interest` | `@classmethod` | `cls` | общую ставку и переданный счёт |

---

## 18. Как создаётся счёт

Выполняется:

```python
account1 = BankMetrics("Tom", 15000)
```

Последовательность:

1. Python создаёт новый объект.
2. Вызывает `__init__`.
3. Передаёт объект как `self`.
4. Сохраняет имя.
5. Сохраняет баланс.
6. Добавляет объект в общий список.
7. Возвращает ссылку на объект.
8. Ссылка записывается в `account1`.

---

## 19. Почему в список добавляется `self`

Код:

```python
BankMetrics.accounts.append(self)
```

Добавляет сам объект.

Это важно.

Если добавить только баланс:

```python
BankMetrics.accounts.append(balance)
```

то список будет содержать числа.

Тогда выражение:

```python
account.balance
```

не сработает, потому что у числа нет атрибута `balance`.

---

## 20. Как считается средний баланс

Код:

```python
total_balance = sum(
    account.balance
    for account in cls.accounts
)
```

Генератор по очереди выдаёт:

```text
15000
20000
10000
```

`sum` складывает:

```text
45000
```

Количество счетов:

```python
len(cls.accounts)
```

равно:

```text
3
```

Среднее:

```text
45000 / 3 = 15000
```

---

## 21. Как считается процент

После изменения ставки на 16%:

```text
balance = 15000
rate    = 16
```

Формула:

```text
15000 × 16 / 100 = 2400
```

Код:

```python
return account.balance * cls.global_bank_rate / 100
```

## 22. Аннотации типов

### `global_bank_rate: float`

Ставка представлена числом с плавающей точкой.

### `accounts: List[BankMetrics]`

Список должен содержать объекты `BankMetrics`.

### `name: str`

Имя является строкой.

### `balance: float`

Баланс является числом.

### `-> None`

Метод ничего не возвращает.

### `-> float`

Метод возвращает число с плавающей точкой.

Аннотации улучшают:

- читаемость;
- подсказки IDE;
- статическую проверку;
- документацию;
- поддержку кода.

---

## 23. Зачем нужен `from __future__ import annotations`

В аннотации используется имя класса внутри его собственного определения:

```python
accounts: List[BankMetrics]
```

Отложенные аннотации позволяют Python не вычислять такую ссылку немедленно.

Без этого в некоторых версиях Python пришлось бы писать:

```python
accounts: List["BankMetrics"]
```

## 24. Почему API важен так же, как результат

Два решения могут возвращать одинаковые числа, но иметь разную структуру.

Автотест может отдельно проверять:

- имя класса;
- имена атрибутов;
- сигнатуры методов;
- декораторы;
- значения;
- PEP8.

Поэтому недостаточно получить:

```text
2400.0
```

Нужно ещё реализовать метод именно в ожидаемой форме.

Это очень похоже на production-разработку:

- API должен быть стабильным;
- клиенты зависят от сигнатур;
- другие модули ожидают конкретный контракт;
- изменение интерфейса может сломать систему.

## 34. Что в учебном коде нельзя переносить напрямую в production

### 34.1 Деньги через `float`

Для денег чаще используют:

```python
Decimal
```

или целые минимальные единицы.

Причина — точность и правила округления.

### 34.2 Глобальный список в памяти

```python
accounts = []
```

не подходит для распределённой системы:

- данные исчезают после перезапуска;
- разные серверы видят разные списки;
- нет транзакций;
- нет резервного копирования;
- нет аудита;
- нет конкурентной согласованности.

### 34.3 Перезапись ставки без истории

Нужно знать:

- когда изменилась ставка;
- кто изменил;
- к какому продукту относится;
- по какой версии считались проценты.

### 34.4 Нет валидации

Учебный код допускает отрицательный баланс или странную ставку.

В production должны быть правила и проверки.

### 34.5 Нет обработки пустого списка

Если счетов нет, деление вызовет ошибку.

В production контракт нужно определить явно.

### 34.6 Нет валюты

Баланс 15000 без валюты неоднозначен.

В реальной системе нужно хранить:

- UZS;
- USD;
- EUR;
- другие валюты.

Нельзя бездумно усреднять балансы в разных валютах.

---

## 35. Типичные ошибки

### Неверный декоратор

Даже при правильном результате структура метода может не соответствовать API.

### Неверный первый аргумент

Для `@classmethod` первым аргументом должен быть `cls`.

### Добавление баланса вместо объекта

В список должен добавляться `self`.

### Сумма вместо среднего

Нужно делить на число счетов.

### Процент без деления на 100

Ставка 16% означает `0.16`, а не `16`.

### Создание тестовых счетов при импорте

Примеры должны находиться внутри:

```python
if __name__ == "__main__":
```

### Переименование методов

Проверяющая система не найдёт ожидаемый API.

## 36. Почему решение должно быть минимальным

В учебной задаче не нужны:

- база данных;
- логирование;
- сложная валидация;
- сторонние библиотеки;
- обработка валют;
- календарь начислений;
- история ставок;
- web API.

Минимальное решение лучше, потому что:

- проще понять;
- проще проверить;
- меньше вероятность ошибки;
- оно соответствует условию;
- оно не добавляет неоговорённое поведение.

## 37. Итоговая модель мышления

Перед написанием метода задайте вопросы:

1. Нужен ли конкретный объект?
2. Нужен ли класс?
3. Нужны ли только переданные аргументы?
4. Какой API ожидает система?
5. Что относится к индивидуальному состоянию?
6. Что относится к общему состоянию?
7. Что является вычислением?
8. Как это будет масштабироваться?

Для этой задачи:

```text
создание счёта                → обычный метод
изменение общей ставки        → staticmethod
среднее по общему реестру     → classmethod
проценты по ставке класса     → classmethod
```

## 38. Главный профессиональный вывод

Декораторы методов — это не просто символы над функцией.

Они описывают:

- какой контекст получает метод;
- где находится состояние;
- кто владеет данными;
- какие зависимости видны из интерфейса;
- как метод будет вызываться;
- как код будет расширяться.

Для DS/MLE это переносится на:

- классы моделей;
- конфигурации экспериментов;
- registry моделей;
- feature transformers;
- inference clients;
- pipeline components;
- metric calculators;
- data validators;
- сервисные SDK;
- production ML-системы.
